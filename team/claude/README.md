# Claude — Implementer Agent Area

This folder is the persistent working area for the implementer agent (Claude, running via Claude Code).

Purpose:
- `debriefs/` — one file per completed implementation slice. Captures what was done, what deviated from the brief, issues encountered, bugs fixed, and lessons learned. Written as the work progresses, not after the fact.
- `observations/` — captured comments and observations left in the codebase or in commit messages by the human (e.g. Dinis's `@dev` notes). Once an observation is captured here and rolled into the dev/other guidelines, the original inline comment can be deleted.

Index:
- `debriefs/index.md` — chronological list of debriefs with one-line summaries.
- `observations/index.md` — chronological list of observations with status (captured / rolled-into-guidelines / deleted-from-source).
