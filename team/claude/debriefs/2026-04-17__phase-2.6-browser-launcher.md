## What was delivered

### `service/Browser__Launcher`

The only class permitted to import `playwright.sync_api` outside of `Step__Executor`. Public surface:

| Method | Purpose |
|--------|---------|
| `start()` | Lazy-init `sync_playwright().start()`. Idempotent. |
| `launch(browser_config)` → `Browser` | Start runtime, validate provider + browser name, build launch kwargs, call `chromium.launch(**kwargs)`. |
| `register(session_id, browser)` | Track per-session for later `stop()`. |
| `stop(session_id)` | Close that session's browser; swallow close errors (browser may already be dead); remove from registry. |
| `stop_all()` | Close every registered browser, then `playwright.stop()`. Shutdown hook. |
| `healthcheck()` → `Schema__Health__Check` | `playwright_started={bool} active_browsers={N}`. Always `healthy=True`. |
| `active_session_ids()` | Read-only view of registry keys. |

### Env-var escape hatch

New constant `ENV_VAR__CHROMIUM_EXECUTABLE = 'SG_PLAYWRIGHT__CHROMIUM_EXECUTABLE'` in `consts/env_vars.py`. When set, the path is passed to `chromium.launch(executable_path=...)`. Three motivating use cases:

1. **This sandbox** — Python `playwright` package is 1.58.0 (matches the project's Docker base image) but the pre-installed Chromium revision on disk is 1194, while 1.58.0's default resolver looks for 1208. The env var lets us point at rev 1194 without reinstalling 180 MB on every fresh session.
2. **Dev laptops** — developers can pin to system Chrome or a custom build.
3. **CI Docker image** — unset → default Playwright resolution works (Dockerfile runs `playwright install`).

### Guarded surface

- `assert_provider_supported` — rejects `CDP_CONNECT` / `BROWSERLESS` as `NotImplementedError` (LOCAL_SUBPROCESS only today).
- `assert_browser_supported` — rejects `FIREFOX` / `WEBKIT` as `NotImplementedError` (CHROMIUM only today).

Both will expand when multi-browser / CDP support lands. The explicit `NotImplementedError` (rather than silent failure) means misconfigured sessions fail fast with a readable message.

### Proxy flattening

`Schema__Proxy__Config` → Playwright's `proxy` dict:
- `server` always included
- `username` / `password` only when both present
- `bypass` list joined with `,` per Playwright's API

## Deviations from brief

- v1 source not in pack — surface derived from callsites (spec lines 453, 469, 507) plus "owns Chromium process lifecycle" rule on line 1222. The resulting shape is minimal.
- `ENV_VAR__CHROMIUM_EXECUTABLE` is a new env var not in spec §7. Added because the sandbox's Chromium revision doesn't match Playwright 1.58.0's default. Documented in this debrief and in the env_vars module.
- `assert_provider_supported` / `assert_browser_supported` raise `NotImplementedError` (not `HTTPException`), since they guard a programmer-facing invariant, not a user-facing request. `Request__Validator` is the class that turns caller mistakes into 422s.
