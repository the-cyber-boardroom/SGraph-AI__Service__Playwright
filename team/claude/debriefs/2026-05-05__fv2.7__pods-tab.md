## What was done

Migrated the nodes-view Pods tab from direct sidecar HTTP calls (cross-origin) to same-origin control-plane proxy calls. Three methods updated: `_fetchContainers`, `_fetchPodLogs`, `_pollLiveLogs`. Response schema adapted for `Schema__Pod__Info` field names. `_portLinks` updated for Docker string port format.

### Files changed

| File | Change |
|------|--------|
| `sp-cli-nodes-view.js` | `_fetchContainers()`: pod list now from `GET /api/nodes/{id}/pods/list` via `apiClient`; host/status still direct sidecar (no proxy yet); `_renderContainers()`: adapts `pod_name`/`state` → `name`/`status`; `role="listitem"` on pod rows; `_portLinks()`: parse Docker string format `"ip:hostPort->containerPort/proto"` instead of dict; `_fetchPodLogs()`: uses `GET /api/nodes/{id}/pods/{name}/logs?tail=200` via `apiClient`; `_pollLiveLogs()`: uses `GET /api/nodes/{id}/pods/{name}/logs?tail=100` via `apiClient` |

## API contract (BV2.3)

| Endpoint | Method | Response |
|----------|--------|---------|
| `/api/nodes/{node_id}/pods/list` | GET | `Schema__Pod__List`: `{ pods: Schema__Pod__Info[] }` |
| `/api/nodes/{node_id}/pods/{name}/logs` | GET | `Schema__Pod__Logs__Response`: `{ container, lines, content, truncated }` |
| `/api/nodes/{node_id}/pods/{name}` | GET | `Schema__Pod__Info` |
| `/api/nodes/{node_id}/pods/{name}/stop` | POST | `Schema__Pod__Stop__Response` |
| `/api/nodes/{node_id}/pods/{name}` | DELETE | `Schema__Pod__Stop__Response` |

**`Schema__Pod__Info` fields:**
- `pod_name: str` — pod/container name
- `node_id: str` — parent node
- `image: str` — Docker image
- `state: Enum__Pod__State` — `running / pending / stopped / failed`
- `ports: str` — Docker string format e.g. `"0.0.0.0:8080->8080/tcp"`

---

## Schema migration

| Old sidecar field | New control-plane field |
|-------------------|------------------------|
| `name` | `pod_name` |
| `status` | `state` |
| `ports` (dict) | `ports` (string) |

`_portLinks` old: `Object.entries(ports)` where `ports = { "8080/tcp": [{HostPort: "8080"}] }`
`_portLinks` new: regex parse `":hostPort->containerPort"` from string

## Unchanged (still direct sidecar)

- `_fetchBootLog()` → `GET ${sidecar}/host/logs/boot` — no control-plane proxy
- Host stats in `_fetchContainers()` → `GET ${sidecar}/host/status` — no control-plane proxy
- Pod stats in `_renderContainers()` → `GET ${sidecar}/pods/{name}/stats` — no control-plane proxy

These will migrate when BV2.x adds control-plane proxy endpoints for boot logs and host/pod stats.
