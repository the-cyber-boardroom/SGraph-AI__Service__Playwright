## Decision: pivoted off Browser__Launcher

Browser__Launcher (v2 §4.8) explicitly defers to v1 §4.3, and the v1 source is not in the dev pack. More importantly, it requires real Chromium (`sync_playwright()`) to unit-test meaningfully — that belongs in integration tests, not the current unit-only slice.

`Session__Manager` is pure state management with no Playwright calls, so it lands cleanly in this slice. Browser__Launcher moves to Phase 2.3 where it'll be validated against the integration test harness.

## What was delivered

### `service/Session__Manager`

In-memory registry. One class owns:
- `sessions         : Dict__Sessions__By_Id`             — `Schema__Session__Info` records
- `browsers         : Dict__Session__Browsers__By_Id`    — opaque Playwright Browser objects
- `capture_configs  : Dict[Session_Id, Schema__Capture__Config]` — per-session capture defaults
- `start_times      : Dict[Session_Id, int]`             — wall-clock ms epoch at `create()`
- `artefacts        : Dict[Session_Id, List__Artefact__Refs]`    — accumulated per-session

### Methods (derived from callsites in routes-catalogue-v2)

| Method | Purpose |
|--------|---------|
| `create(browser, request, trace_id, capabilities)` | New session; caps `lifetime_ms` against `capabilities.max_session_lifetime_ms` |
| `get(session_id)` | Session info or `None` |
| `get_browser(session_id)` | Browser or `None` |
| `get_capture_config(session_id)` | Per-session config |
| `get_start_time(session_id)` | Wall-clock ms epoch at create |
| `get_artefacts(session_id)` | `List__Artefact__Refs` for the session |
| `list_active()` | Sessions with status ACTIVE or IDLE |
| `record_action(session_id, step_result)` | Bumps `total_actions`, `artefacts_captured`, `last_activity_at`; appends step artefacts |
| `add_artefact(session_id, ref)` | Records a single artefact ref out-of-band (e.g. session-level video) |
| `close(session_id)` | Marks status CLOSED; drops browser ref (`Browser__Launcher` owns actual `.close()`) |
| `healthcheck()` | `Schema__Health__Check` with active session count |
| `now_ms()` | Single wall-clock seam — tests subclass and freeze time |

### Design constraints honoured

- **No Playwright calls.** `browsers` values are typed `object`. Actual `page.*` work stays in `Step__Executor`; actual browser `.close()` stays in `Browser__Launcher`. `Session__Manager` only owns references.
- **Deterministic time for tests.** `now_ms()` is the single clock seam. `_FrozenSessionManager(Session__Manager)` in the test module sets `fixed_now` and overrides `now_ms()` — no monkeypatch, no `freezegun`.
- **Idempotent close.** `close()` on a closed or unknown session is a no-op. Same for `record_action` on an unknown session — callers like `Action__Runner` don't have to pre-check.

## Deviations from brief

- **v1 spec not available.** Method surface derived from callsites — `get`, `get_browser`, `get_capture_config`, `get_artefacts`, `get_start_time`, `list_active`, `create`, `close`, `record_action`, `healthcheck` are all referenced in `routes-catalogue-v2.md`. Added `add_artefact` for out-of-band refs (session-level video) that don't come through `record_action` — this is a judgement call, will be revisited once `Artefact__Writer` / video flow is implemented.
- **IDLE auto-transition not implemented.** Sessions transition ACTIVE → IDLE based on time without activity; the v2 spec doesn't say where that lives. Left as a follow-up for when a session janitor / expiry sweep is needed.

## Follow-ups

- `Browser__Launcher` (Phase 2.3) — real Chromium launch + stop + healthcheck. Needs integration test scaffolding, not unit.
- `Artefact__Writer` — sink routing (vault, s3, inline, local_file). Blocks `Credentials__Loader` and the full capture flow.
- Session expiry janitor — background sweeper that flips expired sessions to CLOSED. Not in any current callsite but implied by `expires_at` being populated.
