## What was delivered

### `sgraph_ai_service_playwright/service/Step__Executor.py` — 1 class, 17 methods

- `execute(page, step, step_index, capture_config) -> Schema__Step__Result__Base` — central dispatcher on `step.action`.
- **First-pass handlers (6 of 16 actions):**
  - `execute_navigate` — `page.goto(url, wait_until, timeout)`
  - `execute_click`    — `page.click(selector, button, click_count, delay, force, timeout)`
  - `execute_fill`     — `page.fill()` when `clear_first`; else `page.locator().press_sequentially()`
  - `execute_screenshot` — `page.screenshot(full_page, timeout)` or element-scoped via `page.locator().screenshot()`. Routes bytes through `artefact_writer.capture_screenshot()` — **Step__Executor never writes to a sink directly**.
  - `execute_get_content` — branches on `content_format` (HTML via `page.content()` / `inner_html`, TEXT via `inner_text`); honours `inline_in_response` (inline returns content, else routes through `capture_page_content`).
  - `execute_get_url` — reads `page.url`.
- **Deferred handlers (10 of 16 actions):** `execute_press / select / hover / scroll / wait_for / video_start / video_stop / evaluate / dispatch_event / set_viewport` — each raises `NotImplementedError` with the `Phase 2.11` marker so a stray dispatch produces a clear message, not an `AttributeError`.
- **Result constructors:** `passed_result`, `failed_result` — build the `Schema__Step__Result__Base` with step id, index, action, status, duration, artefacts, error message.
- **Helpers:** `resolve_id` (falls back to `Step_Id(str(step_index))` when the caller didn't set one), `filter_refs` (drops `None` refs from disabled sinks), `now_ms` (single wall-clock seam — tests subclass to freeze time).
- **Error contract:** every handler catches `Exception`, times the step, populates `error_message`, and returns a `FAILED` result. Uniform result shape regardless of outcome — the Action__Runner in Phase 2.10 wants one consistent thing.

### `sgraph_ai_service_playwright/service/Artefact__Writer.py` — 3 new capture_* methods

Per-type pinned wrappers that pass bytes → `write_artefact` with the artefact type pre-set. Keep the `page → bytes → writer.capture_*()` boundary clean: Step__Executor is the only class that captures bytes via `page.*`; Artefact__Writer is the only class that dispatches them to sinks.

- `capture_screenshot(data, sink_config)`   → `write_artefact(Enum__Artefact__Type.SCREENSHOT, …)`
- `capture_page_content(data, sink_config)` → `write_artefact(Enum__Artefact__Type.PAGE_CONTENT, …)`
- `capture_pdf(data, sink_config)`          → `write_artefact(Enum__Artefact__Type.PDF, …)` — defined now; PDF action itself is deferred.

Returns `None` when `sink_config.enabled=False` — same semantics as `write_artefact`.

### CI workflow hotfix — `.github/workflows/ci-pipeline.yml`

Two steps in `run-integration-tests` ("Start local container and run integration tests" + "Run integration tests against local container") were not passing AWS_* secrets. Without them, `Create_Image_ECR` computed its image URI with `account_id=None` and tried to `docker create` from `None.dkr.ecr.eu-west-1.amazonaws.com/sgraph_ai_service_playwright:latest` — a tag that never exists. Added AWS_ACCOUNT_ID / AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY / AWS_DEFAULT_REGION to both steps, matching the pattern already used in `build-docker-image` and `push-to-ecr`.

## Deviations from brief

- **First-pass subset, not the full 16 actions.** The Phase 2.7 debrief called for: "Phase 2.9: `Step__Executor` + `Artefact__Writer.capture_*()`. Playwright integration proper. Start with NAVIGATE / CLICK / FILL / SCREENSHOT / GET_CONTENT / GET_URL; defer VIDEO_START/STOP and TRACE (context-level APIs)." Followed to the letter. The remaining ten actions (PRESS, SELECT, HOVER, SCROLL, WAIT_FOR, VIDEO_START, VIDEO_STOP, EVALUATE, DISPATCH_EVENT, SET_VIEWPORT) each have a deferred stub raising `NotImplementedError` with a `Phase 2.11` marker — a dispatch won't fall through to a Python `AttributeError`.
- **`Safe_Str__Text` sanitisation in unit tests.** First pass used `'a@b.com'` as the fill value — `Safe_Str__Text` sanitises `@` to `_`. Swapped to `'hello world'` for a round-trip-safe test value. The sanitisation is correct spec behaviour (the Schema defines the constraint); tests adapt to match.
- **Fake `_Fake_Page.url` as plain attribute, not a property.** Playwright's `Page.url` is a property, but exposing it as an attribute is simpler and the test contract only needs the return-value shape. `_Fake_Page` is a test double, not a Playwright conformance fixture — it can diverge where it makes tests clearer.
- **No `execute_pdf` even though `capture_pdf` exists.** The PDF action schema is in the spec but the PDF *step* isn't in the Phase 2.9 subset; only the writer-side `capture_pdf` wrapper is. That's fine — `capture_pdf` will be live when Phase 2.11 wires the PDF action.
- **CI hotfix landed alongside Phase 2.9.** Separate concern (CI workflow env vars for integration job), but small enough that it doesn't warrant its own commit and is directly unblocking the branch's next CI run. Discussed inline in the commit message.
