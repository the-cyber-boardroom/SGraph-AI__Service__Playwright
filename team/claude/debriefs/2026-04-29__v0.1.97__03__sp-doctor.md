## What shipped

A new top-level `sp doctor` typer command with two subcommands. `sp pw ensure-passrole` is **dropped entirely** (hard cut per Q3 default).

```
sp doctor                — run all checks (preflight + passrole)
sp doctor passrole       — iam:PassRole check (replaces sp ensure-passrole / sp pw ensure-passrole)
sp doctor preflight      — AWS account / region / ECR access check
```

## How

| File | Role |
|---|---|
| `scripts/doctor.py` | New typer entry — `passrole` + `preflight` subcommands + `@app.callback(invoke_without_command=True)` to run all checks when no subcommand given. Reuses `ensure_caller_passrole` / `aws_account_id` / `aws_region` / `ecr_registry_host` from `Ec2__AWS__Client`. |
| `scripts/provision_ec2.py` (modified) | Mounts `sp doctor` via `add_typer`. Drops the `@pw_app.command(name='ensure-passrole')` decorator + body. Inline iam:PassRole warning updated: `'sp pw ensure-passrole'` → `'sp doctor passrole'`. |

## Per-Q4 default — preflight scope

`sp doctor` covers **before-create** checks only. Per-instance health checks (`sp pw health`, `sp os health`, etc.) stay in their sections. Section-specific preflight (e.g. `sp vnc doctor`) can land later.

## Hard cut on `sp ensure-passrole`

Per Q3 default + plan doc 7 C1 precedent. The command is gone — no transition window, no flat alias. Any operator running `sp ensure-passrole` or `sp pw ensure-passrole` will get a "command not found" message; the inline iam:PassRole warning at provision time now points at the right replacement.

# v0.1.97 — done

All three sub-slices shipped. `sp` now has the clean shape proposed in the brief:

```
Usage: sp [OPTIONS] COMMAND [ARGS]...

  SG Playwright CLI — manage ephemeral EC2 stacks across all sister sections.

Top-level:
  catalog                 Cross-section enumeration (mirrors GET /catalog/*)
  doctor                  Global preflight (AWS account / region / ECR / IAM)

Subgroups (each with its own create / list / info / delete / health / …):
  pw      (playwright)    Playwright + agent-mitmproxy stack
  el      (elastic)       Elasticsearch + Kibana
  os      (opensearch)    OpenSearch + Dashboards
  prom    (prometheus)    Prometheus + cAdvisor + node-exporter
  vnc                     Browser-viewer (chromium + nginx + mitmproxy)
  linux   (lx)            Bare AL2023 EC2
  docker  (dk)            AL2023 + Docker CE EC2
  ob      (observability) AMP + AMG + OpenSearch managed (hidden)
```

Symmetric. Predictable. Every stack section follows the same shape — Playwright is no longer special-cased.
