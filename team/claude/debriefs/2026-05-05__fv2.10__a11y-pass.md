## What was done

ARIA tablist semantics on `sp-cli-nodes-view` detail panel, keyboard access for non-`<button>` interactive elements, `aria-label` on all icon-only buttons, and `:focus-visible` rings across nodes-view, compute-view, and left-nav.

### Files changed

| File | Change |
|------|--------|
| `sp-cli-nodes-view.html` | `role="tablist"` + `aria-label` on `.sgl-panel-tabs`; each tab gets `role="tab"`, `id`, `aria-selected`, `aria-controls`; each panel gets `role="tabpanel"`, `id`, `aria-labelledby`, `tabindex="0"`; icon buttons get `aria-label`; `.node-count` + `.ct-status` + `.bl-status` + `.ec2i-status` get `aria-live`; `.ct-list` gets `role="list"`; `.pod-log-content` gets `aria-live`; `.btn-live-log` gets initial `aria-label` |
| `sp-cli-nodes-view.js` | `_activateTab()` updates `aria-selected` on tabs and `aria-hidden` on panels; `_renderList()` adds `role="button"`, `tabindex="0"`, `aria-label`, Enter/Space keydown handler to node rows; `_renderContainers()` adds `aria-label` to log buttons; `_renderApiKeyRow()` adds `aria-label` to eye and copy buttons; `_toggleLiveLog()` / `_stopLiveLog()` toggle `aria-label` on live button |
| `sp-cli-nodes-view.css` | `:focus-visible` rings on `.node-row`, `.api-key-btn`, `.ct-log-btn`, `.btn-live-log` |
| `sp-cli-compute-view.js` | Spec cards get `role="button"`, `tabindex`, `aria-label`, `aria-disabled`; soon-cards get `tabindex="-1"`; `_selectSpec()` sets `aria-pressed`; icon hidden with `aria-hidden="true"`; Enter/Space keydown handler |
| `sp-cli-compute-view.css` | `:focus-visible` ring on `.spec-card` |
| `sp-cli-left-nav.css` | `:focus-visible` ring on `.nav-item` |

## Scope notes

- `aria-current="page"` on left-nav is technically wrong (these are views, not pages) but it predates this slice and is consistent with the existing implementation. Correct value would be `aria-current="true"` or a custom `aria-selected`-driven tablist pattern. Deferred.
- Colour contrast of `--text-3` text against `--bg-panel` background is ~3.8:1 (below WCAG AA 4.5:1 for body text, above 3:1 for large text). Tokens are shared with sg-layout; changing them is out of scope for this slice.
