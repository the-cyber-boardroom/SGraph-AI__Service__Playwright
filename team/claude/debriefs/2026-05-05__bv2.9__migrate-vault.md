## Good failures (caught early, informed design)

**Route prefix off-by-one**: Initial test URLs used `/spec/...` but `tag='vault'` on `Routes__Vault__Spec` auto-prefixes routes with `/vault/`, making actual paths `/vault/spec/...`. Tests caught this immediately.

**UNKNOWN_SPEC not triggered without registry**: Initial `_validate_spec_id` only checked `self.spec_registry`. When only `write_handles_by_spec` was set, unknown spec_ids passed through. Test failure exposed the missing `elif` branch. Fixed: validate against `write_handles_by_spec.keys()` when no registry is provided.

**Type_Safe forward ref limitation**: After BV2.8 background-agent used `Optional['HelperClass']` string annotations in the 7 circular-import `AWS__Client` files, Type_Safe threw `TypeError` at runtime — it cannot resolve `ForwardRef('ClassName')` to actual class instances. Caught by test run; reverted to `object = None`.

**Fake collaborator type mismatch**: `_FakeAWSClient` set `self.sg = _FakeSG()` where `_FakeSG` didn't extend `EC2__SG__Helper`. Type_Safe rejected the assignment because `Ollama__AWS__Client.sg: Optional[EC2__SG__Helper]`. Fixed by: (1) making all EC2 helper fakes proper subclasses; (2) using `object.__setattr__` to bypass validation for sub-attribute injection.

## Key design decisions

- `Vault__Spec__Writer` accepts either `spec_registry` (registry injection) or `write_handles_by_spec` (plain dict). This keeps tests fully self-contained — no registry dependency.
- `SHARED_STACK_ID = '_shared'` (renamed from `GLOBAL_STACK_ID = '_global'`) is a sentinel for spec-wide blobs not tied to a single node.
- Vault persistence is stubbed throughout. `write()` produces a receipt but doesn't persist. `list_spec()` returns an empty collection. `delete()` returns `True`. Real I/O lands in a BV2.x follow-on.
