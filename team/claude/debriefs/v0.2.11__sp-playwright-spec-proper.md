## What was done

The prior rework (`3f1d0b3`) cloned the `vnc` spec — the *older* convention
(`sgraph_ai_service_playwright__cli/{spec}/`, typer in `scripts/`, no builder).
The user asked for the `vault_app` / `local_claude` convention instead. This
slice rebuilds the playwright spec properly and retires the cli-plugin.

### Created — `sg_compute_specs/playwright/`
Built alongside the existing `core/` (the running-app code), matching
`vault_app`:
- `schemas/` — Create Request/Response, Info, List, Delete Response (pure data)
- `service/Playwright__Service.py` — extends `Spec__Service__Base` (health/exec/
  connect_target inherited)
- `service/Playwright__AWS__Client.py` — wraps the **shared** `sg_compute` EC2
  helpers (`EC2__SG/AMI/Instance/Launch/Tags__*`); no per-spec clones
- `service/Playwright__Compose__Template.py` — **3 containers**: `host-plane`
  (always) + `sg-playwright` (always) + `agent-mitmproxy` (only `--with-mitmproxy`)
- `service/Playwright__User_Data__Builder.py` — `Section__Base` + Docker install
  + .env + (optional) interceptor + compose + ECR login + compose up
- `service/Playwright__Stack__Mapper.py` + `Playwright__AMI__Helper.py`
- `cli/Cli__Playwright.py` — `Spec__CLI__Builder` call: 8 standard verbs
  (list/info/create/wait/health/connect/exec/delete) + `ami` + `cert` sub-typers,
  plus `--with-mitmproxy` / `--intercept-script` / `--image-tag` / `--disk-size` /
  `--no-use-spot` / `--api-key` extras, plus `logs` and `extend` commands
- `cli/Renderers.py` — custom create/info renderers (surfaces the one-shot api_key)
- 4 test files — 30 tests, all green

## NOT done — surfaced as an explicit decision

**Wiring `Cli__Playwright` into `provision_ec2.py`'s `sg` typer.** The inline
`pw_app` there is ~1400 lines / 19 commands — including genuinely-extra commands
(`connect`, `shell`, `env`, `run`, `screenshot`, `smoke`, `forward`, `clean`,
`diagnose`, `create-from-ami`) plus `vault` + `ami` subgroups. Swapping it for
the builder app is a lossy deletion of working commands the user called "the
good stuff", so the wiring approach (full swap / surgical keep-the-extras /
parallel registration) is left as a conscious decision rather than guessed.

Until wired, `sg playwright create` still runs the old `provision_ec2.py` path.
The new spec is reachable programmatically (`Playwright__Service().setup()`).
