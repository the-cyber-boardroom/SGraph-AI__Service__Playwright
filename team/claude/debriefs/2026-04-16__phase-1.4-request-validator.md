# Debrief — Phase 1.4 — Request__Validator + JS__Expression__Allowlist (spec §9)

- **Date:** 2026-04-16
- **Commit(s):** _(pending — this commit)_
- **Source spec:** `/tmp/playwright-dev-pack/dev-specs/schema-catalogue-v2.md` §9

## What was delivered

Two service classes that together own **all** cross-schema validation for the Playwright service. Every rejection raises `HTTPException(422)` carrying a `Schema__Error__Response` payload, so routes and runners surface a stable `error_code` without re-encoding.

### `service/JS__Expression__Allowlist`
- One field: `allowed_expressions : List[Safe_Str__JS__Expression]`.
- Defaults to an empty list → **deny-all**. Operators must explicitly populate the allowlist at boot time (typically from a vault-stored config).
- `is_allowed(expression)` does exact-match comparison — no regex, no substring, no normalisation. Keeps policy auditable.

### `service/Request__Validator`
Single-class home for the validation error codes listed in spec §9:

| Method | Error codes raised |
|--------|--------------------|
| `validate_session_create(request, capabilities)` | `distributed_not_supported`, `persistent_not_supported`, `cdp_missing_endpoint` |
| `validate_step(step, cap_config, capabilities, target)` | `evaluate_expression_not_allowed` (+ delegates to `validate_sink_configs`) |
| `validate_sink_configs(cap_config, capabilities, target)` | `sink_incompatible_with_deployment`, `sink_missing_vault_ref`, `sink_missing_s3_bucket` |
| `validate_step_ids_unique(steps)` | `duplicate_step_ids` |
| `validate_sequence_timeout(total_timeout_ms, capabilities)` | `timeout_exceeds_deployment_limit` |
| `reject(error_code, error_message, capabilities=None)` | Builds the `Schema__Error__Response` envelope and raises `HTTPException(422)` |

Module-level `SINK_FIELDS` enumerates every `Schema__Capture__Config` attribute that carries a sink (screenshot, screenshot_on_fail, video, pdf, har, trace, console_log, network_log, page_content). `validate_sink_configs` iterates this list rather than hardcoding per-attribute branches.

## Issues / bugs hit

- **`Safe_Str__Url` rejects `wss://`** — initial CDP endpoint test used `wss://cdp.example.com/v1`; the primitive only allows `http`/`https`. Fixed by using `http://localhost:9222/devtools` (Playwright's standard local CDP URL).
- **`ids.count()` quadratic** in `validate_step_ids_unique` — acceptable for the small number of steps per sequence, but flagged here so it isn't copy-pasted to a hot path later. A simple `Counter` would be O(n).

## Deviations from brief

- **Allowlist semantics** — spec §9 only says `evaluate_expression_not_allowed` exists; doesn't specify match semantics. Chose **exact-match** because (a) auditable, (b) prevents `document.title;alert(1)` bypass via substring, (c) matches the "deny-by-default" phrasing. If the v1 spec (not yet in the dev pack) wants regex or glob semantics, the API surface stays the same — only `is_allowed` changes.
- `video_not_stopped` is listed in the reality doc but belongs to the runner layer (requires session state) — not implemented in the validator. Flagged as Phase 2 work.
