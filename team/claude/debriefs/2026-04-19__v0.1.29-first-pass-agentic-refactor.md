# Debrief — v0.1.29 — First-pass agentic refactor — 2026-04-19

## Slice summary

Land the first pass of the layered-dynamic-code-fastapi-runtime direction of travel inside the single `sgraph_ai_service_playwright/` repo. Concretely:

1. **Rename** every boot-loader env var from `SG_PLAYWRIGHT__*` to `AGENTIC_*` in a one-shot cutover (no dual-read shim).
2. **Extract** the Lambda boot-loader into L1/L2 subpackages — `agentic_fastapi/` (generic, no AWS) and `agentic_fastapi_aws/` (S3 + image-version + shim).
3. **One-command deploy** — `scripts/deploy_code.py --stage dev` packages, uploads, flips the Lambda env, and smoke-tests the Function URL. Target round-trip ~30 s cloud, ~5 s local via `AGENTIC_CODE_LOCAL_PATH`.
4. **Admin surface** — 8 read-only `/admin/*` endpoints served by `Agentic_Admin_API` (health, info, env, boot-log, error, manifest, capabilities, skills/{name}).
5. **SKILL files + `capabilities.json` stub** — `skills/skill__{human,browser,agent}.md` (placeholder content, TODO-tagged) and a repo-root `capabilities.json` served at `/admin/capabilities`.
6. **Single-track CI** — old docker-image-based jobs (`build-docker-image`, `push-to-ecr`, `run-integration-tests`, `deploy-lambda`, `smoke-test`, `increment-tag`) gated `if: false`; new `deploy-code` job gated on `check-aws-credentials has-credentials == 'true'` and `needs: run-unit-tests`.

Public HTTP surface (10 endpoints) and the 10 live service classes are **unchanged**. Breaking only for the Lambda-image contract (env vars, boot path, deploy pipeline). Supersedes the earlier 3-part plan `v0.1.29__part-{1,2,3}__*.md`.

## Rationale

Two source briefs drove the reshape:

- `arch__layered-dynamic-code-fastapi-runtime-v4.md` — direction-of-travel architecture (L1 generic FastAPI base, L2 AWS boot, L3 app; lockdown layers; admin modes; OpenAPI + SKILLs discovery).
- `brief__playwright-dev-team__first-pass-agentic-refactor.md` — concrete first-pass scope.

The old 3-part plan split the work by "repo-split / boot-shim / migration" which mapped poorly onto a single-repo staged approach. The revised plan sequences by day (rename → bucket + deploy_code → admin + skills + manifest → tests + CI → debrief) and explicitly defers repo-split, PyPI publishing, Docker Hub publishing, sidecar enforcement, `AGENTIC_ADMIN_MODE=full`, lockdown layers, and two-track CI.

### CI hotfix — `AWS_Config` instead of raw boto3
`deploy-code` failed on first invocation with `AttributeError: 'S3' object has no attribute 'region_name'`. Root cause: `scripts/deploy_code.py` called `s3.region_name()` on the osbot-aws `S3` client, and `scripts/package_code.py` was using raw `boto3.session.Session().region_name` + `boto3.client('sts').get_caller_identity()['Account']` in violation of CLAUDE.md rule 14 ("never boto3 directly"). Swapped both for `AWS_Config().region_name()` + `AWS_Config().account_id()`. 390 unit tests green. Commit `e094adc`.

## Good failures

- **CI caught the `s3.region_name()` typo on the first live run** of the new `deploy-code` job. Unit tests don't touch real AWS, so they never hit the missing method — the deploy-via-pytest-style CI job was the correct tripwire. One-line fix + an honest move to `AWS_Config` across both scripts.
- **Safe_Str choice for admin schemas surfaced a real typing question.** `Safe_Str__Text` strips `/` and `:`, so `code_source = "s3:bucket/key"` became `"s3_bucket_key"` on the first attempt. Caught by the admin-api unit test asserting `code_source` startswith `passthrough:`. Switched to `Safe_Str__Text__Dangerous` (preserves punctuation needed for S3 paths) and `Safe_Str__Url__Path` (for clean URL paths without colons). The test existed because every admin schema shape is asserted explicitly — the "assert on contracts" rule paid for itself.
- **Middleware / route-404 interaction documented up front.** Requests to unknown skill audiences like `/admin/skills/mystery` 401 at the middleware layer (the path isn't on the exact-match exclude list) before the route's 404 can fire. The test asserts `status in (401, 404)` with a comment explaining why. Future me doesn't have to rediscover this.
