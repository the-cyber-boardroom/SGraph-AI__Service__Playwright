## What was delivered

### `service/Playwright__Service`

Thin orchestrator. Type_Safe-composed sub-services (Capability__Detector, Session__Manager, Browser__Launcher, Request__Validator). Phase 2.7 scope is the health surface only — session/action/sequence methods arrive in Phase 2.10 when Routes__Session / Routes__Browser / Routes__Sequence need them.

| Method | Returns | Route |
|--------|---------|-------|
| `setup()` | self | (idempotent; primes detector) |
| `get_service_info()` | `Schema__Service__Info` | GET /health/info |
| `get_capabilities()` | `Schema__Service__Capabilities` | GET /health/capabilities |
| `get_health()` | `Schema__Health` (aggregates 3 sub-checks) | GET /health/status |

`get_health()` composes `Browser__Launcher.healthcheck()` + `Session__Manager.healthcheck()` + `Capability__Detector.connectivity_check()` into one `Schema__Health` with `healthy = all(c.healthy for c in checks)`. The connectivity check requires `SG_SEND_BASE_URL` — absent on a laptop, so `/health/status` reports `healthy=false` there, which is the correct answer (we can't reach vault).

### `fast_api/routes/Routes__Health`

`Fast_API__Routes` subclass with `tag='health'`. Three `add_route_get` registrations — pure delegation to `Playwright__Service`. Paths exported as module-level constants (`TAG__ROUTES_HEALTH`, `ROUTES_PATHS__HEALTH`) so tests and spec docs share one source of truth.

### `fast_api/Fast_API__Playwright__Service`

`Serverless__Fast_API` subclass. Holds one `Playwright__Service`. `setup()` calls `config__no_api_key()` (health probes must answer plain curl — Lambda warmers / ALB probes / CI smoke don't sign requests), primes the service, then hands off to the base class which wires middlewares + default routes. `setup_routes()` adds `Routes__Health` with `service=self.service` injected.

API-key enforcement for the session/sequence routes re-enables in Phase 2.10 — the disable is scoped to Phase 2.7 where only the public health surface exists.

## Deviations from brief

- **Route tag `'health'` rather than separate `/info`, `/health`, `/capabilities` top-levels.** The directive said "3 GET endpoints: /health/info, /health/status, /health/capabilities" — all three live under `/health/*`, which is what got built. Matches the directive.
- **API-key middleware disabled unconditionally in setup().** The base `Serverless__Fast_API` enables it by default (requires env vars `FAST_API__AUTH__API_KEY__NAME` + value to pass). For Phase 2.7 health-only we need unauthenticated access. When Phase 2.10 adds mutating routes, re-enabling needs to be scoped to non-health paths (typically via path-based middleware exclusion, or a subclass that re-enables after `setup()` and adds a per-route skip list). Explicit TODO threaded through the inline comment.
- **`get_health().healthy` is `False` on a dev laptop** because the `connectivity_check` returns `healthy=False` when `SG_SEND_BASE_URL` is unset. This is intentional — the health endpoint should report the true state of vault connectivity. Tests cover both paths (vault set vs unset).
- **No session/action/sequence methods on `Playwright__Service` yet.** The directive said stubs that raise `HTTPException(501, "Not yet implemented")` would be fine. I omitted them entirely — no caller exists yet, and unused stubs rot. They land in Phase 2.10 alongside the routes that consume them. Zero production code is blocked by their absence.

## Follow-ups

- **Phase 2.8**: Docker infra classes (`Docker__SG_Playwright__Base`, `Build__Docker__SG_Playwright`, `ECR__`, `Lambda__`, `Local__`) + deploy-via-pytest tests. End-to-end proof: local code → Docker image → ECR → Lambda → `/health/info` returns `deployment_target: "lambda"`.
- **Phase 2.9**: `Step__Executor` + `Artefact__Writer.capture_*()`. Playwright integration proper. Start with NAVIGATE / CLICK / FILL / SCREENSHOT / GET_CONTENT / GET_URL; defer VIDEO_START/STOP and TRACE (context-level APIs).
- **Phase 2.10**: `Action__Runner` + `Sequence__Runner` + `Routes__Session` / `Routes__Browser` / `Routes__Sequence`. Fills in `Playwright__Service` session/action/sequence surface. Full 25-endpoint API live. Re-enables API-key middleware for mutating routes.
- **Middleware scope**: The unconditional `config__no_api_key()` in setup() is Phase-2.7-appropriate but will need revision. Options: (a) per-path exclusion middleware, (b) `setup()` accepts a flag, (c) separate `Fast_API__Playwright__Service__Public` subclass for health-only test contexts. Worth a 10-minute design discussion before Phase 2.10.
