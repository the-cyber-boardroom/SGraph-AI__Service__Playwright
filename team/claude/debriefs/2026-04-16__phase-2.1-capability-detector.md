## Issues / deviations

- **`playwright.__version__` doesn't exist.** Spec code `import playwright; return playwright.__version__` throws `AttributeError` in Playwright 1.58.0. Switched to `importlib.metadata.version('playwright')` + `PackageNotFoundError` fallback.
- **`Safe_Str__Version` rejects `'unknown'`.** Spec uses `'unknown'` as the chromium-detection fallback, which fails the `^v?\d{1,3}(?:\.\d{1,3}){0,2}$` regex. Replaced with `'0.0.0'` (module-level `FALLBACK_VERSION` constant) so `Schema__Service__Info` can always be built.
- **`image_version` fallback.** Spec falls back to `'dev'` if `CODE_S3_KEY` is unset, which also fails the version regex. Changed to `'v0'` (valid under the regex) to keep `/info` working in local dev.
- `Safe_Str__Version` regex accepts `v0.1.0` (the repo version file) â€” no change needed there.

## Lessons

- Version strings that must pass through a `Safe_Str__Version` type need a valid-looking fallback, not a human-readable `'unknown'`. Worth flagging to the spec author once Phase 2 is further along.
- `_EnvScrub` pattern (context manager that snapshots + restores a fixed allowlist of env keys) is cleaner than `monkeypatch` / `patch.dict` for test files that touch many env vars. Keeping it inline rather than in `conftest.py` until a second test file needs the same scrubbing.
