## Failures Encountered

### Shim task deferred ‚Äî implementations have diverged

**Finding:** The brief assumed that `sgraph_ai_service_playwright__cli/<spec>/` and `sg_compute_specs/<spec>/` are import-path-only differences (same code). A `diff -rq` reveals they are NOT equivalent:

- `Schema__Docker__Create__Request` ‚Äî legacy has `api_key_name`, `api_key_value`, `open_to_all`, `use_spot`; sg_compute_specs version has `registry` instead.
- `Docker__Service.py` ‚Äî legacy still uses lazy local imports + `ecr_registry_host()`; sg_compute_specs version uses typed Optional fields with static imports and a different `render()` call signature.
- Many schemas differ across all 8 specs.

Creating thin shims (`from sg_compute_specs.docker.service.Docker__Service import Docker__Service`) would silently replace the legacy class with a different implementation, breaking `scripts/docker_stack.py` and other callers that pass the old API surface.

**Classification:** Good failure ‚Äî caught before breaking anything. The brief's precondition ("implementations are equivalent; only import paths differ") was incorrect.

**Decision:** Defer the shim task. The legacy `sgraph_ai_service_playwright__cli/<spec>/` directories remain as full implementations for now. Their removal is a v0.3.x concern once the scripts and tests have been migrated to `sg_compute_specs.<spec>` APIs.
