## What was done

Migrated the dashboard's data source from the legacy `/catalog/*` endpoints to the new
`/api/specs` + `/api/nodes` API surface. Renamed all field consumers from the legacy
`stack_name` / `type_id` / `container_count` vocabulary to `node_id` / `spec_id` / `pod_count`.

Also fixed a pre-existing bug in `shared/node-state.js` where `NODE_STATE` constants were
uppercase (`'READY'`) instead of lowercase (`'ready'`) to match `Enum__Node__State`.

- **settings-bus schema bumped v2→v3**; `_migrate()` handles v1 and v2 stored values gracefully.
