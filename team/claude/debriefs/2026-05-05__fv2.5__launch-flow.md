## What was done

Switched the `sp-cli-compute-view` launch flow from per-spec legacy endpoints (`create_endpoint_path`) to the unified `POST /api/nodes` control-plane endpoint. Removed the "Open access" checkbox (no backend field in `Schema__Node__Create__Request__Base`).

### Files changed

| File | Change |
|------|--------|
| `sp-cli-compute-view.js` | `_launch()`: body now sends `spec_id`, `node_name` (replaces `stack_name`), `region`, `instance_type`, `max_hours`; posts to `/api/nodes`; removed `public_ingress`; removed `_openCheck` reference and its event listener; `_updateCostBar()` drops open-access branch, always shows `Caller IP` |
| `sp-cli-compute-view.html` | Replaced Open access checkbox + label with a static informational note |
| `sp-cli-compute-view.css` | Removed `.open-row`, `.open-row input` dead selectors |

## API contract (BV2.5)

**Request** `POST /api/nodes` → `Schema__Node__Create__Request__Base`

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `spec_id` | str | — | required |
| `node_name` | str | `''` | auto-generated by backend when empty |
| `region` | str | `''` | backend defaults to `eu-west-2` |
| `instance_type` | str | `t3.large` | |
| `ami_id` | str | `''` | auto-latest AL2023 |
| `caller_ip` | str | `''` | auto-detected from HTTP request |
| `max_hours` | int | `1` | |
| `creation_mode` | enum | `FRESH` | not sent — backend default used |

**Response**: `Schema__Node__Info` (same shape as `GET /api/nodes`) — node starts in `booting` state.

## Scope notes

- Only `docker` spec is live at the backend; other spec IDs raise `NotImplementedError` in `EC2__Platform.create_node`. The frontend shows a launch button for all specs — the error will surface as a catch block message.
- `caller_ip` is intentionally not sent — backend auto-detects from the HTTP request origin. This is more reliable than the old ipify.org approach.
- "Open access" may return as a future field if backend adds `public_ingress: bool` to the schema.
