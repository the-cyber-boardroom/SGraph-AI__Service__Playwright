## Issues / bugs hit

- Had to introduce `Safe_Str__Host` locally because `osbot-utils` has no equivalent.
- Getting the host regex right required `strict_validation = True` alongside MATCH mode. Without `strict_validation`, Safe_Str silently stores the mismatched value instead of raising.

## Lessons

- When the spec references a primitive that osbot-utils doesn't provide, define it locally under `schemas/primitives/` rather than loosening a schema field to `Safe_Str`.
- MATCH mode without `strict_validation` is a silent footgun.
