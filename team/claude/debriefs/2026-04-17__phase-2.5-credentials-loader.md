## What was delivered

### `service/Credentials__Loader`

Glue between vault state (via `Artefact__Writer`) and the Playwright browser context. Two public methods:

| Method | Purpose |
|--------|---------|
| `apply(session_id, session_manager, credentials)` | At session open: load cookies / storage state from the vault and set extra HTTP headers on the session's `BrowserContext`. Silent no-op when session/browser is missing or vault payload is empty. |
| `save_state_to_vault(vault_ref, state: dict)` | Delegates to `artefact_writer.write_to_vault`. Used when persisting updated storage state back after session activity. |

### Boundary notes

- **No direct vault client.** All vault access routes through `Artefact__Writer.read_from_vault` / `write_to_vault` seams. Swapping the real vault client later is a DI change on the writer — zero churn here.
- **No `page.*` calls.** Only `context.add_cookies` and `context.set_extra_http_headers`, which are `BrowserContext` methods, not `Page` methods. Consistent with the `Step__Executor`-owns-`page.*` rule (spec §10).
- **Order of application:** `cookies_vault_ref` → `storage_state_vault_ref.cookies` → `extra_http_headers`. Follows the spec source exactly.
- **Header-name normalisation gotcha:** `Safe_Str__Http__Header__Name` lowercases all header names. The loader's dict comprehension (`{str(k): str(v) for k, v in ...}`) preserves whatever case the schema stored — so `Authorization` becomes `authorization` on the wire. Playwright accepts both; documented in the test.

## Tests

`tests/unit/service/test_Credentials__Loader.py` — 9 cases across 6 classes. Uses a minimal `_FakeBrowser` / `_FakeContext` recorder and the same `_InMemoryWriter(Artefact__Writer)` pattern from Phase 2.4. No mocks, no patches.

- `test_apply__cookies` (2) — vault → `add_cookies`; empty payload skipped.
- `test_apply__storage_state` (2) — extracts `state['cookies']`; missing `cookies` key skipped.
- `test_apply__headers` (2) — sets headers; empty dict no-ops.
- `test_apply__missing_browser` (1) — unknown session is a silent no-op.
- `test_apply__combined` (1) — cookies + state + headers all applied in order.
- `test_save_state_to_vault` (1) — delegates to writer.
