## Design decisions

- **Dynamic discovery at import time** — `_mount_spec_sub_apps()` runs when `Cli__Compute__Spec` is imported. No explicit registration needed; adding `cli/Cli__<Pascal>.py` to a spec is sufficient.
- **Convention over config** — spec_id `docker` → module `sg_compute_specs.docker.cli.Cli__Docker` → attribute `app`. The `_spec_id_to_pascal` helper handles underscores (`open_design` → `OpenDesign`).
- **Graceful degradation** — specs without a CLI module are silently skipped; no crash.
- **Docker is the pilot** — only `docker` has a `Cli__Docker.py` today. Other specs gain theirs in a future slice.
- **`no_args_is_help=True` exits with code 2** — Typer's standard for help-on-no-args; tests use `--help` to get exit 0.
