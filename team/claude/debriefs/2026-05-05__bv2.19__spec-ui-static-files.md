## What was done

Added infrastructure for per-spec UI asset serving. `Fast_API__Compute` now mounts a `StaticFiles` handler at `/api/specs/{spec_id}/ui` for every spec whose `sg_compute_specs/{spec_id}/ui/` directory exists. No spec has a `ui/` folder yet — the mount is silent when absent — so existing behaviour is unchanged. Unblocks FV2.6 (per-spec UI co-location).

### New file

`sg_compute/core/spec/Spec__UI__Resolver.py` — Type_Safe class with:
- `ui_root_override: str = ''` — when set, resolves against that root (for tests)
- `ui_path_for_spec(spec_id) -> Optional[Path]` — importlib discovery for production; override path for tests

### Fast_API__Compute changes

- Added `ui_root_override: str = ''` field
- Added `_mount_spec_ui_static_files()` — iterates `registry.spec_ids()`, calls resolver, mounts `StaticFiles` for each spec with a `ui/` dir
- Called in `setup()` before `_mount_spec_routes()` (per brief: static prefix gets Starlette priority)

### pyproject.toml

Added `"sg_compute_specs/*/ui/**/*"` to the top-level `include` list for Lambda packaging.

## Key design decisions

- `ui_root_override` on `Fast_API__Compute` flows through to `Spec__UI__Resolver`. No mocking needed — pure Type_Safe injection.
- `StaticFiles` mount is skipped silently when no `ui/` folder exists. BV2.19 can ship before any spec migrates its UI.
- No `Cache-Control` header set — that is CloudFront's responsibility. Test confirms absence of the header.
- Mount happens before per-spec API routes (`_mount_spec_routes()`), giving Starlette's path-matching priority to the static prefix.
