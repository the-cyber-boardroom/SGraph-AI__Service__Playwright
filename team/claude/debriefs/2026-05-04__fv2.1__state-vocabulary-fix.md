## Scope notes

The `sp-cli-status-chip.js` `STATE_MAP` received two new entries beyond the brief's explicit scope (`ready` and `terminating`) to be consistent with the canonical `Enum__Node__State` values. No behaviour change for legacy callers — `running` still maps to `state-ready` label.

`podPillClass()` was added to `node-state.js` as an unplanned addition to satisfy the acceptance-criterion grep sweep. Pod status is Docker-vocabulary (`running`/`exited`), not node state — the function name makes that partition explicit.
