## What was done

Reworked the `sp playwright` CLI plugin from a "POD backend" model (commit
7b32a17) to the correct EC2+compose model, matching every other spec section
and the proven `provision_ec2.py` workflow.

### Files deleted
- `sgraph_ai_service_playwright__cli/playwright/service/Host_Plane__Client.py`
- `tests/unit/.../playwright/test_Playwright__Stack__Service.py`

### Files rewritten
- `enums/Enum__Playwright__Stack__State` — PENDING/RUNNING/READY/TERMINATING/TERMINATED/UNKNOWN
- All 6 schemas (Create Request/Response, Info, Delete, List, Health) — EC2 fields
- `service/Playwright__Stack__Service` — EC2-by-tag CRUD on the vnc pattern
- `fast_api/routes/Routes__Playwright__Stack` — `region` query param (not `host_url`)
- `cli/Renderers` — updated field names for EC2 model

### Files created

- `primitives/Safe_Str__Vault_Path` — for `intercept_script` field

- New test files: test_Playwright__Compose__Template, test_Playwright__User_Data__Builder,
