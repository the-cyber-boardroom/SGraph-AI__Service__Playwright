## Issues / bugs hit

- **`Enum__Wait__State.NETWORKIDLE` vs `NETWORK_IDLE`** — initial test used the single-word form; the enum is defined with an underscore. Fixed by reading the enum members directly.
- **`Schema__Step__Result__Get_Content.content_format` has no default** per spec — my initial test asserted it would default to a known member; corrected to assert `None` and added a second case covering explicit construction.
- **`fastapi` not installed in sandbox** — the dispatcher module imports `HTTPException`; I installed `fastapi` locally to run tests. CI already pulls it via `pyproject.toml`.

## Lessons

- When a spec omits defaults (like `content_format` on `Schema__Step__Result__Get_Content`), don't invent one in the schema to make tests simpler. The dispatcher is responsible for filling it at build time; the schema stays faithful to the catalogue.
- Adding a guard like `assert set(STEP_SCHEMAS.keys()) == set(Enum__Step__Action)` in the registry tests is a cheap way to catch drift — if someone adds a new `Enum__Step__Action` variant but forgets to register a schema, the suite fails at collection time rather than in production.
