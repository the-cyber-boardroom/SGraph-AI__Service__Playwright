## What was done

Folded the legacy SP CLI route surface into `Fast_API__Compute` at `/legacy/*`. The single process now serves both `/api/*` (modern) and `/legacy/*` (deprecated). Every `/legacy/*` response carries `X-Deprecated: true` and `X-Migration-Path: /api/specs` headers.

### Route file moves (git mv + shims)

- `sgraph_ai_service_playwright__cli/fast_api/routes/Routes__Ec2__Playwright.py` → `sg_compute/control_plane/legacy_routes/Routes__Ec2__Playwright.py`
- `sgraph_ai_service_playwright__cli/fast_api/routes/Routes__Observability.py` → `sg_compute/control_plane/legacy_routes/Routes__Observability.py`

Old locations replaced with single-line re-export shims. `Fast_API__SP__CLI` imports via shims transparently.

### Fast_API__Compute changes

Added to `setup()`:
- `_mount_legacy_routes()` — wires `Routes__Stack__Catalog` (at `/legacy/catalog/`), `Routes__Ec2__Playwright` (at `/legacy/ec2/playwright/`), `Routes__Observability` (at `/legacy/observability/`), plus all plugin routes (at `/legacy/{tag}/`) discovered via `Plugin__Registry`
- `_add_deprecated_header_middleware()` — FastAPI `@app.middleware("http")` that injects `X-Deprecated: true` and `X-Migration-Path: /api/specs` on every `/legacy*` response

### scripts/run_sp_cli.py

Updated `build_app()` to return `Fast_API__Compute().setup().app()` instead of `Fast_API__SP__CLI().setup().app()`. Comments updated.

## Good failures (caught early, informed design)

**`osbot_fast_api_serverless` not installed in test env**: Initial plan was to mount the whole `Fast_API__SP__CLI` as a sub-app. Failed immediately — `Fast_API__SP__CLI` imports `Serverless__Fast_API` from `osbot_fast_api_serverless` which isn't installed. Fixed by wiring the legacy routes directly into `Fast_API__Compute` using the same `add_routes()` mechanism, bypassing `Fast_API__SP__CLI` entirely for the mount.

**`prefix` replaces the tag (tag doesn't auto-prepend)**: When calling `add_routes(Routes__Stack__Catalog, prefix='/legacy', service=svc)`, the `catalog` tag is NOT applied — routes land at `/legacy/types`, `/legacy/stacks` etc. instead of `/legacy/catalog/types`. Fixed by specifying the full prefix including tag: `prefix='/legacy/catalog'`, `prefix='/legacy/ec2/playwright'`, etc.

## Key design decisions

- `Fast_API__SP__CLI` is NOT moved or refactored — only the two individual route files are moved. The whole legacy app still works standalone (for tests that explicitly instantiate it).
- Legacy plugin routes use `routes_cls().tag` to dynamically compute the prefix — avoids hardcoding 8 plugin names.
- `X-Deprecated` header is added via `@app.middleware("http")` checked against `request.url.path.startswith('/legacy')`. Works for successful responses; 500s from unhandled exceptions may not carry the header in test client mode (acceptable).
- `scripts/run_sp_cli.py` now boots `Fast_API__Compute` — operator scripts now get both `/api/` and `/legacy/` surfaces on port 8080.
