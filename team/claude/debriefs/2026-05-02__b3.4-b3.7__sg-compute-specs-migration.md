## Key Design Decisions

**Prometheus (B3.4):**
- `TAG_SECTION_VALUE = 'prom'` (short, matches CLI)
- Port 9090 HTTP (no nginx, no TLS)
- `nav_group = OBSERVABILITY`
- User data builder PLACEHOLDERS locks all 8 template keys

**OpenSearch (B3.5):**
- `TAG_SECTION_VALUE = 'os'` (short); `TAG_PURPOSE_VALUE = 'opensearch'`
- Port 443 (nginx TLS reverse proxy)
- `dashboards_url = https://<ip>/`, `os_endpoint = https://<ip>:9200/`
- `DEFAULT_INSTANCE_TYPE = 't3.large'`
- Route uses `body: Schema__OS__Stack__Create__Request` directly (no nested complex types)
- `nav_group = DATA`

**Elastic (B3.6):**
- Monolithic `Elastic__AWS__Client` (no split helpers) — preserves CLI fidelity
- `ELASTIC_NAMING = Stack__Naming(section_prefix='elastic')`
- `kibana_url = https://<ip>/`, IAM instance profile `sg-elastic-ec2`
- `DEFAULT_INSTANCE_TYPE = 'm6i.xlarge'`
- 3-container stack: elasticsearch + kibana + nginx
- Health simplified to 4 checks (EC2 state, public IP, elastic probe, kibana probe) — SSM checks deferred
- `nav_group = DATA`

**Firefox (B3.7):**
- `TAG_SECTION_VALUE = 'firefox'`, `TAG_PURPOSE_VALUE = 'firefox'`
- Port 443 (noVNC HTTPS) → 5800 (container), mitmweb port 8081 (internal)
- `viewer_url = https://<ip>/`, `mitmweb_url = http://<ip>:8081/`
- `DEFAULT_INSTANCE_TYPE = 't3.medium'`
- `Enum__Firefox__Stack__State` uses `(str, Enum)` in spec for JSON serialization (CLI uses bare `Enum`)
- Route uses `body: dict + from_json()` — necessary because `Schema__Firefox__Interceptor__Choice` is a nested Type_Safe
- `Firefox__Interceptor__Resolver` locks 8 named examples (`EXAMPLES` dict)
- `nav_group = BROWSERS` (COMPUTE does not exist in `Enum__Spec__Nav_Group`)
- capabilities: `[MITM_PROXY, IFRAME_EMBED, AMI_BAKE]`
- 63 tests across 5 test files (includes `test_Firefox__Interceptor__Resolver`)

## Good Failures (Surfaced Early)

- **`Enum__Spec__Nav_Group.COMPUTE` does not exist** — Firefox CLI uses `Enum__Plugin__Nav_Group.COMPUTE` (different enum from a different package). The spec must use `Enum__Spec__Nav_Group` which only has BROWSERS/DATA/OBSERVABILITY/STORAGE/AI/DEV/OTHER. Fixed: `BROWSERS` for Firefox.
- **`Safe_Str__Elastic__Stack__Name('s')` fails** — Regex `^[a-z][a-z0-9\-]{1,62}$` requires ≥2 chars. Tests used `'s'` (1 char). Fixed: all test stack names padded to `'st'` minimum.
- **`USER_DATA_TEMPLATE` braces in shell literals** — Shell `${...}` expand inside Python `str.format()`. Fixed: escape as `${{...}}` in the template strings.

## Bad Failures

None. All specs imported cleanly and all tests passed on first run.

## Patterns Established

1. **Spec route pattern for interceptor specs:** `body: dict` + `Schema.from_json(body)` avoids pydantic schema-generation failures for nested Type_Safe objects.
2. **`(str, Enum)` always in specs** even when CLI uses bare `Enum` — needed for JSON serialization in routes.
3. **PLACEHOLDERS tuple** in user data builder locks the template keys; tested with `len()` check + per-key `in USER_DATA_TEMPLATE` check.
4. **No `TAG_INTERCEPTOR_KEY` in Firefox tags** — unlike VNC, Firefox doesn't tag the interceptor kind on the instance.
