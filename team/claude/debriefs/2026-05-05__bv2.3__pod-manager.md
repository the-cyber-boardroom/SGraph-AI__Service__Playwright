## Good failures (surfaced early, improved design)

**`Dict[str, str]` rejected by Type_Safe.**
`Schema__Pod__Start__Request` initially declared `ports: Dict[str, str]` and `env: Dict[str, str]`. Type_Safe raises `ValueError` for `typing.Dict` annotations — only immutable types and their subclasses are supported. Fix: created `Dict__Pod__Ports` and `Dict__Pod__Env` as `Type_Safe__Dict` subclasses.

**`Type_Safe__Dict` can't be converted to Pydantic schema by `osbot_fast_api`.**
The route framework uses `Type_Safe__To__BaseModel` to convert request body schemas to Pydantic for FastAPI. `Dict__Pod__Ports` (a `dict` subclass) caused `PydanticSchemaGenerationError: Unable to generate pydantic-core schema`. The `Type_Safe__Dict` collections work fine as service-layer types but cannot be used as FastAPI route body parameters. Fix: `Schema__Pod__Start__Request` simplified to `name/image/type_id` (str fields only). The `ports` and `env` maps are an integration concern — the sidecar contract handles them; typed request fields can be added once a Pydantic-compatible wrapper exists.

## Key decisions

- **`_sidecar_client` is virtual** — `Fake__Pod__Manager` overrides `_sidecar_client()` to return `Fake__Sidecar__Client`. No mock.patch needed.
- **`auth via env var for now** — `SIDECAR_API_KEY = 'SG_COMPUTE__SIDECAR__API_KEY'`; sourced from environment in `_sidecar_client`. BV2.9 migrates this to vault.
- **`Schema__Pod__Start__Request` is minimal** — `ports`/`env` intentionally omitted from typed schema until `Type_Safe__Dict` ↔ Pydantic integration is solved. Collections classes exist and are tested independently.
- **Two routers at `/api/nodes` prefix** — `Routes__Compute__Nodes` and `Routes__Compute__Pods` both use `/api/nodes`; FastAPI merges them cleanly with no path conflicts.
