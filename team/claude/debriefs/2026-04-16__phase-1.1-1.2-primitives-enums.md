## Deviations from brief

- Added `Safe_Str__Host` later (phase 1.3) because `osbot-utils` doesn't ship one and the service/config schemas needed it.
- Initial attempt set `allow_empty = False` on primitives. Flipped to `True` (see "issues" below).

## Issues / bugs hit

- **Type_Safe default-construction failure.** Using a Safe_Str primitive as a field on a `Type_Safe` class triggers default-construction with `None`. Primitives configured with `allow_empty = False` rejected this and raised at class definition time. **Fix:** flipped `allow_empty = True` on every Safe_Str primitive used as a field. Renamed the `test__rejects_empty` cases to `test__allows_empty_for_type_safe_default_construction`.
- **`Timestamp_Now` import path.** Lives at `osbot_utils.type_safe.primitives.domains.identifiers.safe_int.Timestamp_Now`, not directly under `identifiers`. Cost ~1 retry.
- **REPLACE vs MATCH regex mode.** MATCH-mode Safe_Str also needs `strict_validation = True` to raise on mismatch. REPLACE substitutes disallowed characters with `_`, not empty string.

## Lessons

- `Type_Safe` treats every declared field type as default-constructible. Every custom primitive destined for a schema field must be default-constructible.
- Default-construct a primitive early (in a throwaway test) to catch `allow_empty` problems before the schema layer consumes it.
- When in doubt, prefer `REPLACE` mode — it's more forgiving and matches how `Safe_Id` behaves.
