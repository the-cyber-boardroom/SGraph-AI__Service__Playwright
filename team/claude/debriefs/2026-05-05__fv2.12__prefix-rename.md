## What was done

Mechanical sweep renaming all 30 top-level components + 8 _shared components + 8 plugin cards from `sp-cli-*` ‚Üí `sg-compute-*`. Zero `sp-cli-` tag/class/filename references remain in active code paths.

### Operations

1. `git mv` all `components/sp-cli/sp-cli-{name}/` ‚Üí `components/sp-cli/sg-compute-{name}/`
2. `git mv` all `components/sp-cli/_shared/sp-cli-{name}/` ‚Üí `.../_shared/sg-compute-{name}/`
3. `git mv` all `plugins/{name}/v0/v0.1/v0.1.0/sp-cli-{name}-card.js` ‚Üí `sg-compute-{name}-card.js` (+ `.html`, `.css`)
4. `git mv` all individual `.js`/`.html`/`.css` files within renamed directories
5. `find ... | xargs sed -i 's/sp-cli-/sg-compute-/g; s/SpCli/SgCompute/g'` ‚Äî file contents

### What was intentionally NOT changed

- **Event names** (`sp-cli:settings.loaded`, `sp-cli:nav.selected`, `sp-cli:node.selected`, etc.) ‚Äî these are the event vocabulary namespace, not component identifiers. Changing them would break all inter-component communication contracts.
- **localStorage keys** (`sp-cli:admin:root-layout:v2`, `sp-cli:host-api-keys`) ‚Äî changing these would silently drop all saved user layouts.
- **`sg-remote-browser`** ‚Äî already uses `sg-` prefix; unchanged beyond a historical comment update.
- **`sg-header`, `sg-stack-card`, `sg-api-client`, etc.** in `shared/components/` ‚Äî already use `sg-` prefix.
