### Service classes (9 of 12)
- `service/JS__Expression__Allowlist` — deny-all by default; exact-match gate for the EVALUATE step.
- `service/Request__Validator` — single home for cross-schema validation (spec §9). Methods: `validate_session_create`, `validate_step`, `validate_sink_configs`, `validate_step_ids_unique`, `validate_sequence_timeout`, `reject`. Raises `HTTPException(422)` carrying `Schema__Error__Response`.
- `service/Capability__Detector` — env-driven deployment target detection + per-target `Schema__Service__Capabilities` profiles (lambda, claude_web, laptop, ci, container). Builds `Schema__Service__Info` for `/info` and `Schema__Health__Check` for `/health`.
- `service/Session__Manager` — in-memory session lifecycle (create/get/close, capture config, start time, artefact accumulation, action counters, active-session listing, healthcheck). No Playwright calls; Browser objects are held as opaque refs.
- `service/Sequence__Dispatcher` — parse surface (`parse_single_step`, `parse_steps`) wrapping `dispatcher/step_schema_registry.parse_step`. `execute_step` deferred to Phase 2.5 (needs `Step__Executor`).
- `service/Artefact__Writer` — sink routing for INLINE (base64), VAULT, S3, LOCAL_FILE. Assembles `Schema__Artefact__Ref` with size + 10-hex sha256 identity hash. `write_bytes_to_local` fully implemented; `write_bytes_to_vault` / `write_bytes_to_s3` / `read_from_vault` / `write_to_vault` are subclass-overridable seams (real vault + osbot-aws clients not yet wired). `capture_*` deferred to Phase 2.7.
- `service/Credentials__Loader` — vault ↔ Playwright `BrowserContext` glue. `apply(session_id, session_manager, credentials)` loads cookies / storage state via the writer's vault seams and sets `extra_http_headers`. `save_state_to_vault` persists state back. No direct vault client; no `page.*` calls (context-level only).
- `service/Browser__Launcher` — real Chromium process lifecycle. `launch(browser_config)` spawns via `sync_playwright().chromium.launch(...)`; lazy runtime start; per-session `browsers` registry; `stop(session_id)` / `stop_all()`; healthcheck. `ENV_VAR__CHROMIUM_EXECUTABLE` escape hatch for pre-installed Chromium builds. Only LOCAL_SUBPROCESS + CHROMIUM supported today (CDP_CONNECT, BROWSERLESS, Firefox, WebKit raise `NotImplementedError`). The only non-`Step__Executor` class permitted to import `playwright.sync_api`.
- `service/Playwright__Service` — thin orchestrator, Phase 2.7 health surface only. Composes Capability__Detector + Session__Manager + Browser__Launcher + Request__Validator. `setup()` primes detection (idempotent). Methods: `get_service_info()`, `get_capabilities()`, `get_health()` (aggregates browser_launcher + session_manager + connectivity sub-checks into `Schema__Health`). Session/action/sequence methods are NOT on this class yet — they land in Phase 2.10 when Routes__Session / Routes__Browser / Routes__Sequence arrive.

### FastAPI app (Phase 2.7 wiring)
- `fast_api/Fast_API__Playwright__Service` — extends `Serverless__Fast_API` (osbot-fast-api-serverless). Holds one `Playwright__Service`. `setup()` disables API-key middleware (health probes must answer plain curl — re-enables in Phase 2.10 scoped to mutating routes), primes the service, then defers to the base class for middleware + default routes. `setup_routes()` registers only `Routes__Health` today.
- `fast_api/routes/Routes__Health` — `Fast_API__Routes` subclass with `tag='health'`. Three `GET` endpoints: `/health/info` (→ `Schema__Service__Info`), `/health/status` (→ `Schema__Health`), `/health/capabilities` (→ `Schema__Service__Capabilities`). Pure delegation to `Playwright__Service`. Module exports `TAG__ROUTES_HEALTH` + `ROUTES_PATHS__HEALTH` constants.
- `fast_api/lambda_handler.run()` — works end-to-end: `python -m sgraph_ai_service_playwright.fast_api.lambda_handler` boots uvicorn on :8000 and all three `/health/*` endpoints return 200 + valid Schema__* JSON (verified locally 2026-04-17).
