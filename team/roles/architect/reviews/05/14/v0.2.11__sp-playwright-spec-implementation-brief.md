1. The correction in one paragraph
═══════════════════════════════════════════════════════════════════════════════

Commit 7b32a17 built a "POD backend" — a `Host_Plane__Client` that places one
`sg-playwright` container as a pod onto a PRE-EXISTING host. That was wrong.
`sp playwright create` must do what every other spec does and what
`provision_ec2.py` proved: **launch one fresh EC2 instance running a
docker-compose stack of 2–3 containers via user-data**, with a security
group and an auto-terminate timer. The host control plane is a container ON
that instance, not a remote API the CLI calls. This brief maps the rework
file-by-file. It is a structural clone of the `vnc` spec with the
container set swapped.

2. Target model
═══════════════════════════════════════════════════════════════════════════════

```
  sp playwright create [--with-mitmproxy]
        │
        ├── ensure security group  (ingress :8000 from caller /32, opt :443)
        ├── render docker-compose.yml   ← sg-playwright [+ mitmproxy]
        ├── render user-data            ← install docker → write compose →
        │                                 `docker compose up -d` →
        │                                 start host-control container →
        │                                 auto-terminate timer
        └── run_instance(...)           ← InstanceInitiatedShutdownBehavior=terminate
        │
        ▼
  ┌──────────────────────────────────────────────────────────┐
  │  EC2 instance  (auto-terminates after max_hours)          │
  │                                                           │
  │  ┌───────────────────┐  ┌──────────────────┐              │
  │  │ host control plane │  │ sg-playwright     │             │
  │  │ sgraph_ai_service_ │  │ diniscruz/        │             │
  │  │ playwright_host    │  │ sg-playwright     │             │
  │  │ :19009 (ECR image) │  │ :8000 (Docker Hub)│             │
  │  │ docker.sock mount  │  │   HTTP_PROXY ─────┼──┐          │
  │  └───────────────────┘  └──────────────────┘  │          │
  │                          ┌──────────────────┐ │          │
  │                          │ mitmproxy         │◄┘          │
  │                          │ (only --with-     │            │
  │                          │  mitmproxy)       │            │
  │                          │ intercept script  │            │
  │                          └──────────────────┘            │
  └──────────────────────────────────────────────────────────┘
```

- **2 containers** default: host control plane + sg-playwright.
- **3 containers** with `--with-mitmproxy`: + mitmproxy, and sg-playwright is
  pointed at it via `SG_PLAYWRIGHT__DEFAULT_PROXY_URL` +
  `SG_PLAYWRIGHT__IGNORE_HTTPS_ERRORS=1`.
- One model. No "POD backend". No `Host_Plane__Client` for CRUD — create /
  list / info / delete / health are EC2-by-tag operations, exactly like vnc.
  (A host-plane client may return LATER for runtime ops — logs, restart,
  attach — against the running host-control container; out of scope here.)

6. Playwright__Compose__Template — the app-container compose
═══════════════════════════════════════════════════════════════════════════════

Renders `docker-compose.yml` for the APP containers only (the host-control
container is started separately by the user-data builder — see §7, mirrors
`provision_ec2.py`). One service by default, two with `--with-mitmproxy`.

```yaml
services:
  sg-playwright:
    image: diniscruz/sg-playwright:{image_tag}
    container_name: sg-playwright
    ports:
      - "8000:8000"
    environment:
      - FAST_API__AUTH__API_KEY__VALUE={api_key}
      # --with-mitmproxy only:
      - SG_PLAYWRIGHT__DEFAULT_PROXY_URL=http://mitmproxy:8080
      - SG_PLAYWRIGHT__IGNORE_HTTPS_ERRORS=1
    networks: [sg-net]
    restart: unless-stopped
    # depends_on: [mitmproxy]   ← only with --with-mitmproxy

  # --with-mitmproxy only:
  mitmproxy:
    image: mitmproxy/mitmproxy:10.4.2          # pin — same version vnc pins
    container_name: sg-mitmproxy
    command:
      - mitmdump
      - --listen-host=0.0.0.0
      - --listen-port=8080
      - --scripts=/opt/sg-playwright/interceptors/active.py
    volumes:
      - /opt/sg-playwright/interceptors:/opt/sg-playwright/interceptors:ro,z
    networks: [sg-net]
    restart: unless-stopped

networks:
  sg-net:
    driver: bridge
```
- No external port on mitmproxy — docker-network-only (matches the vault-app
  v2-delta plan §8: passive, no admin surface, no published ports).
- `render(image_tag, with_mitmproxy)` — the mitmproxy service block and the
  three proxy-related lines on sg-playwright are emitted only when
  `with_mitmproxy=True`. Lock the placeholder set with a test.

7. Playwright__User_Data__Builder — the boot script
═══════════════════════════════════════════════════════════════════════════════

`render(stack_name, region, compose_yaml, api_key, with_mitmproxy,
intercept_script, registry, max_hours) -> str`. Bash sections, in order
(mirrors `Vnc__User_Data__Builder` + the host-control block from
`provision_ec2.py:288-310`):

1. **header** — `set -euo pipefail`, log to `/var/log/sg-playwright-boot.log`.
2. **docker install** — `dnf install -y docker`; enable; install the compose
   CLI plugin (same curl as vnc).
3. **layout** — `mkdir -p /opt/sg-playwright/interceptors`.
4. **intercept script** (only if `with_mitmproxy` and `intercept_script`) —
   write the resolved addon script to
   `/opt/sg-playwright/interceptors/active.py`. If `intercept_script` is
   empty, write a no-op passthrough that logs NDJSON to stdout (the
   vault-app v2-delta default). Resolution of a `vault://` path is a
   `Playwright__Interceptor__Resolver` helper — mirror
   `Vnc__Interceptor__Resolver`; if that is too large for this slice, accept
   an inline script string first and add vault resolution as a follow-up.
5. **compose** — heredoc `{compose_yaml}` to
   `/opt/sg-playwright/docker-compose.yml`; `docker compose up -d`.
6. **host control plane** — inline `docker run` of
   `{registry}/sgraph_ai_service_playwright_host:latest`, `--privileged`,
   docker.sock mounted, `-p 19009:8000`, with its own generated API key
   pushed to SSM (verbatim from `provision_ec2.py:288-310`). This is the
   2nd/3rd container's sibling — always present.
7. **shutdown** — `Section__Shutdown.render(max_hours)` when `max_hours>0`.

The `api_key` (the sg-playwright app key) is already baked into the compose
YAML by §6, so step 5 carries it. The host-control key is separate and
generated on-box per step 6.

8. Routes__Playwright__Stack — changes
═══════════════════════════════════════════════════════════════════════════════

Same five endpoints, same paths. Query params change from
`host_url`/`host_api_key` to `region` (defaulting to `DEFAULT_REGION` in the
service), exactly like `Routes__Vnc__Stack`:

```
POST   /playwright/stack                  body: dict → from_json → create_stack
GET    /playwright/stacks                 ?region=  → list_stacks
GET    /playwright/stack/{name}           ?region=  → get_stack_info (404 on None)
DELETE /playwright/stack/{name}           ?region=  → delete_stack (404 if no terminated ids)
GET    /playwright/stack/{name}/health    ?region=  → health
```
`create` keeps the `body: dict` + `from_json` round-trip (Swagger-friendly).

11. What this is NOT (scope guard)
═══════════════════════════════════════════════════════════════════════════════

- NOT a host-plane RPC client. CRUD is EC2-by-tag.
- NOT two backends. One: EC2 + compose stack.
- NOT a baked-AMI fast path (that is a later optimisation — fresh-install
  user-data first, exactly as vnc shipped).
- NOT vault-write capability in the launcher (the running sg-playwright app
  already stages artefacts via its VAULT sink; the launcher writes nothing).
- Runtime ops against the host-control container (logs / restart / attach)
  are a FOLLOW-UP — that is where a typed host-plane client legitimately
  returns, matching the vault-app v2-delta plan's Slice 2.
</content>
