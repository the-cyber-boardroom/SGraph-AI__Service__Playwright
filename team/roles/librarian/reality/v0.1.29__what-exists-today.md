**Changes since v0.1.24 (non-breaking for the public HTTP surface; breaking for env-var names + Lambda-image contract):**

- Version file bumped `v0.1.24 → v0.1.29` (v0.1.25–v0.1.28 consumed by the S3-zip hot-swap slice and the rename-to-agentic refactor).
- **L1/L2 subpackage scaffolding.** New subpackages under `sgraph_ai_service_playwright/`:
  - `agentic_fastapi/` — L1 generic FastAPI base, no AWS:
    - `Agentic_FastAPI.py` — base class; `Fast_API__Playwright__Service` extends this.
    - `Agentic_Admin_API.py` — `/admin/*` surface (read-only).
    - `Agentic_Boot_State.py` — module-level ring buffer (`append_boot_log / get_boot_log`) + last-error holder (`set_last_error / get_last_error`) + `reset_boot_state` test hook. Lives in L1 so the admin API can read it without pulling in L2.
    - `schemas/Schema__Agentic__{Health,Info,Env,Boot_Log,Error,Manifest,Skill,Capabilities}.py` — wire schemas for every admin endpoint.
  - `agentic_fastapi_aws/` — L2 AWS-aware boot:
    - `Agentic_Code_Loader.py` — resolves code source: `local:` env-override → `s3:bucket/key` → `passthrough:sys.path`; mutates `sys.path` when S3 or local is used.
    - `Agentic_Boot_Shim.py` — Lambda boot orchestration. Reads `/var/task/image_version`, calls `Agentic_Code_Loader.resolve()`, imports `Fast_API__Playwright__Service`, pins `(handler, app)`. Writes progress to `Agentic_Boot_State` (three log lines on success, `status=degraded` + `set_last_error(…)` on failure). Inside Lambda it swallows the import error and returns an error string; outside Lambda it re-raises.
  - `skills/` — markdown SKILL files: `skill__human.md`, `skill__browser.md`, `skill__agent.md` (first-pass placeholders, TODO-tagged for v0.2).
- **Env-var rename.** One-shot cutover from `SG_PLAYWRIGHT__*` boot-loader vars to `AGENTIC_*`:
  - `ENV_VAR__AGENTIC_APP_NAME / AGENTIC_APP_STAGE / AGENTIC_APP_VERSION` — identity.
  - `ENV_VAR__AGENTIC_IMAGE_VERSION` — baked by Dockerfile at build time.
  - `ENV_VAR__AGENTIC_CODE_SOURCE` — `s3:bucket/key` | `local:/path` | `passthrough:sys.path` — surfaced on `/admin/health` + `/admin/info`.
  - `ENV_VAR__AGENTIC_CODE_LOCAL_PATH` — local iteration escape hatch (boots in ~5 s).
  - `ENV_VAR__AGENTIC_CODE_SOURCE_S3_BUCKET / _S3_KEY` — S3 fetch coordinates.
  - No dual-read compat shim. Old `SG_PLAYWRIGHT__*` loader vars do nothing.
- **New S3 bucket + path scheme.** `{account}--sgraph-ai--{region}` (e.g. `745506449035--sgraph-ai--eu-west-2`). Code zips live at `apps/sg-playwright/{stage}/v{X.Y.Z}.zip`. Old `{account}--sg-playwright--{region}` bucket is left alone (historical).
- **New one-command deploy.** `scripts/deploy_code.py --stage <dev|main|prod>` — ensures bucket, packages the app, uploads the zip, updates the Lambda's env to point at the new version (and strips the v0.1.28-era `SG_PLAYWRIGHT__*` loader vars), smoke-tests the Function URL `/admin/health`. Target round-trip ~30 s cloud, ~5 s local.
- **`/admin/*` surface landed.** Eight read-only endpoints, unauthenticated (they bypass the API-key middleware via an enumerated path list in `Agentic_FastAPI.setup()`). Full list under API surface below.
- **`capabilities.json` stub at repo root.** Declares axioms `["statelessness", "least-privilege-by-declaration", "self-description"]` and `declared_narrowing = []` (lockdown layers deferred). Served at `/admin/capabilities`. Preferred at `/var/task/capabilities.json` when present (baked into the Lambda image).
- **`Fast_API__Playwright__Service` extends `Agentic_FastAPI`.** Inheritance chain now: `Serverless__Fast_API → Agentic_FastAPI → Fast_API__Playwright__Service`. `setup_routes()` calls `super().setup_routes()` before wiring playwright routes, so the admin surface lands on every agentic app for free.
- **CI single-track, docker-based jobs disabled.** `build-docker-image`, `push-to-ecr`, `run-integration-tests`, `increment-tag`, `deploy-lambda`, `smoke-test` all gated `if: false` in `.github/workflows/ci-pipeline.yml`. New `deploy-code` job runs `python scripts/deploy_code.py --stage ${{ inputs.target_deploy }}`, gated on `check-aws-credentials has-credentials == 'true'` and needs `run-unit-tests`.
- **Public HTTP surface unchanged.** Same 10 endpoints as v0.1.24 + 8 new `/admin/*` endpoints.

### FastAPI app

- `agentic_fastapi/Agentic_FastAPI` — base class. `setup()` extends `AUTH__EXCLUDED_PATHS` with the 8 admin paths (middleware uses exact match, so unknown `/admin/skills/{name}` audiences 401 before the route's 404 can land — accepted). `setup_routes()` calls `super().setup_routes()` then `self.add_routes(Agentic_Admin_API, skills_dir, capabilities_path)`. `resolve_skills_dir()` defaults to the in-tree `skills/` folder; `resolve_capabilities_path()` prefers `/var/task/capabilities.json` when present (baked into the Lambda image), else the repo-root stub.
- `agentic_fastapi/Agentic_Admin_API` — `Fast_API__Routes` subclass registering the 8 admin routes. `env()` filters `os.environ` by `AGENTIC_` prefix; `skills__name(name)` reads `skills/skill__{name}.md` and 404s on unknown; `capabilities()` reads the json file, returns `Schema__Agentic__Capabilities`.
- `agentic_fastapi/Agentic_Boot_State` — module-level ring buffer (`BOOT_LOG_MAX_LINES = 200`) + `_last_error` string; `get_boot_log()` returns a copy (callers can't mutate); `set_last_error(None)` coerces to `''`; `reset_boot_state()` for tests.
- `agentic_fastapi_aws/Agentic_Boot_Shim` — writes to `Agentic_Boot_State` on each stage (`image_version=…`, `code_source=…`, `status=loaded`) and on failure (`status=degraded …` + `set_last_error(CRITICAL ERROR: …)`).
- `fast_api/Fast_API__Playwright__Service` — now extends `Agentic_FastAPI`; `setup_routes()` starts with `super().setup_routes()` so the admin surface always lands.
- `fast_api/lambda_handler.run()` — unchanged (still fires everything on import).
