**Changes since v0.1.23 (breaking — "stateless one-shot surface"):**

- Version file bumped `v0.1.23 → v0.1.24`.
- **Stateful session API deleted.** Removed:
  - Routes: `Routes__Session.py`, `Routes__Quick.py`
  - Services: `Session__Manager.py`, `Action__Runner.py`
  - Schemas: `Schema__Session__{Create__Request,Create__Response,Close__Response,State__Save__Request,State__Save__Response,Info}.py`, `Schema__Action__{Request,Response}.py`, all `Schema__Quick__*` request/response schemas, `Enum__Session__Status`.
  - Tests: `test_Action__Runner.py`, `test_Session__Manager.py`, `test_Routes__Session.py`, `test_Routes__Quick.py`.
- **`/browser/*` reimplemented as stateless one-shots.** Six POST endpoints, each launches fresh `sync_playwright` + Chromium, runs one action (with optional pre-click), tears down in try/finally:
  - `POST /browser/navigate`    → `Schema__Browser__One_Shot__Response`
  - `POST /browser/click`       → `Schema__Browser__One_Shot__Response`
  - `POST /browser/fill`        → `Schema__Browser__One_Shot__Response`
  - `POST /browser/screenshot`  → raw `image/png` + `X-*-Ms` timing headers
  - `POST /browser/get-content` → `Schema__Browser__One_Shot__Response` (with `html`)
  - `POST /browser/get-url`     → `Schema__Browser__One_Shot__Response`
  Each body is flat — `url` mandatory + action-specific params + optional `browser_config` (with `proxy` incl. `Schema__Proxy__Auth__Basic`) + optional `wait_until` / `timeout_ms`. No nested `capture_config` at the caller surface.
- **`/sequence/execute` unchanged URL; schema simplified.** `Schema__Sequence__Request` drops `session_id` and `close_session_after`. `Sequence__Runner.resolve_session` collapses to always-ad-hoc launch via `browser_launcher.launch` + try/finally teardown.
- **New schemas:**
  - `schemas/browser/Schema__Browser__Navigate__Request`
  - `schemas/browser/Schema__Browser__Click__Request`
  - `schemas/browser/Schema__Browser__Fill__Request`
  - `schemas/browser/Schema__Browser__Screenshot__Request`
  - `schemas/browser/Schema__Browser__Get_Content__Request`
  - `schemas/browser/Schema__Browser__Get_Url__Request`
  - `schemas/browser/Schema__Browser__One_Shot__Response` (shared JSON response shape; `url`, `final_url`, `duration_ms`, `timings`, action-specific payload fields)
  - `schemas/browser/Schema__Browser__Screenshot__Result` (internal; pairs `png_bytes` with `timings` so the route can emit HTTP headers alongside the PNG body — analogue of the v0.1.13 `Schema__Quick__Screenshot__Result` it replaces)
- **`Playwright__Service` surface:**
  - Removed: `session_create / session_list / session_get / session_save_state / session_close / execute_action / quick_html / quick_screenshot`.
  - Added: `browser_navigate / browser_click / browser_fill / browser_screenshot / browser_get_content / browser_get_url` — each builds a throwaway sequence via `Sequence__Runner` and extracts the relevant result.
  - Kept: `get_service_info / get_capabilities / get_health / execute_sequence / generate_trace_id`.
  - `setup()` no longer wires `Session__Manager` or `Action__Runner`; only primes `Capability__Detector`, shares `capability_detector / request_validator / browser_launcher / credentials_loader / proxy_auth_binder` into `Sequence__Runner`.
- **Capabilities flags:**
  - Removed: `supports_session_lifecycle`, `max_concurrent_sessions`.
  - Added: `supports_one_shot_browser_actions` (always `true`).
  - Kept: `supports_proxy_auth_basic` (from v0.1.23), `supports_inline_artefacts`, `supports_vault_refs`, `supports_local_file_sinks`.

## Summary

**Phase 1 complete + Phase 2 largely complete + v0.1.13 clean-state milestone + v0.1.23 proxy-auth fix + v0.1.24 stateless-surface refactor.**

**API surface: 10 endpoints.**
- `GET  /health/info`
- `GET  /health/status`
- `GET  /health/capabilities`
- `POST /browser/navigate`
- `POST /browser/click`
- `POST /browser/fill`
- `POST /browser/screenshot`
- `POST /browser/get-content`
- `POST /browser/get-url`
- `POST /sequence/execute`

Plus osbot-fast-api's `/auth/set-cookie-form` HTML UI + `/auth/set-auth-cookie` POST (cookie-setting helpers, not in the API-key middleware scope).

**Service classes: 10 of 10 live** (down from v0.1.13's 12 — `Session__Manager` and `Action__Runner` deleted).

Every mutating request launches a fresh `sync_playwright()` Node subprocess and a fresh Chromium process, runs exactly what was asked, and tears both down in `try/finally` before the response returns. No server-side state carries between requests. Cold-start-per-call is ~550 ms fixed overhead (~420 ms sync_playwright + ~130 ms Chromium) — accepted tradeoff for Lambda safety + security.

One-class-per-file throughout. CI is green (unit → build → integration → push → deploy → smoke on Lambda).
