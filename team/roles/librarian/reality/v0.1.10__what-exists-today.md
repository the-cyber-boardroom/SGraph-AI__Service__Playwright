### Service classes (10 of 12)
- `service/JS__Expression__Allowlist` ‚Äî deny-all by default; exact-match gate for the EVALUATE step.
- `service/Request__Validator` ‚Äî single home for cross-schema validation (spec ¬ß9). Methods: `validate_session_create`, `validate_step`, `validate_sink_configs`, `validate_step_ids_unique`, `validate_sequence_timeout`, `reject`. Raises `HTTPException(422)` carrying `Schema__Error__Response`.
- `service/Capability__Detector` ‚Äî env-driven deployment target detection + per-target `Schema__Service__Capabilities` profiles (lambda, claude_web, laptop, ci, container). Builds `Schema__Service__Info` for `/info` and `Schema__Health__Check` for `/health`.
- `service/Session__Manager` ‚Äî in-memory session lifecycle (create/get/close, capture config, start time, artefact accumulation, action counters, active-session listing, healthcheck). No Playwright calls; Browser objects are held as opaque refs.
- `service/Sequence__Dispatcher` ‚Äî parse surface (`parse_single_step`, `parse_steps`) wrapping `dispatcher/step_schema_registry.parse_step`. `execute_step` deferred to Phase 2.5 (needs `Step__Executor`).
- `service/Artefact__Writer` ‚Äî sink routing for INLINE (base64), VAULT, S3, LOCAL_FILE. Assembles `Schema__Artefact__Ref` with size + 10-hex sha256 identity hash. `write_bytes_to_local` fully implemented; `write_bytes_to_vault` / `write_bytes_to_s3` / `read_from_vault` / `write_to_vault` are subclass-overridable seams (real vault + osbot-aws clients not yet wired). Typed capture wrappers `capture_screenshot(data, sink_config)` / `capture_page_content(...)` / `capture_pdf(...)` (Phase 2.9) pin the artefact type so Step__Executor callers don't repeat it; each returns `None` when the sink_config is disabled.
- `service/Credentials__Loader` ‚Äî vault ‚Üî Playwright `BrowserContext` glue. `apply(session_id, session_manager, credentials)` loads cookies / storage state via the writer's vault seams and sets `extra_http_headers`. `save_state_to_vault` persists state back. No direct vault client; no `page.*` calls (context-level only).
- `service/Browser__Launcher` ‚Äî real Chromium process lifecycle. `launch(browser_config)` spawns via `sync_playwright().chromium.launch(...)`; lazy runtime start; per-session `browsers` registry; `stop(session_id)` / `stop_all()`; healthcheck. `ENV_VAR__CHROMIUM_EXECUTABLE` escape hatch for pre-installed Chromium builds. Only LOCAL_SUBPROCESS + CHROMIUM supported today (CDP_CONNECT, BROWSERLESS, Firefox, WebKit raise `NotImplementedError`). The only non-`Step__Executor` class permitted to import `playwright.sync_api`.
- `service/Step__Executor` ‚Äî **only class (alongside `Browser__Launcher`) permitted to import `playwright.sync_api`**. `execute(page, step, step_index, capture_config)` dispatches on `step.action`. First-pass handlers (Phase 2.9): `execute_navigate` / `execute_click` / `execute_fill` / `execute_screenshot` / `execute_get_content` / `execute_get_url`. Screenshot and GET_CONTENT bytes are handed to `artefact_writer.capture_*()` ‚Äî Step__Executor never writes to a sink directly. Every handler catches `Exception`, times the step, and returns a uniformly-shaped `Schema__Step__Result__*` (PASSED or FAILED with error_message). Deferred handlers (`execute_press / select / hover / scroll / wait_for / video_start / video_stop / evaluate / dispatch_event / set_viewport`, 10 total) raise `NotImplementedError` with a `Phase 2.11` marker rather than falling through to AttributeError. Helpers: `passed_result`, `failed_result`, `resolve_id` (falls back to `Step_Id(str(step_index))`), `filter_refs` (drops None refs from disabled sinks), `now_ms` (wall-clock seam for tests).
- `service/Playwright__Service` ‚Äî thin orchestrator. Composes Capability__Detector + Session__Manager + Browser__Launcher + Credentials__Loader + Artefact__Writer + Request__Validator. `setup()` primes detection (idempotent). Health methods: `get_service_info()`, `get_capabilities()`, `get_health()` (aggregates browser_launcher + session_manager + connectivity sub-checks into `Schema__Health`). Session methods (Phase 2.10 Slice A ‚Äî commit `4053374`): `create_session(request)` / `list_sessions()` / `get_session(session_id)` / `save_session_state(session_id, request)` / `close_session(session_id)`. All validation flows through `Request__Validator`; 404s are raised as `HTTPException` at the service boundary. Action / sequence methods still outstanding (Slice B/C).

### Docker infrastructure (Phase 2.8)
- `docker/Docker__SGraph_AI__Service__Playwright__Base` ‚Äî shared init for all four Docker classes. `setup()` instantiates `Create_Image_ECR` against `docker/images/sgraph_ai_service_playwright/` and `Deploy_Lambda` against `fast_api.lambda_handler.run`. Helpers: `api_docker()`, `image_uri()` (`:latest`), `repository()`, `path_docker()`, `path_dockerfile()`, `lambda_function()`. - `docker/Build__Docker__SGraph_AI__Service__Playwright` ‚Äî `build_docker_image()` stages the dockerfile + requirements.txt + whole package source in a tempdir, then calls the docker SDK `client.images.build()` **directly** (bypassing osbot_docker's `Docker_Image.build()` because it's `@catch`-decorated and swallows `BuildError`/`APIError` into a `{'status': 'error'}` dict that was silently slipping past the `assert result is not None` check). Passes `dockerfile='dockerfile'` explicitly because the daemon defaults to `Dockerfile` (capital D, case-sensitive on Linux); returns `{'status': 'ok', 'image_id', 'tags'}` which the test verifies contain the ECR URI. - `docker/ECR__Docker__SGraph_AI__Service__Playwright` ‚Äî `ecr_setup()` (idempotent repo create), `publish_docker_image()` (runs `check_for_docker_config_json()` first, then `push_image()`). `check_for_docker_config_json()` carries forward the Docker Desktop credsStore workaround (`{'credsStore': 'desktop'}` breaks the Python Docker API ECR login ‚Üí delete the file).
- `docker/Lambda__Docker__SGraph_AI__Service__Playwright` ‚Äî `create_lambda(delete_existing, wait_for_active)` builds Lambda at `memory_size=5120 MB` (osbot_aws reads `memory_size`, **not** `memory` ‚Äî wrong attr silently falls back to 512 MB and Playwright OOMs at cold start with `Runtime.ExitError`), `architecture=x86_64`, `timeout=300s`, wires env vars, deletes existing if asked, creates, waits for `Active`, then creates the Function URL. `set_lambda_env_vars()` propagates seven env vars from the process env to the Lambda env (five `SG_PLAYWRIGHT__*` + two `FAST_API__AUTH__API_KEY__*` for the middleware) and pins `SG_PLAYWRIGHT__DEPLOYMENT_TARGET='lambda'`. `create_lambda_function_url()` does the delete ‚Üí osbot_aws `function_url_create_with_public_access()` (statement 1: `lambda:InvokeFunctionUrl` + `StringEquals lambda:FunctionUrlAuthType=NONE`) ‚Üí boto3 `add_permission(InvokedViaFunctionUrl=True)` (statement 2: `lambda:InvokeFunction` + `Bool lambda:InvokedViaFunctionUrl=true`) dance ‚Äî AWS requires **both** statements for public Function URLs created after October 2025 (osbot_aws only emits the first; without the second AWS returns 403 at the URL). `update_lambda_function()` + `function_url()` for post-deploy ops. Tuning constants: `LAMBDA_MEMORY_MB=5120`, `LAMBDA_ARCHITECTURE='x86_64'`, `LAMBDA_TIMEOUT_SECS=300`.
- `docker/Local__Docker__SGraph_AI__Service__Playwright` ‚Äî label-based container reuse (`source=local__docker__sgraph_ai_service_playwright`). `create_or_reuse_container()` / `delete_container()`, `GET()` / `POST()` helpers via `requests`, `local_url()` prefixes `http://localhost:8888`, `uvicorn_server_running()` + `wait_for_uvicorn_server_running()` poll container logs for `Uvicorn running on `.
