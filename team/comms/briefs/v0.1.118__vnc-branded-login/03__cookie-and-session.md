# 03 — Cookie + session shape

## Cookie

```
Set-Cookie: sg-vnc-session=<token>; Path=/; HttpOnly; Secure;
            SameSite=Lax; Max-Age=28800
```

- `HttpOnly` — JS in the chromium viewer can't read it (defence in depth).
- `Secure` — only sent over the TLS we already have.
- `SameSite=Lax` — works for the admin-UI iframe (top-level navigation
  follows from the operator clicking "Open Viewer"). If we ever need it
  to flow with cross-site fetches use `None` + extra CSRF protection.
- `Max-Age=28800` (8h) — matches a typical work session; logout clears it.

## Token shape

A signed token, NOT a session-store entry — keeps the sidecar stateless.

```
token = base64url( payload || hmac_sha256( payload, secret ) )

payload = {
  "stack" : "vnc-clever-noether",  // for log lines, not authz
  "user"  : "operator",
  "iat"   : 1727612400,
  "exp"   : 1727641200
}
```

Verifier rejects if:
- `hmac_sha256(payload, secret)` doesn't match the trailing 32 bytes
- `exp` ≤ now
- `stack` doesn't match the env-var the sidecar was launched with

## Where the secret lives

Generated at boot, written to `/opt/sg-vnc/auth/secret` (32 bytes from
`/dev/urandom`, base64-encoded). Bind-mounted into `sg-auth` at
`/etc/sg-vnc/auth-secret:ro`. Not bind-mounted into nginx or chromium.

If the EC2 reboots, the secret is regenerated by user-data and all
existing cookies become invalid → users see the login form again. That's
the right behaviour for an ephemeral debug stack.

## Why HMAC vs JWT

Same idea, simpler shape, no library needed beyond stdlib `hmac` +
`secrets`. JWT brings algorithm-confusion CVEs and a YAML of options
we don't need. Plain HMAC is auditable in 30 lines of Python.

## CSRF

Login is a single POST from a same-origin form — Tornado/Starlette's
default form parsing handles CSRF via the `SameSite=Lax` cookie.

If the admin UI later needs to hit `/logout` from a cross-origin context,
add a `_csrf` form field tied to the session cookie. Out of scope for
the first slice.

## Audit log

Every successful login and every 401 from `/auth` writes a single line to
`/var/log/sg-vnc-auth.log`:

```
2026-04-29T11:02:14Z LOGIN_OK     operator 31.94.32.66
2026-04-29T11:08:01Z AUTH_REJECT  -        31.94.32.66 reason=expired
```

Read via `sp vnc connect <stack>` then `sudo tail /var/log/sg-vnc-auth.log`.
Volume terminates with the EC2 — no cross-stack persistence.
