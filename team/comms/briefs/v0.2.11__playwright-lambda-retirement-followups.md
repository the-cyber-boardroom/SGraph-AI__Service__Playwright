2. Slice 3 — Flatten `core/` + reparent off `Agentic_FastAPI` (BIG ONE)
═══════════════════════════════════════════════════════════════════════════════

The highest-value deferred item. Riskiest because it is import-churn across
the whole spec — needs its own focused session with a full test run.

2.1 Reparent the FastAPI base class
  - `Fast_API__Playwright__Service` currently extends `Agentic_FastAPI`
    (which extends `Serverless__Fast_API`). Reparent it directly onto
    `Serverless__Fast_API`.
  - This is the precondition for deleting `agentic_fastapi/` (see §3).

2.2 Delete `sg_compute_specs/playwright/core/agentic_fastapi/`
  - `Agentic_FastAPI`, `Agentic_Admin_API`, `Agentic_Boot_State`, and the
    `schemas/Schema__Agentic__*` family.
  - Decision D7 in the review: this is Lambda-era self-description
    scaffolding; the `/admin/*` surface's `info`/`env`/`boot-log`/`error`
    endpoints only return meaningful data when populated by the (now
    deleted) `Agentic_Boot_Shim`. The useful part (`manifest` /
    `capabilities` / `skills`) belongs in the agentic FastAPI of the
    claude-code / claude-ollama specs, rebuilt cleanly there.
  - Delete the matching tests under `tests/unit/agentic_fastapi/`.
  - NOTE: `tests/unit/fast_api/test_Fast_API__Playwright__Service.py`
    asserts `handler()` returns a Mangum handler "for AWS LWA / Lambda
    integration" — update or drop those assertions in the same pass.

2.3 Flatten `core/` into the spec root
  - Per `sg_compute/brief/05__stack_contract.md`, a spec is
    `sg_compute_specs/<name>/{schemas,service,cli}/` — not a nested
    `core/` tree. Move `core/{service,schemas,dispatcher,fast_api,
    client,metrics,skills,consts}` up one level into
    `sg_compute_specs/playwright/`.
  - Update every `from sg_compute_specs.playwright.core.*` import
    across the repo (and the Dockerfile CMD, and `manifest.py`'s
    `_read_version()` which currently reads `core/version`).
  - The `version` symlink (`playwright/version` → `core/version`)
    collapses to a plain file at the spec root.

2.4 Rename the entry module
  - `core/fast_api/lambda_handler.py` is now just a uvicorn runner — the
    name is a Lambda-era leftover. Rename to `entrypoint.py` (or similar),
    drop the `sys.path.append('/opt/python')` LWA workaround, and update
    the Dockerfile CMD (`python3 -m sg_compute_specs.playwright.fast_api.entrypoint`).
