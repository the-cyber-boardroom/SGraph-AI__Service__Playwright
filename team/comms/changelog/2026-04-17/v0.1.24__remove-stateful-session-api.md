# v0.1.24 — Remove stateful session API, repurpose /browser/* as stateless one-shots — 2026-04-17

## What shipped

### Surface reshape (breaking change, pre-GA)

Stateful session lifecycle deleted; `/browser/*` and `/quick/*` collapse into a single stateless one-shot surface.

| Before (v0.1.23) — 25 routes | After (v0.1.24) — 10 routes |
|------------------------------|-----------------------------|
| `POST /session/create` | *(removed)* |
| `GET  /session/list` | *(removed)* |
| `GET  /session/get/by-id/{id}` | *(removed)* |
| `POST /session/save-state/{id}` | *(removed)* |
| `DELETE /session/close/{id}` | *(removed)* |
| `POST /browser/navigate` *(stateful — required prior `/session/create`)* | `POST /browser/navigate` *(stateless — takes `url` + optional `browser_config`)* |
| `POST /browser/click` *(stateful)* | `POST /browser/click` *(stateless — `url` + `selector` + optional `browser_config`)* |
| `POST /browser/fill` *(stateful)* | `POST /browser/fill` *(stateless — `url` + `selector` + `value` + optional `browser_config`)* |
| `POST /browser/screenshot` *(stateful)* | `POST /browser/screenshot` *(stateless — `url` + optional `selector`/`full_page`; returns raw `image/png`)* |
| `POST /browser/get-content` *(stateful)* | `POST /browser/get-content` *(stateless — `url`; returns HTML + final URL + timings)* |
| `POST /browser/get-url` *(stateful)* | `POST /browser/get-url` *(stateless — `url` + optional click; returns post-navigation URL)* |
| `POST /quick/html` | *(merged into `POST /browser/get-content`)* |
| `POST /quick/screenshot` | *(merged into `POST /browser/screenshot`)* |
| `POST /sequence/execute` | `POST /sequence/execute` *(unchanged URL; `session_id` + `close_session_after` fields removed)* |
| `GET /health/info` / `GET /health/status` / `GET /health/capabilities` | *(unchanged)* |

Every `/browser/*` request now:
1. Launches a fresh `sync_playwright` + Chromium process from `browser_config` (or defaults if omitted).
2. Runs the one declared action (plus optional pre-click).
3. Tears the browser down in a `try/finally` before the response is returned.

No state carries between requests. No `Session_Id` on the wire anywhere.

### Rationale

- **Lambda-hostile.** `Session__Manager` stored `browser` / `context` handles in an in-process dict. Two consecutive HTTP calls can land on different warm containers — the second returns `404 Session X not found` even though the client "just created" it. Not fixable without cross-container state (SQS, Redis, DynamoDB), and none of those are warranted.
- **Security risk.** Warm containers reuse the Python process. A leaked / un-closed session could carry cookies, storage state, or proxy credentials from user A into user B's request on the same warm container. The stateless model makes that impossible by construction.
- **No caller benefit.** Neither the SG/Send orchestrator nor the ad-hoc probes in `tests/local/` depend on the stateful path — every real call already used `/sequence/execute` (which auto-creates an ad-hoc session when `session_id` is None) or `/quick/*`. Deleting the stateful branch deletes unused code.
- **API simplicity.** `/quick/*` existed only because the stateful `/browser/*` required a prior `/session/create` — which was unfriendly in Swagger. Collapsing the two surfaces removes the "which one do I call?" decision.

### Bug #4 (`Session not found` on sequence/execute) resolution

The v0.1.23 Sequence__Runner fix (commit `90db20d`) reordered the `browser_config is None` check to fire before the 404 path, catching the case where osbot-fast-api auto-generates a `Session_Id` when the wire body omits it. That fix **collapses entirely** in v0.1.24 — `session_id` is no longer a field on `Schema__Sequence__Request`, so `resolve_session` always takes the ad-hoc create path. The regression test lives on under a renamed assertion.

### Routes

`Routes__Browser` rewritten. Six POST endpoints, each wrapping `Playwright__Service.browser_<action>(request)` and returning either `Schema__Browser__One_Shot__Response.json()` or (screenshot only) a raw `image/png` Response with `X-*-Ms` timing headers.

### Service

- `Playwright__Service.execute_action()` / `quick_html()` / `quick_screenshot()` / all `session_*` methods **deleted**.
- `Playwright__Service.browser_{navigate,click,fill,screenshot,get_content,get_url}()` added — each composes a throwaway sequence via `Sequence__Runner` and extracts the relevant result.
- `Sequence__Runner.resolve_session` collapses to always-ad-hoc; `close_after` always true.
- `Session__Manager` dependency removed from `Playwright__Service`, `Sequence__Runner`, `Browser__Launcher`.

All green locally (159 unit, 6 L-tier, 0 skipped when mitmproxy + chromium are available).

## What was deferred

- **Warm-browser pool.** v0.1.13's "fresh Chromium per call" is retained (200–400 ms cold-start per request). If latency becomes a problem, a per-container pool can be added behind `Playwright__Service` without breaking the stateless wire contract. Not needed yet — SG/Send tolerates sub-2 s end-to-end.
- **Batch one-shot.** No `/browser/batch` that takes `[navigate, click, screenshot]` in one call. Callers who need that use `/sequence/execute`.
- **Capabilities flag `supports_per_session_proxy`.** Moot now that per-session is gone; replaced by `supports_proxy_auth_basic` on `/health/capabilities` (already shipped).

## Good failures

- **osbot-fast-api Session_Id auto-gen surfaced through the 404 bug (v0.1.23).** That incident made it clear the stateful API's "session_id optional" contract was a footgun at the wire layer. Without that surfacing, the wholesale removal in v0.1.24 would have been a harder sell — it was the concrete symptom that proved the abstract point.
- **`/quick/*` existed because `/browser/*` was stateful.** The duplication was a smell. Collapsing the surfaces made the smell go away — one `/browser/navigate` endpoint rather than `/quick/html` + `/session/create`+`/browser/navigate` doing overlapping work.
- **No mocks in test suite.** Because every test ran real in-memory Playwright, deleting the stateful code path caused precisely the tests that exercised it to fail — no silent dead code. Good failures from good tests.

## Bad failures

- **No CI integration test for the new one-shot surface yet.** Same gap QA flagged for Bug #1 in v0.1.23: local-only tests protect my laptop, not CI. Filed as a follow-up for QA + DevOps in the v0.1.24 QA brief.
- **No deploy-via-pytest coverage of the new routes.** The `tests/deploy/` harness still references `/session/create` and `/browser/navigate` with the old stateful payload. Updated in this slice, but a real Lambda smoke test requires user-triggered merge → dev → deploy (CLAUDE.md rule 29).

## Migration path

Callers on v0.1.23 and older:

```text
v0.1.23                                    →  v0.1.24
─────────────────────────────────────────     ─────────────────────────────────────────
POST /quick/html    {url, ...}             →  POST /browser/get-content {url, ...}
POST /quick/screenshot {url, ...}          →  POST /browser/screenshot  {url, ...}

POST /session/create {browser_config}      →  (no longer needed — pass browser_config
 → POST /browser/navigate {session_id,...}      directly on the one-shot call)
 → DELETE /session/close/{id}              →  POST /browser/navigate {url, browser_config}

POST /sequence/execute {session_id, ...}   →  POST /sequence/execute {browser_config, ...}
POST /sequence/execute {close_session_     →  (close_session_after removed — always true;
                        after: false}          stateless by design)
```

All new one-shot bodies accept `browser_config.proxy` (including `Schema__Proxy__Auth__Basic`), so the Bug #1 proxy-auth fix is available on every endpoint, not just `/sequence/execute`.
