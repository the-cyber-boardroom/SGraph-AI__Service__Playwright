## Rules for any new section

1. **Service class first.** Write `{Section}__Service` extending `Type_Safe` before either CLI or routes exist. No `Console`, no `typer`, no `sys.exit` inside it.
2. **Schemas are pure data.** One file per `Schema__*` under `{section}/schemas/`. No methods.
3. **AWS boundary in one file.** `{Section}__AWS__Client` is the only place that calls boto3 (narrow documented exception per CLAUDE.md). All other classes go through it.
4. **CLI is a wrapper.** Every typer command body fits in ~5 lines: build inputs, call service, format with Rich. No business logic.
5. **FastAPI route is a wrapper.** Every route body fits in ~3 lines: call service, return `.json()`. No business logic.
6. **Routes ship in the same PR as commands.** No "API will follow" — that's how `sp el` ended up CLI-only.
7. **Result schemas, not dicts.** `Image__Build__Service.build()` returns `Schema__Image__Build__Result`, not `{'status': 'ok', ...}`.

## Image build — the consolidation

The two existing builders duplicate ~70% of their code. Target:

```
sgraph_ai_service_playwright__cli/
  image/
    service/
      Image__Build__Service.py            ← orchestrator (tempdir, copytree, Docker SDK, push)
      Image__Build__Context.py            ← per-image build-context resolution (path, dockerfile, ignore)
    schemas/
      Schema__Image__Build__Request.py    ← image_name, tags, build_args, registry
      Schema__Image__Build__Result.py     ← image_id, tags, push_result, duration_ms
      Schema__Image__Push__Result.py
    enums/
      Enum__Image__Target.py              ← PLAYWRIGHT, SP_CLI, AGENT_MITMPROXY, NVM_BROWSER
```

`Build__Docker__SGraph_AI__Service__Playwright` and `Docker__SP__CLI` both reduce to thin wrappers that pre-populate a `Schema__Image__Build__Request` and call `Image__Build__Service.build(...)`. The Type_Safe result lets routes return `.json()` cleanly.

## EC2 lifecycle — finishing the duality

`scripts/provision_ec2.py` still hosts a lot of the EC2 logic that should live in `Ec2__Service`. Migration:

| Step | What | Risk |
|---|---|---|
| 1 | Move userdata templating (`build_user_data_*`) from script into `Ec2__Service` | Low — pure refactor. |
| 2 | Move tag/SG/IAM helpers into `Ec2__AWS__Client` | Low — already isolated logic. |
| 3 | Reduce `provision_ec2.py` to typer commands that call `Ec2__Service` | Low — mechanical. |
| 4 | Add `Routes__Ec2__Playwright` coverage for `health`, `wait`, `connect`-equivalents | Medium — `connect` is interactive SSM; API equivalent is "return the SSM session URL", not "open a shell". |

## What does NOT change

- typer stays as the CLI framework.
- Rich stays for human output.
- `Fast_API__SP__CLI` and `Fast_API__Playwright__Service` remain separate apps (they serve different audiences).
- The Playwright service's own routes (`Routes__Browser`, `Routes__Session`, etc.) are unaffected.
- `__init__.py` stays empty in every package.
