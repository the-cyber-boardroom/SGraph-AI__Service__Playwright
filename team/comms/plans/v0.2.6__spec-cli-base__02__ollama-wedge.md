## Goal

Take the Ollama spec from "barely-working pilot" to "first spec that fully
satisfies the v0.2.6 CLI contract", and use that migration to deliver:

1. The GPU Deep-Learning AMI as a first-class option.
2. `gpt-oss:20b` as the default model.
3. Optional `--with-claude` boot flag that runs `ollama launch claude`
   under tmux.
4. A small agent-tool surface (`run_python`, `read_file`, `write_file`,
   `http_get`, `http_post`) on top of the existing `host_plane/shell/`
   sidecar.

This is **the** integration test for Phases 1+2. If any of these don't fit
neatly on top of the Builder, the Builder needs revision — not a workaround
in Ollama.

**`Agent__Tool__Allowlist`:** Type_Safe with attributes:

- `cwd_roots: List[Safe_Str__Path]` — paths under which `run_python`,
  `read_file`, `write_file` may operate. Empty list = deny all (fail
  closed).
- `http_hosts: List[Safe_Str__Host]` — hostnames the HTTP tools may
  reach. Empty = deny all.
- `max_runtime_sec: Safe_Int__Hours = Safe_Int__Hours(0)` (or new
  primitive `Safe_Int__Seconds`) — process kill threshold for
  `run_python`.
- `max_payload_bytes: Safe_Int__Bytes` — body size limit for HTTP
  responses captured.

`Agent__Tool__Executor` enforces:

- `read_file/write_file`: `Path(p).resolve()` must be under one of
  `cwd_roots`. Reject symlinks that escape.
- `run_python`: writes the snippet to a temp file inside an approved
  cwd; runs `python3 -m venv`'s python at
  `/home/ec2-user/venvs/agent-tools/bin/python` with `cwd` set;
  enforces `max_runtime_sec` via `subprocess.run(timeout=...)`.
- `http_get/http_post`: parses URL; rejects if host not in
  `http_hosts`. Caps response capture at `max_payload_bytes`.

Every call appends a JSON line to
`/var/log/sg-compute/agent-tools.log` — rotated by logrotate (handled
in `Section__Agent_Tools` host-side; add a `/etc/logrotate.d/sg-compute`
drop-in there).

## Risk Notes

- **R3 reminder:** if AWS renames the DLAMI SSM parameter, the
  `latest_dlami_oss_pytorch_26` call returns empty. The boot fails fast
  because `Section__Ollama` then runs against an unknown AMI's user-data
  layout. Mitigation: Phase 3 lands a deploy-via-pytest test
  (`tests/deploy/test_5__ollama_dlami_resolves.py`) that hits the
  parameter in `eu-west-2` and asserts `ami-` prefix.
- **R4 reminder:** `g4dn.xlarge` (16 GiB VRAM) cannot host
  `gpt-oss:20b`. Default to `g5.xlarge` (24 GiB) and document the
  fallback for `qwen2.5-coder:7b`-class models.
