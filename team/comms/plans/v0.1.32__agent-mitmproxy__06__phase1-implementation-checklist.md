### Step 2 — Addons

Files:
- `agent_mitmproxy/agent_mitmproxy/addons/__init__.py` — empty.
- `agent_mitmproxy/agent_mitmproxy/addons/default_interceptor.py`:
  - class `Default_Interceptor` with mitmproxy hooks `request` + `response`.
  - module-level `addons = [Default_Interceptor()]` so mitmweb `-s` picks it up.
  - Request hook: generate 12-char id, stash in `flow.metadata['x-agent-mitmproxy-id']` + set request header. Stamp `X-Agent-Mitmproxy-Request-Ts`.
  - Response hook: echo the id header on the response; compute elapsed ms from `flow.request.timestamp_start`; set `X-Agent-Mitmproxy-Version`.
- `agent_mitmproxy/agent_mitmproxy/addons/audit_log_addon.py`:
  - class `Audit_Log` with `response` hook only.
  - Builds the JSON dict, `sys.stdout.write(json.dumps(...) + '\n')`, `sys.stdout.flush()`.
- `agent_mitmproxy/agent_mitmproxy/addons/addon_registry.py`:
  - Module-level `addons = [Default_Interceptor(), Audit_Log()]`.

Tests:
- `tests/unit/agent_mitmproxy/addons/test_default_interceptor.py`:
  - Exercise `request()` + `response()` on a `mitmproxy.test.tflow.tflow()` and assert header presence.
- `tests/unit/agent_mitmproxy/addons/test_audit_log_addon.py`:
  - Capture `sys.stdout`, call `response()`, assert one JSON line with expected keys.

### Step 4 — CA + Config routes

Files:
- `agent_mitmproxy/agent_mitmproxy/fast_api/routes/Routes__CA.py`.
- `agent_mitmproxy/agent_mitmproxy/fast_api/routes/Routes__Config.py`.
- `agent_mitmproxy/agent_mitmproxy/schemas/ca/Schema__CA__Cert__Info.py`.
- `agent_mitmproxy/agent_mitmproxy/schemas/config/Schema__Interceptor__Source.py`.

`Routes__CA` logic:
- `/ca/cert`: opens `PATH__CA_CERT_PEM` (env-overridable); returns `Response(content=bytes, media_type='application/x-pem-file')`. 503 if not present.
- `/ca/info`: parses the PEM for notBefore/notAfter/SHA256 fingerprint (use `cryptography` — it's a mitmproxy dep already).

`Routes__Config` logic:
- `/config/interceptor`: reads `PATH__CURRENT_INTERCEPTOR`, returns `{path, size_bytes, source}`.

Tests:
- `test_Routes__CA.py` — write a dummy PEM to a temp path via env-override, assert `/ca/cert` returns it.
- `test_Routes__Config.py` — temp file, env-override, assert `/config/interceptor` matches.

### Step 5 — Web proxy route

Files:
- `agent_mitmproxy/agent_mitmproxy/fast_api/routes/Routes__Web.py`.

Logic:
- `GET /web/{path:path}` — strip `X-API-Key` header, use `httpx.AsyncClient` to `GET http://127.0.0.1:8081/{path}`, stream body back.
- Authenticated via the same X-API-Key middleware.
- No WebSocket support (documented as Phase 2).

Tests:
- `test_Routes__Web.py` — mock upstream via `httpx.MockTransport`, assert body passthrough and header strip.
  - (Exception: "no mocks" rule is about service dependencies and DB; mocking `httpx` at transport level for a pure-pass-through route is acceptable — equivalent to a fake socket. If contested, spin up a lightweight `aiohttp` server in the test instead.)

docker run --rm -p 8080:8080 -p 8000:8000 \
    -e AGENT_MITMPROXY__PROXY_AUTH_USER=agent \
    -e AGENT_MITMPROXY__PROXY_AUTH_PASS=test \
    -e FAST_API__AUTH__API_KEY__NAME=X-API-Key \
