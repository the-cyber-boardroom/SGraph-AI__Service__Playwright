## File-by-file — what goes in each

### `agent_mitmproxy/agent_mitmproxy/__init__.py`

Empty except for the `path` export pattern already used by Playwright:

```python
import os
path = os.path.dirname(os.path.abspath(__file__))
```

Lets `Docker__Agent_Mitmproxy__Base` resolve `docker/images/agent_mitmproxy/dockerfile` without hardcoding.

### `agent_mitmproxy/agent_mitmproxy/version`

Plain text file, content `v0.1.32`. Read by `consts/version.py:version__agent_mitmproxy`.

### `agent_mitmproxy/agent_mitmproxy/consts/env_vars.py`

```python
# ── Agent Mitmproxy env vars ─────────────────────────────────────────────────
ENV_VAR__PROXY_AUTH_USER          = 'AGENT_MITMPROXY__PROXY_AUTH_USER'
ENV_VAR__PROXY_AUTH_PASS          = 'AGENT_MITMPROXY__PROXY_AUTH_PASS'
ENV_VAR__CA_CERT_PATH             = 'AGENT_MITMPROXY__CA_CERT_PATH'          # default /root/.mitmproxy/mitmproxy-ca-cert.pem
ENV_VAR__INTERCEPTOR_PATH         = 'AGENT_MITMPROXY__INTERCEPTOR_PATH'      # default /app/current_interceptor.py
ENV_VAR__MITMWEB_HOST             = 'AGENT_MITMPROXY__MITMWEB_HOST'          # default 127.0.0.1
ENV_VAR__MITMWEB_PORT             = 'AGENT_MITMPROXY__MITMWEB_PORT'          # default 8081
ENV_VAR__ADMIN_API_PORT           = 'AGENT_MITMPROXY__ADMIN_API_PORT'        # default 8000

# Reuses the Playwright / Serverless convention — same middleware class
ENV_VAR__API_KEY_NAME             = 'FAST_API__AUTH__API_KEY__NAME'
ENV_VAR__API_KEY_VALUE            = 'FAST_API__AUTH__API_KEY__VALUE'
```

### `agent_mitmproxy/agent_mitmproxy/addons/default_interceptor.py`

mitmproxy addon that injects:

- On request:
  - `X-Agent-Mitmproxy-Request-Id` = random 12-char id, stamped on flow.metadata for the response phase
  - `X-Agent-Mitmproxy-Request-Ts` = ISO-8601 timestamp
- On response:
  - `X-Agent-Mitmproxy-Request-Id` = same as above (echo so client can correlate)
  - `X-Agent-Mitmproxy-Elapsed-Ms` = request-start → response time
  - `X-Agent-Mitmproxy-Version` = `version__agent_mitmproxy`

No external I/O. Pure header manipulation. ~30 lines.

### `agent_mitmproxy/agent_mitmproxy/addons/audit_log_addon.py`

mitmproxy addon, `response` hook. Emits one JSON line to stdout per flow:

```json
{"ts":"...","flow_id":"...","method":"GET","scheme":"https","host":"api.example.com",
 "path":"/v1/foo","status":200,"bytes_request":123,"bytes_response":456,
 "elapsed_ms":78,"client_addr":"1.2.3.4","proxy_user":"agent-user"}
```

Uses `sys.stdout.write(json.dumps(...) + '\n')` + `flush()`. No buffering. Fluent-bit-friendly.

### `agent_mitmproxy/agent_mitmproxy/addons/addon_registry.py`

```python
from agent_mitmproxy.addons.default_interceptor import addons as interceptor_addons
from agent_mitmproxy.addons.audit_log_addon     import addons as audit_addons

addons = [*interceptor_addons, *audit_addons]    # mitmweb -s <this file>
```

### `agent_mitmproxy/agent_mitmproxy/fast_api/Fast_API__Agent_Mitmproxy.py`

Subclasses `Serverless__Fast_API` (the same base Playwright uses). Registers the four route classes. No logic — pure composition.

### `agent_mitmproxy/agent_mitmproxy/fast_api/app.py`

```python
from agent_mitmproxy.fast_api.Fast_API__Agent_Mitmproxy import Fast_API__Agent_Mitmproxy
app = Fast_API__Agent_Mitmproxy().setup().app()
```

uvicorn entrypoint. `CMD ["uvicorn", "agent_mitmproxy.fast_api.app:app", "--host", "0.0.0.0", "--port", "8000"]`.

### `agent_mitmproxy/agent_mitmproxy/fast_api/routes/Routes__Health.py`

Mirrors `sgraph_ai_service_playwright/fast_api/routes/Routes__Health.py` exactly — same three endpoints:

- `GET /health/info` — returns service name, version, deployment target
- `GET /health/status` — `Schema__Health__Check` with checks for: mitmweb_running, ca_cert_exists
- `GET /health/capabilities` — stub for Phase 1, populated in Phase 2

### `agent_mitmproxy/agent_mitmproxy/fast_api/routes/Routes__CA.py`

- `GET /ca/cert` — reads `PATH__CA_CERT_PEM` and returns the PEM bytes as `Content-Type: application/x-pem-file`.
- `GET /ca/info` — returns fingerprint, validity dates, path on disk (schema-wrapped).

Returns 503 if the cert doesn't exist yet (mitmweb writes it on first start; if someone hits this before that, give a useful error).

### `agent_mitmproxy/agent_mitmproxy/fast_api/routes/Routes__Config.py`

- `GET /config/interceptor` — returns the current interceptor script source as `Schema__Interceptor__Source{path, size_bytes, source}`.

No PUT in Phase 1. Phase 2 will add upload; requires persistence across container restarts which deserves its own design.

### `agent_mitmproxy/agent_mitmproxy/fast_api/routes/Routes__Web.py`

- `GET /web/**` — reverse-proxies to `http://127.0.0.1:8081/**`.

Uses `httpx.AsyncClient` to fetch + stream. Strips the `X-API-Key` header from the upstream request (it's for our middleware, not mitmweb). WebSocket passthrough is NOT required for Phase 1 — we're fine with the HTTP-only views.

If mitmweb's UI needs WebSockets for live flow streaming, we'll either:
(a) call it out as "use `curl -x` + read `docker logs` for live tail" in Phase 1 docs,
(b) add a separate `/web-ws` upgrade route in Phase 2.

### `agent_mitmproxy/docker/Docker__Agent_Mitmproxy__Base.py`

Same shape as `Docker__SGraph_AI__Service__Playwright__Base`:

```python
IMAGE_NAME = 'agent_mitmproxy'
LOCAL_PORT = 8888
PROXY_PORT = 8080
ADMIN_PORT = 8000

class Docker__Agent_Mitmproxy__Base(Type_Safe):
    image_name       : str = IMAGE_NAME
    path_images      : str = ''
    create_image_ecr : object = None

    def setup(self):
        self.path_images      = path_combine(agent_mitmproxy.path, 'docker/images')
        self.create_image_ecr = Create_Image_ECR(image_name  = self.image_name,
                                                 path_images = self.path_images)
        return self

    def image_uri(self):
        return f'{self.create_image_ecr.image_repository()}:latest'

    def path_docker(self):
        return path_combine(agent_mitmproxy.path, f'docker/images/{self.image_name}')

    def path_dockerfile(self):
        return f'{self.path_docker()}/dockerfile'
```

### `agent_mitmproxy/docker/images/agent_mitmproxy/entrypoint.sh`

```bash
#!/bin/sh
set -eu

# Seed the live interceptor file from the baked default.
if [ ! -f /app/current_interceptor.py ]; then
    cp /app/agent_mitmproxy/addons/default_interceptor.py /app/current_interceptor.py
fi

exec supervisord -c /app/supervisord.conf
```

### `agent_mitmproxy/docker/images/agent_mitmproxy/supervisord.conf`

Two programs:

```ini
[supervisord]
nodaemon=true
user=root
logfile=/dev/null
logfile_maxbytes=0
pidfile=/tmp/supervisord.pid

        --proxyauth "%(ENV_AGENT_MITMPROXY__PROXY_AUTH_USER)s:%(ENV_AGENT_MITMPROXY__PROXY_AUTH_PASS)s"

        --web-port 8081
        --listen-port 8080

        --set confdir=/root/.mitmproxy

stdout_logfile_maxbytes=0

stderr_logfile_maxbytes=0

[program:admin_api]
command=uvicorn agent_mitmproxy.fast_api.app:app --host 0.0.0.0 --port 8000

stdout_logfile=/dev/stdout
stdout_logfile_maxbytes=0
```

### `scripts/provision_mitmproxy_ec2.py`

~90% copy of `scripts/provision_ec2.py` with these differences:

| Constant | Playwright value | Agent-mitmproxy value |
|---|---|---|
| `EC2__INSTANCE_TYPE` | `t3.large` | `t3.small` (mitmproxy is light — 2GB RAM enough) |
| `EC2__APP_PORT` | `8000` | `8000` (admin API) — plus explicit `PROXY_PORT = 8080` |
| `SG__NAME` | `playwright-ec2-spike` | `agent-mitmproxy-ec2-spike` (no `sg-` prefix) |
| `IAM__ROLE_NAME` | `sg-playwright-ec2-spike` | `agent-mitmproxy-ec2-spike` |
| `TAG__NAME` | `sg-playwright-ec2-spike` | `agent-mitmproxy-ec2-spike` |
| `USER_DATA_TEMPLATE` | Playwright flags | mitmproxy flags (PROXY_AUTH_USER/PASS instead of SG_PLAYWRIGHT__*) |
| SG ingress | `:8000` only | `:8000` + `:8080` |

## Design rules applied

Every file obeys CLAUDE.md:
- **§17 Type_Safe** everywhere.
- **§19 one class per file**: every `Schema__*` lives in its own file. Registries are the one exception (`addon_registry.py` is module-level composition, not a class).
- **§20 empty `__init__.py`**: nothing re-exported.
- **§7 `═══` 80-char headers**: every file.
- **§14 responsibility boundaries**: routes have no logic — they delegate to services (Phase 2 adds `services/` when we have persistence).
- **§8 no docstrings, inline comments only**.
- **Testing §1 no mocks**: tests exercise the real FastAPI via `TestClient`, spin mitmproxy with real addons in-process for the addon tests.
