## Step 1 — Write the contract document

Create `library/docs/specs/v0.2.6__spec-cli-contract.md`. Sections:

### §1. Canonical command set — every spec MUST expose these

| Verb | Signature | Behaviour |
|------|-----------|-----------|
| `list` | `--region` | List all stacks for spec in region. |
| `info` | `[name?]`, `--region` | Show one stack. `name` resolved by the §3 rules. |
| `create` | `[name?]`, `--region`, `--instance-type`, `--max-hours=1`, `--ami`, `--caller-ip`, `--wait` | Provision. `name` auto-generated when omitted. `--wait` polls health until ready. **`--max-hours=1` for every spec — D2.** |
| `delete` | `[name?]`, `--region`, `--yes`, `-y` | Terminate. Confirms unless `--yes`. **D3 — confirms even after auto-pick.** |
| `wait` | `[name?]`, `--region`, `--timeout=600`, `--poll=10` | Block until healthy. Same probe as `health --timeout 600`. |
| `health` | `[name?]`, `--region`, `--timeout=0` | Instant probe by default; `--timeout > 0` makes it a `wait`. **D7 — same code path.** |
| `connect` | `[name?]`, `--region` | `os.execvp('aws', ['ssm', 'start-session', ...])`. |
| `exec` | `[name?]`, `<command>`, `--region`, `--timeout=60`, `--cwd=''` | Run a shell command on the host. **D1 — sidecar first, SSM SendCommand fallback.** Returns stdout/stderr/exit_code. |

### §2. Optional verbs

Each spec **may** add extra verbs (e.g. `ollama models`, `firefox set-interceptor`).
Add them with `app.command()` on the Builder's returned `typer.Typer`.

### §3. `name` resolution rule

When a verb takes optional `name` and none is supplied:

1. List all stacks in region.
2. **0 stacks** → print "No {spec} stacks in {region}. Run: sg-compute spec
   {spec_id} create" to stderr in yellow; exit 1.
3. **1 stack** → print "One stack found — using {name}" to stdout in dim;
   use that name.
4. **>1 stacks** → number them, prompt "Pick a stack number"; exit 1 on
   invalid.

(Implementation: `Spec__CLI__Resolver` — see Step 4.)

### §4. Error handling

Every command is wrapped by the `@spec_cli_errors` decorator from
`Spec__CLI__Errors`. It:

- Catches `KeyboardInterrupt` and re-raises (no traceback).
- Catches `typer.Exit` and re-raises.
- Catches AWS-credential exceptions (`'credential' in str(e).lower()` OR
  `type(e).__name__` ends with `'NoCredentialsError'`) → prints
  "AWS credentials not configured: {e}" to stderr; exit 1.
- Catches everything else → red `✗ {ExceptionType}: {msg}`; with `--debug`
  prints the full traceback; exit 2.

### §5. `Service` contract — what every `{Pascal}__Service` must offer

The minimum surface the builder calls. Inherit defaults from
`Spec__Service__Base` (Step 9) — most specs only need to override the four
spec-specific bits.

| Method | Default impl in `Spec__Service__Base`? | Spec must override? |
|--------|----------------------------------------|---------------------|
| `setup()` | n/a | yes |
| `create_stack(request)` | n/a | yes |
| `list_stacks(region)` | n/a | yes |
| `get_stack_info(region, name)` | n/a | yes |
| `delete_stack(region, name)` | n/a | yes |
| `health(region, name, timeout_sec=0, poll_sec=10) -> Schema__CLI__Health__Probe` | yes — uses `Health__Poller` against `health_path` from `cli_spec()` | only when probe URL needs auth/path tricks |
| `exec(region, name, command, timeout_sec=60, cwd='') -> Schema__CLI__Exec__Result` | yes — sidecar `/host/shell/execute` with SSM fallback | rarely |
| `connect_target(region, name) -> Safe_Str__Instance__Id` | yes — pulls `instance_id` from `get_stack_info` | rarely |
| `cli_spec() -> Schema__Spec__CLI__Spec` | n/a | yes — declares `health_path`, `health_port`, etc. |

### §6. Reference implementation pointers

- `sg_compute/cli/base/Spec__CLI__Builder.py` — factory.
- `sg_compute_specs/ollama/cli/Cli__Ollama.py` (Phase 3) — first consumer,
  3 extra verbs (`models / pull / claude`).

### §7. Migration mode

Specs migrate one PR at a time. The legacy CLI under
`sgraph_ai_service_playwright__cli/{spec}/cli/` is removed in the same PR
that wires the spec into the builder.

## Step 4 — `Spec__CLI__Resolver`

`sg_compute/cli/base/Spec__CLI__Resolver.py`:

```python
# ═══════════════════════════════════════════════════════════════════════════════
# SG/Compute — Spec__CLI__Resolver
# Implements the "auto-pick / prompt / error" rule for optional name args.
# ═══════════════════════════════════════════════════════════════════════════════

from typing import List, Optional, Protocol

import typer
from rich.console import Console

from osbot_utils.type_safe.Type_Safe import Type_Safe


class _ListsStacks(Protocol):
    def list_stacks(self, region: str): ...


class Spec__CLI__Resolver(Type_Safe):

    def resolve(self, service: _ListsStacks, provided: Optional[str], region: str,
                spec_id: str) -> str:
        if provided:
            return provided
        listing      = service.list_stacks(region)
        names        = [str(s.stack_name) for s in listing.stacks if str(s.stack_name)]
        region_label = str(getattr(listing, 'region', '') or region)

        if len(names) == 0:
            Console(highlight=False, stderr=True).print(
                f'\n  [yellow]No {spec_id} stacks in {region_label}.[/]'
                f'  Run: [bold]sg-compute spec {spec_id} create[/]\n')
            raise typer.Exit(1)

        if len(names) == 1:
            Console(highlight=False).print(
                f'\n  [dim]One stack found — using [bold]{names[0]}[/][/]')
            return names[0]

        c = Console(highlight=False)
        c.print(f'\n  [bold]Multiple {spec_id} stacks in {region_label}:[/]')
        for idx, name in enumerate(names, start=1):
            c.print(f'    {idx}. {name}')
        raw = typer.prompt('\n  Pick a stack number', type=int)
        try:
            choice = int(raw)
        except (TypeError, ValueError):
            choice = -1
        if choice < 1 or choice > len(names):
            Console(highlight=False, stderr=True).print(f'\n  [red]Invalid selection: {raw}[/]\n')
            raise typer.Exit(1)
        return names[choice - 1]
```

## Step 5 — `Spec__CLI__Errors`

`sg_compute/cli/base/Spec__CLI__Errors.py`:

```python
# ═══════════════════════════════════════════════════════════════════════════════
# SG/Compute — Spec__CLI__Errors
# Shared @spec_cli_errors decorator.  Caller toggles --debug for tracebacks.
# Module-level _DEBUG flag is set by the @app.callback() that the Builder
# attaches; tests can flip it directly.
# ═══════════════════════════════════════════════════════════════════════════════

import functools
import traceback

import typer
from rich.console import Console


_DEBUG = False


def set_debug(value: bool) -> None:
    global _DEBUG
    _DEBUG = bool(value)


def spec_cli_errors(fn):
    @functools.wraps(fn)
    def wrapped(*args, **kwargs):
        try:
            return fn(*args, **kwargs)
        except (typer.Exit, KeyboardInterrupt):
            raise
        except Exception as exc:
            c   = Console(highlight=False, stderr=True)
            msg = str(exc)
            if 'credential' in msg.lower() or type(exc).__name__.endswith('NoCredentialsError'):
                c.print(f'\n  [red]✗[/]  AWS credentials not configured: {exc}\n')
                raise typer.Exit(1)
            c.print(f'\n  [red]✗[/]  [bold]{type(exc).__name__}[/]: {exc}')
            if _DEBUG:
                c.print('\n[dim]── traceback ────────────────────────────────────[/]')
                c.print(traceback.format_exc(), end='')
            else:
                c.print('     [dim]› Re-run with --debug to see the full traceback.[/]')
            c.print()
            raise typer.Exit(2)
    return wrapped
```

**Tests:** all four classes — `typer.Exit` re-raised, AWS-creds branch
exits 1, generic exception exits 2 with traceback when `_DEBUG=True`,
without when False. Use `capsys` for stderr capture; `set_debug(True)` then
`set_debug(False)` between cases.

## Step 8 — `Spec__CLI__Builder`

`sg_compute/cli/base/Spec__CLI__Builder.py`. Returns a `typer.Typer` with
all eight standard verbs registered.

**Critical implementation notes:**

- The Builder is **not** a `Type_Safe` — typer's command decorator does
  reflection on argument annotations that conflicts with Type_Safe's
  metaclass machinery. Risk R1 in the overview. Plain class is fine.
- Each command body is a closure over `(cli_spec, console)` to avoid
  globals.
- The `@spec_cli_errors` decorator wraps every command body.
- The root callback accepts `--debug`/`-D` and pipes to `set_debug()`.
- `extra_create_options` is a `List[Tuple[str, type, Any, str]]`:
  `(name, py_type, default, help)`.

```python
# ═══════════════════════════════════════════════════════════════════════════════
# SG/Compute — Spec__CLI__Builder
# Factory that produces a fully-wired typer.Typer for one spec.  Each
# Cli__{Pascal}.py file is a 5-line call into this builder.
# ═══════════════════════════════════════════════════════════════════════════════

import os
from typing                                                                 import Any, List, Optional, Tuple

import typer
from rich.console                                                           import Console

from sg_compute.cli.base.Schema__Spec__CLI__Spec                           import Schema__Spec__CLI__Spec
from sg_compute.cli.base.Spec__CLI__Defaults                               import (DEFAULT_EXEC_TIMEOUT  ,
                                                                                    DEFAULT_MAX_HOURS     ,
                                                                                    DEFAULT_POLL_SEC      ,
                                                                                    DEFAULT_REGION        ,
                                                                                    DEFAULT_TIMEOUT_SEC   )
from sg_compute.cli.base.Spec__CLI__Errors                                 import set_debug, spec_cli_errors
from sg_compute.cli.base.Spec__CLI__Renderers__Base                        import (render_exec_result   ,
                                                                                    render_health_probe  ,
                                                                                    render_info          ,
                                                                                    render_list          )
from sg_compute.cli.base.Spec__CLI__Resolver                               import Spec__CLI__Resolver


class Spec__CLI__Builder:

    def __init__(self, cli_spec: Schema__Spec__CLI__Spec,
                       extra_create_options: Optional[List[Tuple[str, type, Any, str]]] = None):
        self.cli_spec             = cli_spec
        self.extra_create_options = extra_create_options or []
        self.resolver             = Spec__CLI__Resolver()

    def build(self) -> typer.Typer:
        app = typer.Typer(no_args_is_help=True,
                          help=f'Manage {self.cli_spec.display_name} stacks.',
                          add_completion=False)

        @app.callback()
        def _root(debug: bool = typer.Option(False, '--debug', '-D',
                                             help='Show full Python traceback on errors.')):
            set_debug(debug)

        self._register_list   (app)
        self._register_info   (app)
        self._register_create (app)
        self._register_wait   (app)
        self._register_health (app)
        self._register_connect(app)
        self._register_exec   (app)
        self._register_delete (app)
        return app

    def _service(self):
        return self.cli_spec.service_factory()

    # ── verbs ─────────────────────────────────────────────────────────────────
    # (one method per verb; bodies follow the patterns from
    # sg_compute_specs/ollama/cli/__init__.py.  Wrap each command with
    # @spec_cli_errors.  Use self.resolver.resolve(svc, name, region, spec_id)
    # for any verb that takes optional name.)

    def _register_list(self, app):
        @app.command(name='list')
        @spec_cli_errors
        def list_stacks(region: str = typer.Option(DEFAULT_REGION, '--region', '-r')):
            listing = self._service().list_stacks(region)
            render_list(listing, Console(highlight=False, width=200))

    def _register_info(self, app):
        @app.command()
        @spec_cli_errors
        def info(name  : Optional[str] = typer.Argument(None,
                          help='Stack name; auto-selected when only one exists.'),
                 region: str           = typer.Option(DEFAULT_REGION, '--region', '-r')):
            svc  = self._service()
            name = self.resolver.resolve(svc, name, region, self.cli_spec.spec_id)
            data = svc.get_stack_info(region, name)
            if data is None:
                Console(highlight=False, stderr=True).print(
                    f'  [red]✗  No {self.cli_spec.spec_id} stack matched {name!r}[/]')
                raise typer.Exit(1)
            render_info(data, Console(highlight=False, width=200))

    def _register_create(self, app):
        # Must dynamically build the function signature because
        # extra_create_options vary per spec.  Use typer.Option in **kwargs
        # — see the implementation note below.
        ...                                                                 # detailed code in Step 8b

    # ... (same pattern for wait/health/connect/exec/delete)
```

### Step 8b — `_register_create` dynamic signature

typer reads parameter annotations to build the CLI. Since
`extra_create_options` is variable per spec, we **construct a Python
function with the right signature dynamically** using
`functools.partial` + `inspect.Signature.replace`.

The straightforward pattern:

```python
def _register_create(self, app):
    cli_spec = self.cli_spec
    resolver = self.resolver
    service_factory = self._service

    base_params = [
        ('name',          Optional[str], typer.Argument(None, help='Stack name; auto-generated if omitted.')),
        ('region',        str,           typer.Option(DEFAULT_REGION, '--region', '-r')),
        ('instance_type', str,           typer.Option(cli_spec.default_instance_type, '--instance-type', '-t')),
        ('max_hours',     int,           typer.Option(DEFAULT_MAX_HOURS, '--max-hours')),
        ('ami',           str,           typer.Option('', '--ami',       help='AMI ID; resolved from spec helper if blank.')),
        ('caller_ip',     str,           typer.Option('', '--caller-ip', help='Source IP for SG; auto-detected if blank.')),
        ('wait',          bool,          typer.Option(False, '--wait',   help='Block until healthy.')),
    ]
    extra_params = [
        (n, t, typer.Option(d, f'--{n.replace("_", "-")}', help=h))
        for (n, t, d, h) in self.extra_create_options
    ]
    all_params = base_params + extra_params

    def create_impl(**kwargs):
        svc       = service_factory()
        request   = cli_spec.create_request_cls()
        request.region.__init__(kwargs['region']) if hasattr(request, 'region') else None
        request.stack_name.__init__(kwargs['name'] or svc.name_gen.generate())
        request.instance_type     = kwargs['instance_type']
        request.max_hours         = kwargs['max_hours']
        request.from_ami          = kwargs['ami']
        request.caller_ip         = kwargs['caller_ip']
        if cli_spec.extra_create_field_setters:
            cli_spec.extra_create_field_setters(request, **{
                k: kwargs[k] for (k, _, _, _) in self.extra_create_options
            })
        resp = svc.create_stack(request)
        from sg_compute.cli.base.Spec__CLI__Renderers__Base import render_create
        render_create(resp, Console(highlight=False, width=200))
        if kwargs['wait']:
            stack_name = str(resp.stack_info.stack_name) if hasattr(resp, 'stack_info') else str(resp.stack_name)
            instance_id = str(resp.stack_info.instance_id) if hasattr(resp, 'stack_info') else str(resp.instance_id)
            self._wait_healthy(svc, kwargs['region'], stack_name, instance_id)

    # Build and decorate
    decorated = self._build_typed_function(create_impl, all_params, name='create')
    app.command()(spec_cli_errors(decorated))
```

`_build_typed_function` uses `inspect.Signature` to graft the param list
onto a wrapper. This is the only piece of metaprogramming in the builder
— pin a clear test for it.

## Step 9 — `Spec__Service__Base`

`sg_compute/core/spec/Spec__Service__Base.py`. Optional base class that
specs may extend to inherit `health` / `exec` / `connect_target` defaults.
Existing `Docker__Service` etc. are left unchanged in this phase — Phase 4

```python
# ═══════════════════════════════════════════════════════════════════════════════
# SG/Compute — Spec__Service__Base
# Optional base class that gives every spec free implementations of:
#   - health(region, name, timeout_sec=0, poll_sec=10) → Schema__CLI__Health__Probe
#   - exec  (region, name, command, timeout_sec=60, cwd='') → Schema__CLI__Exec__Result
#   - connect_target(region, name) → Safe_Str__Instance__Id
# Sub-classes override only when probe path / sidecar address / etc. differ.
# ═══════════════════════════════════════════════════════════════════════════════

from osbot_utils.type_safe.Type_Safe import Type_Safe

# (full implementation: see test list below for the exact behaviours required)


class Spec__Service__Base(Type_Safe):

    def cli_spec(self):                                                       # subclass MUST override
        raise NotImplementedError

    def health(self, region, name, timeout_sec=0, poll_sec=10):               # default impl
        ...

    def exec(self, region, name, command, timeout_sec=60, cwd=''):
        ...                                                                   # sidecar→SSM fallback

    def connect_target(self, region, name):
        info = self.get_stack_info(region, name)
        if info is None:
            raise ValueError(f'no {self.cli_spec().spec_id} stack matched {name!r}')
        return str(info.instance_id)
```
