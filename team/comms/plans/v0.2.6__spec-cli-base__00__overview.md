# v0.2.6 — Spec CLI Base + Ollama Wedge — Overview

**Status:** PLANNED.  Phase 1 ready to start.
**Owner:** Dev (any agent persona).  **Reviewer:** Architect.
**Target service version:** sg_compute v0.2.6 → v0.2.7.

---

## Why

The new `sg_compute_specs/` migration left per-spec CLIs less capable than the
legacy `sgraph_ai_service_playwright__cli/` originals. The new
`sg_compute_specs/docker/cli/Cli__Docker.py` only has `list/info/create/delete`
— no `wait`, `health`, `connect`, or `exec`, no `--wait` on create, and the
"auto-pick when only one stack exists" UX is gone.

Defaults also drifted: `max_hours` is `1` for docker but `4` for ollama. The
duplicated `_resolve_stack_name` and `_err_handler` helpers in legacy and
ollama prove the pattern is shared but copy-pasted.

This plan:

1. **Locks down a single CLI contract every spec must satisfy** (Phase 1).
2. **Builds a `Spec__CLI__Builder` factory** so each per-spec CLI shrinks from
   ~120 LOC to ~10 (Phase 2).
3. **Pilots the refactor on Ollama**, simultaneously upgrading it to the GPU
   DLAMI / `gpt-oss:20b` / optional Claude integration described in the
   2026-05-09 brief (Phase 3).
4. **Rolls out to the remaining 9 specs**, one PR each (Phase 4).
5. **Decommissions the legacy SP CLI** once every spec has migrated (Phase 5).

## Cross-Cutting Rules — apply to every phase

These are restated from `.claude/CLAUDE.md` so the executing agent doesn't
have to keep cross-referencing.

1. **Type_Safe everywhere.** No Pydantic. No `typing.Literal`. Every class
   extends `Type_Safe`.
2. **Zero raw primitives in schema attributes.** Use `Safe_Str__*` /
   `Safe_Int__*` / `Enum__*` / typed collections.  When you need a new
   primitive, add it under `sg_compute/primitives/` (cross-spec) or
   `sg_compute_specs/{spec_id}/primitives/` (per-spec). One class per file.
3. **No boto3 imports outside the documented exception files.** Stay on
   `osbot-aws`. Existing exceptions are listed in `Docker__Launch__Helper`
   and similar — do not add new ones without a decision-log entry.
4. **No mocks, no patches in tests.** Build Type_Safe-friendly fakes (e.g.
   `Fake__Platform`) and inject. The `Docker__Launch__Helper` test added in
   commit 9f24d98 shows the right pattern: extract a pure builder method,
   test that.
5. **One class per file.** File name == class name. Empty `__init__.py`.
   Never re-export through `__init__.py`. Registries (module-level constants)
   are the single exception, in `*_registry.py` files.
6. **80-char `═══` headers** on every file. Inline `#` comments only — no
   docstrings.
7. **No underscore prefix for private methods.**
8. **Branch:** `claude/{description}-{session-id}`. Never push to `dev`. PR
   from feature branch.
9. **Reality doc updated in the same commit** as code that adds, removes, or
   renames a spec / endpoint / class.

When in doubt, copy from the docker spec — it is the canonical reference and
ships the most complete pattern today.

## Risk Register

| # | Risk | Mitigation |
|---|------|------------|
| R1 | Builder factory breaks Type_Safe due to dynamic typer command registration | Use closures + `functools.wraps`; do **not** subclass `Type_Safe` for the builder itself — it's a plain factory class. The *generated* commands are plain functions (typer doesn't need Type_Safe). |
| R2 | Per-spec extras (`ollama models`, `firefox set-interceptor`) break the builder | Builder returns a `typer.Typer`; specs add extras with `app.command(...)`. Hard-coded escape hatch. |
| R3 | DLAMI changes its SSM parameter path | Pin to `oss-nvidia-driver-gpu-pytorch-2.6` per the brief; capture the exact name in `Ollama__AMI__Helper.DLAMI_SSM_PARAM` constant. Add a deploy-via-pytest test that reads the SSM parameter in `eu-west-2` (skipped without AWS creds). |
| R4 | `gpt-oss:20b` doesn't fit on `g4dn.xlarge` (16 GiB VRAM) | Default `--instance-type` to `g5.xlarge` (24 GiB) in Phase 3; document the `--instance-type g4dn.xlarge` fallback for smaller models. |
| R5 | Phase 4 migrations diverge from the contract because each spec has a slightly different service surface | Phase 2 contract test `test_Spec__CLI__Builder__contract.py` runs against every spec id at Phase 4 time. Each spec must pass before its rollout PR merges. |
| R6 | Sidecar exec route opens a remote-shell vector | `Shell__Executor` is allowlist-gated by default. Phase 3 adds `run_python` etc. on top of that allowlist; never widens it. |
