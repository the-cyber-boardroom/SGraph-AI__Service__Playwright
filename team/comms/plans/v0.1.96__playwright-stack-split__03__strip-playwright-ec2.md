## What gets deleted, where

The compose-template constants (`COMPOSE_SVC_*`) currently live inline in `scripts/provision_ec2.py`. They are removed from that file and **moved**, not deleted, into the corresponding sister section's compose template:

- `COMPOSE_SVC_PROMETHEUS`, `COMPOSE_SVC_CADVISOR`, `COMPOSE_SVC_NODE_EXPORTER` → `prom/compose/templates/`
- `COMPOSE_SVC_BROWSER`, `COMPOSE_SVC_BROWSER_PROXY` → `vnc/compose/templates/`
- `COMPOSE_SVC_FLUENT_BIT` → `os/compose/templates/` (it's the log shipper feeding OpenSearch)
- `COMPOSE_SVC_DOCKGE` → **deleted** unless we deliberately revive it under `sp vnc`.

## How the strip lands in code

Convenient discovery: the bundled containers are **already** gated behind conditional flags at `scripts/provision_ec2.py:967–979`:

```python
services += [COMPOSE_SVC_PLAYWRIGHT.format(**fmt)]
services += [COMPOSE_SVC_MITMPROXY.format(**fmt)]
if upstream_url:           # browser VNC + nginx
    services += [COMPOSE_SVC_BROWSER.format(**fmt), COMPOSE_SVC_BROWSER_PROXY.format(**fmt)]
if amp_remote_write_url:   # metrics stack
    services += [COMPOSE_SVC_CADVISOR, COMPOSE_SVC_NODE_EXPORTER, COMPOSE_SVC_PROMETHEUS]
if opensearch_endpoint:    # log shipper
    services += [COMPOSE_SVC_FLUENT_BIT]
services += [COMPOSE_SVC_DOCKGE.format(**fmt)]
```

The strip is therefore a **deletion of these branches** plus the now-orphaned `COMPOSE_SVC_*` constants and the `prometheus_data` volume line at `:981–983`. The `_url`/`_endpoint` parameters they gate disappear from the function signature and from any caller. Net code shape after strip:

```python
services  = ['services:\n']
services += [COMPOSE_SVC_PLAYWRIGHT.format(**fmt)]
services += [COMPOSE_SVC_MITMPROXY.format(**fmt)]
return '\n'.join(services) + '\n' + 'networks:\n  sg-net:\n    driver: bridge\n'
```

## Userdata + IAM impact

- **Drop CloudWatch agent / xray-daemon** if present in the userdata (they were observability extras).
- **Drop ECR-pull permissions** for the moved images (Prometheus, browser, etc.); the Playwright EC2 only pulls 2 images now.
- **Drop the `prometheus_data` volume** from the compose footer (`provision_ec2.py:981–983`); it moves with the prometheus service to `sp prom`. (`grafana_data` / `loki_data` were never declared in code despite the v0.1.31 reality doc mentioning them — nothing to drop.)
- **Drop ports** 9090, 3000, 5001, 8080 from the SG (Prometheus, browser-VNC, Dockge, mitmproxy proxy port — confirmed S2). The mitmproxy admin port 8001 stays open (S1: API-key protected). Port 8000 (Playwright FastAPI) stays.

## AMI implications

- The current "everything baked" AMI grows to ~3 GB. After the strip it should land closer to ~1.2 GB (Playwright base image dominates).
- Bake-AMI flow is unchanged; the saving is automatic.
- Existing AMIs are tagged but not deleted by this plan — they age out naturally.
