# Cross-Role Plans

Plans that span more than one role. A single-role plan belongs in that role's own `reviews/` folder.

## Naming

`v{version}__{plan-name}.md`

## Anatomy

1. **Goal** — one sentence.
2. **Roles involved** — who does what.
3. **Phases / milestones** — with clear exit criteria.
4. **Risks** — with mitigations.
5. **Definition of done** — measurable.
6. **Cross-references** — specs, briefs, reality-doc sections affected.

## Status

Once a plan is finished, annotate the top with `**Status: Delivered (commit X)**` and move the file to `archive/`.
