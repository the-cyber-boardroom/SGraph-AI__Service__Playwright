### One-time edit on spec #1 (docker)

`sg_compute/cli/Cli__Compute__Node.py`:

- Replace the `_DISPATCHERS` table with a generic delegator that calls
  `EC2__Platform.create_node(spec_id=spec_id, base_request=...)`.
- Drop the per-spec `_create_*` helpers.
- Add a `Schema__Node__Create__Request__Base` field for `disk_size_gb`
  so the cross-spec path can also size disks.

```python
@app.command()
def create(spec_id      : str = typer.Argument(..., help='Spec identifier (docker, podman, …)'),
           region       : str = typer.Option('eu-west-2', '--region', '-r'),
           instance_type: str = typer.Option('t3.medium', '--instance-type', '-t'),
           max_hours    : int = typer.Option(1,           '--max-hours'),
           disk_size    : int = typer.Option(0,           '--disk-size',
                                              help='Root volume in GiB; 0 keeps AMI default.'),
           name         : str = typer.Option('',          '--name')):
    from sg_compute.platforms.ec2.EC2__Platform                         import EC2__Platform
    from sg_compute.core.node.schemas.Schema__Node__Create__Request__Base import Schema__Node__Create__Request__Base

    base = Schema__Node__Create__Request__Base(
        spec_id        = spec_id,
        node_name      = name,
        region         = region,
        instance_type  = instance_type,
        max_hours      = max_hours,
        disk_size_gb   = disk_size,
    )
    info = EC2__Platform().setup().create_node(base)
    typer.echo(f'Created : {info.node_id}')
    typer.echo(f'Instance: {info.instance_id}  ({info.instance_type})')
    typer.echo(f'Region  : {info.region}')
    typer.echo(f'State   : {info.state.value}')
```

`Schema__Node__Create__Request__Base` change adds `disk_size_gb:
Safe_Int__Disk__GB` (D8).

`EC2__Platform.create_node` already dispatches by `spec_id` — no change
required there beyond surfacing `disk_size_gb` to the underlying spec
service. Each migrated spec's `create_node` reads `base.disk_size_gb` and
forwards it to its own request schema.

3. `sg-compute spec list` still returns 12 specs (no regression).
4. `sg-compute node create {spec_id}` works (cross-spec path).
5. `git diff --stat` shows ≥ 50 LOC removed (the legacy `__init__.py`
   plus duplication).
6. Reality doc updated.

### CI guard

After spec #1, add a CI job that runs:

```bash
pytest sg_compute__tests/cli/base/test_Spec__CLI__Builder__contract.py -v
```

This test (added in Phase 2) iterates every registered spec_id, builds its
`Cli__{Pascal}.app`, and asserts the 8 standard commands are present.
Stops a spec from drifting back to a hand-rolled CLI silently.

## Rollback Plan

Per-spec PRs in Phase 4 are independent — revert any single PR without
affecting others. Phase 5 is the single point of no return; if anything
under `sgraph_ai_service_playwright__cli/` is still referenced after
Phase 4, abort Phase 5 and chase the dangling import first.
