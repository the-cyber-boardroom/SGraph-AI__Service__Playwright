## Context

The revised EC2 architecture brief graduates the `agent_mitmproxy` sidecar from a
proxy-auth workaround into a permanent HTTP gateway for all EC2 deployments. The
key shift: **proxy is a boot-time infrastructure concern, not an API parameter**.

| Deployment | Proxy | Who decides |
|---|---|---|
| Lambda | None â€” Playwright goes direct to internet | Environment (no `SG_PLAYWRIGHT__PROXY_URL`) |
| EC2 / container | Always via `agent_mitmproxy` sidecar `:8080` | Environment (`SG_PLAYWRIGHT__PROXY_URL=http://agent-mitmproxy:8080`) |
| Laptop / CI | None by default; optional if local mitmproxy running | Environment |

The sidecar itself decides whether to forward via an authenticated upstream proxy
at **its own boot time** â€” controlled by `AGENT_MITMPROXY__UPSTREAM_*` env vars.

| `agent_mitmproxy/version` | Bump to `v0.1.33` |

### Decisions

1. **Wrapper script pattern.** `entrypoint.sh` writes `/tmp/run_mitmweb.sh` with
   the full `exec mitmweb ...` command (flags determined by env vars). `supervisord`
   calls `command=/bin/sh /tmp/run_mitmweb.sh`. This avoids generating a dynamic
   `supervisord.conf` and keeps the logic in shell where it's easy to read.

2. **`--proxyauth` is optional.** Only added when both
   `AGENT_MITMPROXY__PROXY_AUTH_USER` and `AGENT_MITMPROXY__PROXY_AUTH_PASS` are
   set. The paired-Playwright deployment leaves them unset â€” security comes from
   Docker network isolation, not HTTP auth on `:8080`.

3. **Upstream mode.** When `AGENT_MITMPROXY__UPSTREAM_URL` is set, mitmweb runs
   with `--mode upstream:<url>`. When both `UPSTREAM_USER` and `UPSTREAM_PASS` are
   also set, `--set upstream_auth=<user>:<pass>` is added. The auth is preemptive
   (mitmproxy's built-in `upstream_auth` option), which is exactly what Phase 1.11
   validated.

4. **`proxy_mode` on `/health/info`.** Surfaces the boot-time mode (`direct` or
   `upstream`) so operators can confirm the running container matches expectations
   without shelling in.

### Env var contract (after P1)

```
# Downstream proxy auth â€” optional; omit when paired with Playwright
AGENT_MITMPROXY__PROXY_AUTH_USER   = <user>   # if unset, --proxyauth is omitted
AGENT_MITMPROXY__PROXY_AUTH_PASS   = <pass>

# Upstream forwarding â€” optional; set for authenticated-proxy deployments
AGENT_MITMPROXY__UPSTREAM_URL      = http://upstream:8080
AGENT_MITMPROXY__UPSTREAM_USER     = <user>   # optional even in upstream mode
AGENT_MITMPROXY__UPSTREAM_PASS     = <pass>

# Admin API auth (unchanged)
FAST_API__AUTH__API_KEY__NAME      = X-API-Key
FAST_API__AUTH__API_KEY__VALUE     = <secret>
```

### Validation

Re-run Phase 1.11 test against the deployed EC2 spike with upstream env vars set.
Expected: Chromium nav ~235 ms, Firefox nav ~323 ms, all response headers carrying
upstream markers.

## Priority 2 â€” Playwright service proxy cleanup

**Depends on:** P1 landing (sidecar must support running without `--proxyauth`)
**Target:** next Playwright service version after current

### What changes

| File | Change |
|---|---|
| `sgraph_ai_service_playwright/schemas/browser/Schema__Browser__Config.py` | Remove `proxy: Schema__Proxy__Config` field |
| `sgraph_ai_service_playwright/schemas/browser/Schema__Browser__Launch__Result.py` | Remove `proxy: Schema__Proxy__Config` field |
| `sgraph_ai_service_playwright/schemas/browser/Schema__Proxy__Config.py` | **Delete** â€” no longer in API surface |
| `sgraph_ai_service_playwright/schemas/browser/Schema__Proxy__Auth__Basic.py` | **Delete** â€” no longer in API surface |
| `sgraph_ai_service_playwright/service/Browser__Launcher.py` | `build_launch_kwargs()` reads `ENV_VAR__DEFAULT_PROXY_URL` from env; `build_proxy_dict()` returns `{'server': url}` (no auth, no browser-name branching). Remove the Chromium/Firefox/WebKit auth branch. |
| `sgraph_ai_service_playwright/service/Sequence__Runner.py` | `get_or_create_page()` reads `ENV_VAR__IGNORE_HTTPS_ERRORS` from env (not from `proxy.ignore_https_errors`). Remove `proxy_auth_binder.bind()` call. |
| `sgraph_ai_service_playwright/service/Proxy__Auth__Binder.py` | **Delete** |
| `sgraph_ai_service_playwright/service/Playwright__Service.py` | Remove `proxy_auth_binder` field + import |
| Tests touching proxy schemas / binder | Update to remove proxy-path coverage |

### Decisions

1. **`ignore_https_errors` moves to env var.** `ENV_VAR__IGNORE_HTTPS_ERRORS`
   (`SG_PLAYWRIGHT__IGNORE_HTTPS_ERRORS`) already exists in `env_vars.py`.
   On EC2, ops sets it to `true` alongside `SG_PLAYWRIGHT__PROXY_URL`.
   `Sequence__Runner` reads it directly; no schema needed.

2. **`Schema__Proxy__Config` + `Schema__Proxy__Auth__Basic` are deleted.**
   They are only used in the `browser_config.proxy` path which is being removed.
   No other code references them after P2.

3. **`Proxy__Auth__Binder` is deleted.** The CDP Fetch dance was the workaround
   for the exact bug the sidecar now solves. It was never fully validated on Lambda.
   Dead code from the moment P1 lands.

4. **`browser_config` stays on request schemas.** Callers still pass
   `browser_name`, `headless`, `viewport`, `user_agent`, `locale`, `timezone`,
   `launch_args` â€” all of which remain useful. Only the `proxy` field is removed.
