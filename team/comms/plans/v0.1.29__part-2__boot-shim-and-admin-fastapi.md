1. A generic **S3-zip code loader** (candidate for future contribution to OSBot_AWS).
2. An always-up **admin FastAPI** with the user's application mounted on top when healthy.

---

## 1. Shape of the new boot

Today (v0.1.28):
```
CMD python3 lambda_entry.py
  └─ boot()
      ├─ resolve code source (local / S3 / passthrough)
      ├─ import Fast_API__Playwright__Service
      ├─ if OK  → uvicorn.run(app)
      └─ if BAD → raise → process dies → Function URL returns ''
```

After v0.1.29:
```
CMD python3 lambda_entry.py
  └─ boot()
      ├─ build admin_app (ALWAYS — no external deps beyond osbot-*)
      ├─ try resolve + import user app
      │     ├─ OK  → mount user_app at "/"  , admin_app at "/admin"
      │     └─ BAD → ONLY admin_app at "/" , error pinned in admin_app.state
      └─ uvicorn.run(root_app) — ALWAYS starts, always serves /admin/*
```

The container is never dark. If the user app is broken, `curl /admin/info` still returns JSON explaining what failed.

## 2. Generic code loader

Target package path: `container/sg_lambda_runtime/loader/`.

Nothing in this package imports `sgraph_ai_service_playwright` or references Playwright. The loader's job ends at "the user package is on sys.path"; it never instantiates user code.

### Public surface

```python
# container/sg_lambda_runtime/loader/Code_Loader.py
class Code_Loader(Type_Safe):

    # Config — all three mandatory, all from env vars set by Lambda config
    lambda_name         : Safe_Str__Display_Name    # e.g. 'sg-playwright-api'
    stage               : Safe_Str__Display_Name    # 'dev' | 'main' | 'prod'
    code_version        : Safe_Str__Version         # 'v0.1.29'

    # Optional overrides
    local_path_override : Safe_Str__File__Path      # if set, skip S3 entirely
    cache_root          : Safe_Str__File__Path      # default '/tmp/sg-code'

    def resolve(self) -> Schema__Load_Result:       # precedence: local > S3 > passthrough
        ...

    def _load_from_local(self) -> Schema__Load_Result | None:
        ...

    def _load_from_s3(self) -> Schema__Load_Result | None:
        ...
```

### Precedence (unchanged from v0.1.28)

1. `local_path_override` is set → use it, skip S3.
2. `AWS_REGION` is set AND all three required env vars are set → download `lambdas/{name}/{stage}/{version}.zip`, extract to `/tmp/sg-code/{name}/{stage}/{version}/`, prepend to `sys.path`.
3. Otherwise → passthrough (whatever `sys.path` already has — laptop / pytest / local uvicorn).

### ETag cache (unchanged from v0.1.28)

Warm-invocation short-circuit: if `/tmp/.../v0.1.29/.etag` matches S3's current ETag, skip the download. Saves ~200ms per warm hit.

### Result schema

```python
class Schema__Load_Result(Type_Safe):
    code_source : Safe_Str__Text      # 'local:…', 's3:…', 'passthrough:sys.path'
    sys_path    : Safe_Str__File__Path # the directory that was inserted (or '' for passthrough)
    bytes_read  : int                 # 0 for cache hit / passthrough / local
    cache_hit   : bool
    etag        : Safe_Str__Id        # S3 ETag if applicable
```

This schema is the contract between the loader and the admin FastAPI (which surfaces it on `/admin/info`).

### Error handling

The loader never raises on a *missing* piece — it returns a `Schema__Load_Result` with `code_source='error:<reason>'` and lets the boot layer decide whether that's fatal. Only genuine bugs (malformed zip, IO errors mid-extraction) raise.

### Why this is the extractable unit

Everything in `Code_Loader` is about "fetch a Python zip from S3 and put it on sys.path." That's generic. The only inputs that are project-specific are:
- The bucket naming pattern (`{account}--sg-playwright--{region}`) — parametric; can be overridden per caller.
- The env var names — the module accepts them via constructor; the container's shim reads `SG_PLAYWRIGHT__*` and passes the values in.

When this moves to OSBot_AWS, `Code_Loader` is the class that moves; the SG_PLAYWRIGHT-specific shim stays here.

## 3. Admin FastAPI

Target package path: `container/sg_lambda_runtime/admin_api/`.

Minimal dependencies: `osbot-utils`, `osbot-fast-api-serverless`. NO Playwright, NO `sgraph_ai_service_playwright`.

### Routes

| Route | Returns | Always available |
|---|---|---|
| `GET /admin/health` | `{status: 'ok'}` — the admin FastAPI itself is up | Yes — even if user code is broken |
| `GET /admin/info` | `Schema__Admin__Info` — image_version, code_source, lambda_name/stage/version, boot_status, user_app_loaded (bool) | Yes |
| `GET /admin/boot-log` | Tail of boot stdout/stderr (captured into a ring buffer during `boot()`) | Yes |
| `GET /admin/error` | `{error: <string or null>}` — the pinned error if user app failed to load | Yes |
| `GET /admin/env` | Redacted env-var snapshot (keys only for anything matching `*_KEY*`, `*_TOKEN*`, `*_SECRET*`; full value otherwise) | Yes |
| `POST /admin/reload` | Re-runs `boot()` without process restart. Returns the new `Schema__Load_Result`. | Yes — useful for "I just uploaded a new zip; pick it up" |

### Wire example

```
GET /admin/info

{
  "image_version"    : "v1",
  "lambda_name"      : "sg-playwright-api",
  "stage"            : "dev",
  "code_version"     : "v0.1.29",
  "code_source"      : "s3:745506449035--sg-playwright--eu-west-2/lambdas/sg-playwright-api/dev/v0.1.29.zip→/tmp/sg-code/…",
  "cache_hit"        : true,
  "bytes_read"       : 0,
  "user_app_loaded"  : true,
  "boot_status"      : "ok",
  "boot_duration_ms" : 142
}
```

When the user app is broken:
```
GET /admin/info

{
  "image_version"    : "v1",
  "lambda_name"      : "sg-playwright-api",
  "stage"            : "dev",
  "code_version"     : "v0.1.29",
  "code_source"      : "s3:…/v0.1.29.zip→/tmp/sg-code/…",
  "cache_hit"        : false,
  "bytes_read"       : 428731,
  "user_app_loaded"  : false,
  "boot_status"      : "error",
  "boot_error"       : "ImportError: cannot import name 'Foo' from 'sgraph_ai_service_playwright.bar'",
  "boot_duration_ms" : 983
}
```

### Why this matters

This is the single biggest behavioural change in v0.1.29: **the container is always reachable for diagnostics**, even when the user's zip is broken. Three real scenarios this fixes:

1. **Env var misconfigured** — `/admin/info` shows `code_source: error:missing SG_PLAYWRIGHT__CODE_S3_VERSION`. Operator fixes the env var, hits `/admin/reload`.
2. **Zip missing in S3** — `/admin/info` shows `code_source: error:NoSuchKey lambdas/sg-playwright-api/dev/v0.1.99.zip`. Operator re-uploads or rolls back via env var.
3. **Import error in user code** — `/admin/info` shows `ImportError: …`. Operator rolls back `SG_PLAYWRIGHT__CODE_S3_VERSION` to the last known good, hits `/admin/reload`.

No more "empty body" mystery (today's failure mode).

## 4. Mounting the user FastAPI

The container owns the root ASGI app. The user's `Fast_API__Playwright__Service` is mounted under `/`.

```python
# container/lambda_entry.py (sketch)

from fastapi import FastAPI
from sg_lambda_runtime.admin_api.Admin_API import Admin_API
from sg_lambda_runtime.loader.Code_Loader  import Code_Loader

def build_root_app():
    root = FastAPI()
    root.mount('/admin', Admin_API().app())      # ALWAYS mounted

    result = Code_Loader(...).resolve()          # Puts user code on sys.path
    if result.is_error():
        root.state.boot_error = result.error     # surfaced on /admin/error
        return root                              # User routes absent; 503 on '/'

    try:
        from sgraph_ai_service_playwright.fast_api.Fast_API__Playwright__Service import Fast_API__Playwright__Service
        user_app = Fast_API__Playwright__Service().setup().app()
        root.mount('/', user_app)
    except Exception as exc:
        root.state.boot_error = f'{type(exc).__name__}: {exc}'

    return root
```

**Mount order matters:** admin mounts FIRST so it wins on path collisions (user code can't accidentally hide `/admin/*`).

**Catch-all 503:** if `root.state.boot_error` is set and no user app mounted, any non-`/admin` request returns `503` with a pointer: `{"error": "user app not loaded", "see": "/admin/info"}`.

## 6. Stability contract for the container FastAPI

The container's admin FastAPI is **the stable surface**. Changes to it are rare and backwards-compatible. Contrast with the user FastAPI (hot-swapped via S3 zip every push).

Rule: any breaking change to `/admin/*` requires a new image (Track A) and coordinated rollout. Any addition is allowed in Track A releases.

This matters because operators script against `/admin/info` — if it changes shape, their tooling breaks. The point of the admin surface is that it's boring and reliable.

## 8. Decisions locked by this part

1. ✅ Generic code loader lives under `container/sg_lambda_runtime/loader/`.
2. ✅ Admin FastAPI is always mounted at `/admin/*`, user FastAPI at `/`.
3. ✅ Container is never dark: uvicorn always starts, admin always responds.
4. ✅ `/admin/reload` supports in-place code refresh without process restart.
5. ✅ Loader returns `Schema__Load_Result`; errors are returned, not raised (except genuine bugs).
