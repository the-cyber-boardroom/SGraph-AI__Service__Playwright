## Strategy

The Playwright pipeline is a monolithic `ci-pipeline.yml` with staged jobs (unit tests → build image → push ECR → provision Lambdas). For agent-mitmproxy we'll ship a **separate, independent workflow** (`ci__agent_mitmproxy.yml`) — different ECR repo, different EC2 instance, different provisioner. Keeps blast-radius small: breaking the Playwright pipeline should never block mitmproxy deploys, and vice versa.

Both workflows live under `.github/workflows/` and share GHA secrets; nothing else.

## CI workflow — `.github/workflows/ci__agent_mitmproxy.yml`

**Jobs:**

```
detect-changes          ← dorny/paths-filter; guards whether to build
   │
   ▼
unit-tests              ← tests/unit/agent_mitmproxy/ + tests/unit/scripts/test_provision_mitmproxy_ec2.py
   │
   ▼
build-and-push-image    ← gated on detect-changes; builds docker, pushes ECR
   │
   ▼
provision-ec2           ← idempotent upsert of agent-mitmproxy EC2
                          (--mode=full if image changed, no-op otherwise)
```

**Triggers:**

- `push` to `dev` when any of `agent_mitmproxy/**`, `scripts/provision_mitmproxy_ec2.py`, `tests/unit/agent_mitmproxy/**`, `.github/workflows/ci__agent_mitmproxy.yml`, or `tests/unit/scripts/test_provision_mitmproxy_ec2.py` changed.
- `workflow_dispatch` with `force_image_rebuild` and `should_provision` booleans.

**Paths-filter rules** (follows the Playwright `v0.1.31` pattern — filter is narrow enough that touching docs / unrelated service doesn't spin the pipeline):

```yaml
filters: |
  code:
    - 'agent_mitmproxy/**'
    - 'scripts/provision_mitmproxy_ec2.py'
    - 'tests/unit/agent_mitmproxy/**'
    - '.github/workflows/ci__agent_mitmproxy.yml'
  image:
    - 'agent_mitmproxy/docker/images/**'
    - 'agent_mitmproxy/agent_mitmproxy/**'      # source code is baked into the image
    - 'agent_mitmproxy/requirements.txt'
```

`build-and-push-image` runs when `image` changes OR `force_image_rebuild=true`.
`provision-ec2` runs on every merge (idempotent upsert — picks up env-var changes without rebuilding).

**Secrets required** (on top of what Playwright already uses):

| Secret | Purpose |
|---|---|
| `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_DEFAULT_REGION`, `AWS_ACCOUNT_ID` | Shared with Playwright |
| `AGENT_MITMPROXY__PROXY_AUTH_USER` | Basic proxy auth user (e.g. `agent`) |
| `AGENT_MITMPROXY__PROXY_AUTH_PASS` | Basic proxy auth password |
| `FAST_API__AUTH__API_KEY__NAME`, `FAST_API__AUTH__API_KEY__VALUE` | Admin API auth (can reuse Playwright's or have a distinct one; Phase 1 picks distinct for blast-radius) |

## ECR repo

- **Name**: `agent-mitmproxy` (osbot_aws's `Create_Image_ECR` auto-creates it on first push).
- **Tag**: `:latest` only in Phase 1. Versioned tags are a Phase-5 hardening item.
- **Same account + region** as Playwright's ECR (`<account>.dkr.ecr.<region>.amazonaws.com/agent-mitmproxy`).

## EC2 provisioning — `scripts/provision_mitmproxy_ec2.py`

Mirrors `scripts/provision_ec2.py` exactly in structure. Differences:

### Constants

```python
EC2__INSTANCE_TYPE              = 't3.small'                  # mitmproxy is light — 2GB RAM
EC2__AMI_NAME_AL2023            = 'al2023-ami-2023.*-x86_64'   # Same AL2023 pattern
EC2__AMI_OWNER_AMAZON           = 'amazon'
EC2__PROXY_PORT                 = 8080                         # HTTP CONNECT proxy
EC2__ADMIN_PORT                 = 8000                         # FastAPI admin
EC2__MITMWEB_PORT_LOCAL         = 8081                         # bound 127.0.0.1; not exposed on SG

IAM__ROLE_NAME                  = 'agent-mitmproxy-ec2-spike'
IAM__ECR_READONLY_POLICY_ARN    = 'arn:aws:iam::aws:policy/AmazonEC2ContainerRegistryReadOnly'
IAM__SSM_CORE_POLICY_ARN        = 'arn:aws:iam::aws:policy/AmazonSSMManagedInstanceCore'
IAM__POLICY_ARNS                = (IAM__ECR_READONLY_POLICY_ARN, IAM__SSM_CORE_POLICY_ARN)
IAM__ASSUME_ROLE_SERVICE        = 'ec2.amazonaws.com'

SG__NAME                        = 'agent-mitmproxy-ec2-spike'   # NOT 'sg-*' — AWS reserves that prefix
SG__DESCRIPTION                 = 'Agent Mitmproxy EC2 spike - ingress :8080 (proxy) + :8000 (admin)'

TAG__NAME                       = 'agent-mitmproxy-ec2-spike'
TAG__STAGE_KEY                  = 'stage'
DEFAULT_STAGE                   = 'dev'

IMAGE_NAME                      = 'agent_mitmproxy'             # From Docker__Agent_Mitmproxy__Base
```

### UserData template

```sh
#!/bin/bash
set -euxo pipefail

dnf install -y docker
systemctl enable --now docker

TOKEN=$(aws ecr get-login-password --region {region})
echo "$TOKEN" | docker login --username AWS --password-stdin {registry}

docker pull {image_uri}

docker run -d \
    --restart=always \
    --name agent-mitmproxy \
    -p 8080:8080 \
    -p 8000:8000 \
    -e AGENT_MITMPROXY__PROXY_AUTH_USER='{proxy_user}' \
    -e AGENT_MITMPROXY__PROXY_AUTH_PASS='{proxy_pass}' \
    -e FAST_API__AUTH__API_KEY__NAME='{api_key_name}' \
    -e FAST_API__AUTH__API_KEY__VALUE='{api_key_value}' \
    {image_uri}
```

### Ingress rules

Two rules on the SG:

```python
ec2.security_group_authorize_ingress(security_group_id=sg_id, port=EC2__ADMIN_PORT)   # :8000
ec2.security_group_authorize_ingress(security_group_id=sg_id, port=EC2__PROXY_PORT)   # :8080
```

Port `:8081` is NOT opened — mitmweb UI is bound to localhost and accessed via admin API proxy routes.

### CLI surface

Same flags as `provision_ec2.py`:

```bash
python scripts/provision_mitmproxy_ec2.py --stage dev        # upsert
python scripts/provision_mitmproxy_ec2.py --stage dev --image-uri <custom>
python scripts/provision_mitmproxy_ec2.py --terminate        # nuke all tagged instances
```

Return shape:

```json
{
  "action": "create",
  "instance_id": "i-...",
  "public_ip": "18.x.x.x",
  "proxy_url": "http://<user>:<pass>@18.x.x.x:8080",
  "admin_url": "http://18.x.x.x:8000",
  "image_uri": "...",
  "ami_id": "...",
  "stage": "dev"
}
```

Note the `proxy_url` contains creds — printed for operator convenience, but shell history hygiene applies.

## IAM

Single role/instance profile `agent-mitmproxy-ec2-spike` with:

1. Trust: `ec2.amazonaws.com` can assume.
2. Managed policies attached:
   - `AmazonEC2ContainerRegistryReadOnly` (docker pull from ECR)
   - `AmazonSSMManagedInstanceCore` (SSM Session Manager)
3. Phase-3 additions (deferred):
   - `AmazonPrometheusRemoteWriteAccess`
   - `AmazonOpenSearchServiceFullAccess` (or a narrower inline policy for the specific domain)

## SG

Single SG `agent-mitmproxy-ec2-spike` with:

- Inbound `tcp/8000` from `0.0.0.0/0` (admin API, X-API-Key auth)
- Inbound `tcp/8080` from `0.0.0.0/0` (proxy, Basic auth)
- Outbound: default (all to all)

## Deploy flow — first-time

1. Push to `dev` → `ci__agent_mitmproxy.yml` fires.
2. `unit-tests` passes.
3. `build-and-push-image` — Docker builds `agent_mitmproxy:latest`, pushes to ECR (creating the repo if missing).
4. `provision-ec2` — runs `python scripts/provision_mitmproxy_ec2.py --stage dev`. Creates role+SG+instance.
5. Instance boots (~45-90s — dnf install, ECR login, docker pull, container start, mitmproxy first-cert generation).
6. Operator smoke-tests:
   - `curl -H "X-API-Key:$KEY" http://<ip>:8000/health/info` → 200.
   - `curl -x http://user:pass@<ip>:8080 https://example.com` → 200 with `X-Agent-Mitmproxy-*` headers.
7. If both pass, record `<ip>` in the Phase-4 integration doc for Playwright to use.

## Deploy flow — subsequent pushes

Path-filter gating keeps most PRs cheap:

| Change | unit-tests | build | provision |
|---|---|---|---|
| `agent_mitmproxy/**` source | ✅ | ✅ | ✅ |
| `agent_mitmproxy/docker/images/**` | ✅ | ✅ | ✅ |
| `scripts/provision_mitmproxy_ec2.py` | ✅ | ❌ | ✅ |
| `tests/unit/agent_mitmproxy/**` | ✅ | ❌ | ❌ |
| Docs only / other service | ❌ | ❌ | ❌ |

`provision` runs on every green merge. Idempotent — existing IAM/SG reused; instance terminated+recreated only if `--terminate` was passed first, otherwise we'd add an `--update` mode later (same story as Playwright's `--mode=code-only`).

## Phase 1 — cost estimate

| Resource | $/day |
|---|---|
| EC2 t3.small on-demand | ~$0.50 |
| ECR storage (one small image) | <$0.10 |
| Data transfer (dev-scale) | <$0.20 |
| **Total** | **<$1/day** |

Spike-grade. Leave running for the week; `--terminate` when done.

## Open items for DevOps review

1. **Separate ECR repo namespace?** (e.g. `sgraph-ai/agent-mitmproxy`). osbot_aws's `Create_Image_ECR` uses flat names by default; we'd need a small override. Deferred.
2. **Same-account or cross-account ECR?** Phase 1 assumes same account as Playwright. Cross-account image sharing is a separate story.
3. **Dev vs main vs prod stages.** Phase 1 hardcodes `dev`. `main` + `prod` variants can reuse the same script with `--stage` — but need SG/IAM name-suffixing to avoid collisions. Easy extension in Phase 2.
