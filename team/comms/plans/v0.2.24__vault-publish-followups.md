## 2. Deferred items — out of scope for this plan

These three are deliberately **not** in this PR. They will be re-evaluated after the operator runs `sg vp bootstrap` against live AWS.

### 2.1 Route 53 wildcard ALIAS record (item 1)

Original plan called for `bootstrap()` to create `*.aws.sg-labs.app` → CF distribution ALIAS. Operator has already created this manually in the AWS console. No code change.

**Future re-evaluation trigger:** if a second zone or environment ever needs bootstrap, the manual step doesn't scale — implement then.

### 2.2 SSM persistence (item 2)

Original plan called for SSM keys under `/sg-compute/vault-publish/bootstrap/`. Skipped — `bootstrap()` returns the values in the response; operator captures them manually.

**Future re-evaluation trigger:** if re-running `bootstrap` accidentally creates orphan distributions, add SSM lookup as a guard.

### 2.3 Idempotency (item 3)

Tied to 2.2 — without SSM, the bootstrap can't look up the existing distribution. Observe first, decide later.

## 3. Work plan — what lands in this PR

Grouped by area. Every change is additive except where noted.

### 3.1 New CLI sub-verbs — `sg vp waker info / logs / invoke` (item 4)

**Scope:**

| Path | Action | What |
|------|--------|------|
| `sg_compute_specs/vault_publish/cli/Cli__Vault_Publish.py` | EXTEND | New `waker` sub-typer with three commands |
| `sg_compute_specs/vault_publish/service/Vault_Publish__Service.py` | EXTEND | `waker_info()`, `waker_logs(tail, follow)`, `waker_invoke(slug, path)` |
| `sg_compute_specs/vault_publish/schemas/Schema__Vault_Publish__Waker__Info__Response.py` | NEW | `lambda_name / function_arn / function_url / runtime / state / last_modified` |
| `sg_compute_specs/vault_publish/schemas/Schema__Vault_Publish__Waker__Logs__Response.py` | NEW | `lambda_name / lines / content / truncated` |
| `sg_compute_specs/vault_publish/schemas/Schema__Vault_Publish__Waker__Invoke__Response.py` | NEW | `lambda_name / slug / path / status_code / body_preview / elapsed_ms` |

**Behaviour:**

- `sg vp waker info` — reads via `Lambda__AWS__Client.get_function` + `get_function_url` and renders a Rich table.
- `sg vp waker logs --tail N --follow` — pulls from CloudWatch Logs (`/aws/lambda/sg-compute-vault-publish-waker`). Uses `osbot_aws.aws.logs.Logs` if available; otherwise add a thin boto3 `EXCEPTION` wrapper at `sgraph_ai_service_playwright__cli/aws/logs/service/Logs__AWS__Client.py` (NEW).
- `sg vp waker invoke <slug> [--path /]` — directly invokes the Lambda via the Function URL with a synthetic `Host: <slug>.aws.sg-labs.app` header, returns status + body preview.

**Factory seam:** add `_logs_client_factory` to `Vault_Publish__Service` so tests can inject an in-memory log store.

**Tests:** unit tests with in-memory fakes for each verb. No golden files — these are inspection commands; their output schema is unstable by design.

### 3.3 Reuse opportunities (items 11, 12)

**Item 11 — `Slug__From_Host` composes `Route53__Zone__Resolver`:**

Replace today's manual suffix-strip with `Route53__Zone__Resolver().resolve_zone_for_fqdn(host)` + extract leaf label. Two-line change. Existing tests (10 of them) must all still pass.

**Item 12 — `Endpoint__Resolver__EC2` uses `osbot_aws.aws.ec2.EC2_Instance`:**

Replace raw `boto3.client('ec2').describe_instances` with `EC2_Instance(instance_id).{state, ip_address, start}`. Removes the `# EXCEPTION` comment header. The resolver becomes thinner — most logic moves to upstream `osbot-aws`.

**Risk:** `EC2_Instance` may not expose a way to list-instances-by-tag. If it does not, keep the `describe_instances` call as an EXCEPTION but switch the per-instance operations to `EC2_Instance`. Verify before refactor.
