### 3.6 `sg aws iam` sub-package — locked-down role lifecycle + audit (item 14)

**Driver:** the operator's v0.2.23 test run forced manual `aws iam create-role` + `aws iam put-role-policy` invocations from the test guide. Every future Lambda / EC2 role hits the same pattern. **This sub-package makes least-privilege the default path** — every role created this way carries an explicit `Condition` block, never `Resource: "*"` without a written exception, and ships with a `check` verb that audits existing roles for over-permissive grants.

**Project-rule anchors:** CLAUDE.md rule #12 (no AWS creds in Git), rule #14 (no `sg-` prefix on SG GroupName). New: every role created via `sg aws iam` must satisfy a least-privilege contract enforced by the same code that creates it. The contract is a typed schema, not a comment.

#### 3.6.3 The least-privilege contract

Every `Schema__IAM__Statement` constructed by this package must satisfy:

1. **`actions`** is a `List__Safe_Str__Aws__Action` — never the raw string `"*"`. Wildcard-suffix actions like `"ec2:Describe*"` are allowed; bare `"*"` is rejected at construction.
2. **`resources`** is a `List__Safe_Str__ARN`. Bare `"*"` requires an explicit `allow_wildcard_resource: bool = False` flag on the statement, defaulted to deny. Tests assert the default rejects.
3. **Power actions** — `iam:PassRole`, `ec2:Start*`, `ec2:Stop*`, `ec2:Terminate*`, `lambda:Invoke*` — require a `Condition` block. `IAM__Trust_Policy__Builder` and the policy templates enforce this at build time, not at AWS-call time.

These are not advisory — they're constructor invariants. A caller that wants to bypass them (e.g. CloudWatch logs needing `logs:*` on `*`) has to set the explicit flag and the audit report will permanently flag the role at severity WARN.

#### 3.6.4 `sg aws iam role check` — the audit verb

`IAM__Policy__Auditor.audit(role: Schema__IAM__Role) -> Schema__IAM__Audit__Report` runs the role's inline + managed policies through a battery of pattern detectors. Each detector emits a `Schema__IAM__Audit__Finding`.

| Detector | Severity | What it flags |
|----------|----------|---------------|
| `WildcardActionDetector` | CRITICAL | any statement with `Action: "*"` |
| `WildcardResourceDetector` | WARN | `Resource: "*"` outside an allowlist (logs, CloudWatch metrics, X-Ray) |
| `PowerActionMissingConditionDetector` | CRITICAL | `iam:PassRole` / `ec2:Start*` / `ec2:Stop*` / `ec2:Terminate*` / `lambda:Invoke*` without a `Condition` block |
| `AdminAccessAttachedDetector` | CRITICAL | `AdministratorAccess` or `PowerUserAccess` managed policy attached |
| `StaleRoleDetector` | INFO | role unused for >90 days (last-used timestamp from IAM) |
| `OverlyBroadServiceDetector` | WARN | service prefix wildcarded (e.g. `iam:*`, `s3:*`) |
| `MissingTagConditionDetector` | WARN | `Resource` references EC2 / S3 / Lambda without a tag-based `Condition` |

Each finding includes a `remediation_hint` field — short, actionable string.

Render order in the CLI: CRITICAL first, then WARN, then INFO. Exit codes: 0 = all clean; 1 = WARN findings; 2 = CRITICAL findings. Operators can wire this into pre-merge checks.

#### 3.6.5 Policy templates (reusable artefacts)

These are typed Python objects, not JSON files. The Waker bootstrap calls them.

**`Waker__Policy__Template.build() -> Schema__IAM__Policy`** — what `sg vp bootstrap` needs the Lambda exec role to carry:

```python
Schema__IAM__Policy(
    version='2012-10-17',
    statements=List__Schema__IAM__Statement([
        Schema__IAM__Statement(                            # CloudWatch logs — accepted exception
            effect='Allow',
            actions=['logs:CreateLogGroup', 'logs:CreateLogStream', 'logs:PutLogEvents'],
            resources=['arn:aws:logs:*:*:*'],
            allow_wildcard_resource=True),                 # explicit opt-in; audit flags WARN
        Schema__IAM__Statement(                            # EC2 read
            effect='Allow',
            actions=['ec2:DescribeInstances'],
            resources=['*'],
            allow_wildcard_resource=True),                 # EC2 Describe has no resource-level perms; AWS limitation
        Schema__IAM__Statement(                            # EC2 start — tag-conditioned, narrow ARN
            effect='Allow',
            actions=['ec2:StartInstances'],
            resources=['arn:aws:ec2:*:*:instance/*'],
            condition={'StringEquals': {'aws:ResourceTag/StackType': 'vault-app'}}),
    ]))
```

Audit report on this template (target state): 1 INFO (logs wildcard), 1 INFO (EC2 Describe wildcard). 0 WARN. 0 CRITICAL.

**`Vault_App__Stop__Policy__Template.build()`** — formalises the inline policy from `Ec2__AWS__Client` (P1a). Same tag-conditioned shape.
