## 3. Env vars (single-shot cutover)

| Old | New | Notes |
|---|---|---|
| `SG_PLAYWRIGHT__CODE_SOURCE` | `AGENTIC_CODE_SOURCE` | Resolved value written by boot shim (`s3:…`, `local:…`, `passthrough:sys.path`). |
| — | `AGENTIC_CODE_SOURCE_S3_BUCKET` | Optional override; defaults to `{account}--sgraph-ai--{region}`. |
| `SG_PLAYWRIGHT__CODE_S3_KEY` | `AGENTIC_CODE_SOURCE_S3_KEY` | Key only, e.g. `apps/sg-playwright/dev/v0.1.29.zip`. |
| `SG_PLAYWRIGHT__CODE_LOCAL_PATH` | `AGENTIC_CODE_LOCAL_PATH` | Escape hatch for passthrough/local dev. |
| — | `AGENTIC_APP_NAME` | e.g. `sg-playwright`. Used to build S3 path and manifest. |
| — | `AGENTIC_APP_STAGE` | `dev` / `main` / `prod`. |
| — | `AGENTIC_APP_VERSION` | e.g. `v0.1.29`. |
| — | `AGENTIC_ADMIN_MODE` | `disabled` / `read_only` / `full`. Default `read_only`. `full` deferred. |

Boot shim composes the key as `apps/{AGENTIC_APP_NAME}/{AGENTIC_APP_STAGE}/{AGENTIC_APP_VERSION}.zip` if `AGENTIC_CODE_SOURCE_S3_KEY` is not set explicitly.

## 5. Admin surface (read_only mode, first pass)

All mounted under `/admin/`:

| Route | Method | Returns |
|---|---|---|
| `/admin/health` | GET | `Schema__Agentic__Health` — always 200; boot status (`loaded` / `degraded`). |
| `/admin/info` | GET | `Schema__Agentic__Info` — app name, stage, version, code_source, image_version, python version. |
| `/admin/env` | GET | `Schema__Agentic__Env` — redacted view of `AGENTIC_*` vars only. No AWS creds. |
| `/admin/boot-log` | GET | Last N lines of the boot shim's in-memory log. |
| `/admin/error` | GET | Last failed-load error (if any). |
| `/admin/skills/{name}` | GET | Raw markdown from `skills/skill__{name}.md`. |
| `/admin/manifest` | GET | `Schema__Agentic__Manifest` — points at OpenAPI + available SKILLs + capabilities.json URL. |
| `/admin/capabilities` | GET | Serves `capabilities.json` verbatim. |

**`/admin/reload` (POST) is deferred** — requires auth design and admin_mode=full.

## 8. `scripts/deploy_code.py` (one-command end-to-end)

```bash
python scripts/deploy_code.py --stage dev
```

Steps:
1. Read `sgraph_ai_service_playwright/version` → `v0.1.29`.
2. Package source via `S3__Zip_Bytes` (existing code from `package_code.py`).
3. Upload to `s3://{account}--sgraph-ai--{region}/apps/sg-playwright/{stage}/v0.1.29.zip`.
4. Update Lambda env: `AGENTIC_APP_VERSION=v0.1.29`, `AGENTIC_APP_STAGE=dev`, `AGENTIC_CODE_SOURCE_S3_KEY=apps/sg-playwright/dev/v0.1.29.zip`.
5. Smoke test: GET `/admin/health` on Lambda Function URL; assert `{"status": "loaded", "code_source": "s3:…"}`.

Target round-trip: **~30 seconds cloud** (brief §4). Local iteration (no S3) target: **~5 seconds** via `AGENTIC_CODE_LOCAL_PATH`.
