## File-by-file plan

### 1. `sg_compute_specs/docker/cli/Cli__Docker.py` — REWRITE

Replace the entire file with a Builder-driven shell. Mirror `Cli__Ollama.py` closely.

```python
# ═══════════════════════════════════════════════════════════════════════════════
# SG/Compute Specs — Docker: Cli__Docker
# Builder-driven CLI; 8 standard verbs come from Spec__CLI__Builder.
# Spec extras on `create`: --registry, --api-key, --enable-shell.
# Custom renderers from sg_compute_specs.docker.cli.Renderers (legacy preserved).
# ═══════════════════════════════════════════════════════════════════════════════

from sg_compute.cli.base.Schema__Spec__CLI__Spec import Schema__Spec__CLI__Spec
from sg_compute.cli.base.Spec__CLI__Builder      import Spec__CLI__Builder

from sg_compute_specs.docker.cli.Renderers                          import (render_create,
                                                                            render_delete,
                                                                            render_info,
                                                                            render_list)
from sg_compute_specs.docker.schemas.Schema__Docker__Create__Request import Schema__Docker__Create__Request
from sg_compute_specs.docker.service.Docker__Service                 import Docker__Service


def _set_extras(request, registry='', api_key='', enable_shell=False):
    request.registry         = registry
    request.api_key_ssm_path = api_key
    request.enable_shell     = bool(enable_shell)


_cli_spec = Schema__Spec__CLI__Spec(
    spec_id                    = 'docker'                          ,
    display_name               = 'Docker'                          ,
    default_instance_type      = 't3.medium'                       ,
    create_request_cls         = Schema__Docker__Create__Request   ,
    service_factory            = lambda: Docker__Service().setup() ,
    health_path                = '/health'                         ,   # sidecar exposes /health
    health_port                = 8080                              ,   # Fast_API__Host port
    health_scheme              = 'https'                           ,   # sidecar nginx terminates TLS
    extra_create_field_setters = _set_extras                       ,
    render_list                = render_list                       ,
    render_info                = render_info                       ,
    render_create              = render_create                     ,
    render_delete              = render_delete                     ,
)


app = Spec__CLI__Builder(
    cli_spec             = _cli_spec,
    extra_create_options = [
        ('registry'    , str , '' , 'ECR registry host (enables sidecar).'),
        ('api_key'     , str , '' , 'SSM parameter path for the sidecar API key.'),
        ('enable_shell', bool, False, 'Disable the shell command allowlist on the sidecar.'),
    ],
).build()
```

**Watch-outs:**

- The legacy CLI exposed `--registry / --api-key / --enable-shell` as **kebab-case** typer flags (`--registry`, `--api-key`, `--enable-shell`). The Builder converts underscore → kebab (`('api_key', ...)` → `--api-key`). That preserves backward compat. Verify with `sg-compute spec docker create --help`.
- `health_path`, `health_port`, `health_scheme` need confirming against the actual sidecar. Docker's sidecar is the host control plane FastAPI; check `sg_compute/host_plane/Fast_API__Host.py` and `sg_compute_specs/docker/service/Docker__User_Data__Builder.py` to confirm port/scheme. **If port is 80/HTTP behind nginx, set scheme accordingly — same lesson as Ollama-on-11434/HTTP.**
- `Schema__Spec__CLI__Spec` accepts `render_list/info/create/delete` (per the v0.2.6 contract). If the field names differ in the actual schema, align — do NOT add new fields.

### 2. `sg_compute_specs/docker/service/Docker__Service.py` — EDIT

Three changes:

#### 2a. Extend `Spec__Service__Base`

```python
# Replace:
from osbot_utils.type_safe.Type_Safe import Type_Safe
class Docker__Service(Type_Safe):

# With:
from sg_compute.core.spec.Spec__Service__Base import Spec__Service__Base
class Docker__Service(Spec__Service__Base):
```

This brings free implementations of `health()`, `exec()`, `connect_target()` (D1: SSM-only).

#### 2b. Bridge or replace the existing `health()` method

Today's `Docker__Service.health()` returns `Schema__Docker__Health__Response`, but `Spec__CLI__Builder` calls it expecting `Schema__CLI__Health__Probe` (see `Spec__CLI__Builder._wait_healthy` and `_register_health`). Two options:

- **Option A (recommended):** Delete the spec-specific `health()` method entirely. Inherit `Spec__Service__Base.health()` which returns `Schema__CLI__Health__Probe`. The legacy `Schema__Docker__Health__Response` is only consumed by the legacy CLI we're deleting and (verify with grep) any control-plane wrapper.
- **Option B:** Keep both — rename the existing one to `health_legacy()` if anything still calls it. Less clean; only do this if grep finds external callers.

**Run `grep -rn "Schema__Docker__Health__Response\|\.health(" sg_compute/ sg_compute_specs/ tests/` before deciding.** If only the deleted CLI consumes it, go Option A and delete `Schema__Docker__Health__Response.py` + `Docker__Health__Checker.py`. Note net LOC reduction in the PR description.

#### 2c. Implement `cli_spec()` (matches the Ollama pattern)

Add this method on `Docker__Service` so anything that asks the service for its CLI shape gets it from the service, not the CLI module:

```python
def cli_spec(self):
    from sg_compute.cli.base.Schema__Spec__CLI__Spec import Schema__Spec__CLI__Spec
    return Schema__Spec__CLI__Spec(
        spec_id               = 'docker'                          ,
        display_name          = 'Docker'                          ,
        default_instance_type = DEFAULT_INSTANCE_TYPE             ,
        create_request_cls    = Schema__Docker__Create__Request   ,
        service_factory       = lambda: Docker__Service().setup() ,
        health_path           = '/health'                         ,
        health_port           = 8080                              ,
        health_scheme         = 'https'                           ,
    )
```

#### 2d. Fix the `req.stack_name.__init__(sname)` legacy bug if it's reachable from anywhere else

Grep for `req.stack_name.__init__` and `req.region.__init__` across the repo. If any legacy callers remain, switch them to `setattr(req, 'stack_name', sname)`. (Same fix Ollama needed in `c0e9387`.)

## Lessons baked-in from the Ollama wedge

These caused merged-then-fixed bugs after v0.2.7. Each MUST be checked on Docker as part of this PR — not deferred:

| # | Lesson | Where it bites Docker | Action |
|--:|--------|------------------------|--------|
| L1 | `req.stack_name.__init__(name)` is a no-op (Safe_Str inherits immutable str). | `Cli__Docker.py:83-84` (legacy) — about to be deleted, but verify no copy-paste in `Docker__Service.create_stack`. | Use `setattr(req, 'stack_name', name)` exclusively. Builder already does this since `c0e9387`. |
| L2 | `Spec__Service__Base.health()` HTTP scheme defaulted to `https`; an HTTP-only API silently fails the probe. | Docker sidecar TLS termination — confirm scheme. | Set `health_scheme` explicitly on `Schema__Spec__CLI__Spec`. **Document the choice in a one-line `# scheme` comment on the field.** |
| L3 | Health probe returned early when no public IP; `--wait` failed instantly. | Docker uses public IP for sidecar reach — same path. | Inherited fix already in `Spec__Service__Base.health()` from `0afd0f5`. Verify by reading the method, don't re-implement. |
| L4 | EC2 launch needed `instance_profile_name='playwright-ec2'` for SSM. | `Docker__Service.create_stack` already passes `PROFILE_NAME='playwright-ec2'` (L83 of `Docker__Service.py`). ✓ | Verify; no action expected. |
| L5 | Disk size silently dropped — default became `0` (use AMI default). | Docker has `disk_size_gb: Safe_Int__Disk__GB` in `Schema__Docker__Create__Request`, default not set — yields `0`. | Decide: keep `0` (match legacy), or pick a sane default (e.g. `30`). **Recommend keeping `0` — Docker is more polymorphic than Ollama; users who need 200 GiB pass `--disk-size 200`.** |
| L6 | Bool-default-true typer options need `--flag/--no-flag` (Builder handles automatically since `c0e9387`). | None of Docker's bool extras default to `True`. ✓ | No action; smoke test `--enable-shell` flag exists. |
| L7 | `ssm-user` was missing from the docker group — every SSM session needed `sudo docker`. | Docker stacks ARE Docker. Critical. | Already fixed in `Section__Docker.py` (`fabd828`). Verify via `grep "usermod -aG docker ssm-user" sg_compute/platforms/ec2/user_data/`. |
| L8 | `provision_ec2.py` imported `app` from empty `cli/__init__.py`. | Same risk for Docker. | Verify the import path in `scripts/provision_ec2.py`. |
