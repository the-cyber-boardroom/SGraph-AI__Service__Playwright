## What shipped

### Schema redesign (your proposal, adopted verbatim)

- `Schema__Proxy__Auth__Basic` nested under `Schema__Proxy__Config.auth` (None ⇒ open proxy).
- **New primitives** — `Safe_Str__Basic_Auth__Username` / `Safe_Str__Basic_Auth__Password`. The osbot `Safe_Str__Username` silently underscored hyphens (`qa-user` → `qa_user`), which would have masked the fix in testing. The new types preserve hyphens, dots, and common punctuation verbatim. **If your QA probe uses real proxy creds with hyphens, this is the reason they now round-trip cleanly.**
- `Schema__Browser__Launch__Result` now retains the proxy config so the page-creation site can read `ignore_https_errors` + `auth` without re-parsing.

### Runtime wiring

- **`Browser__Launcher.build_proxy_dict`** emits **only** `server` + `bypass` to `chromium.launch`. Username/password never appear in launch kwargs. There is an explicit regression test for this: `tests/unit/service/test_Browser__Launcher.py::test__proxy_auth_never_reaches_launch_kwargs`.
- **`Proxy__Auth__Binder`** (new class) owns the CDP dance:
  - `Fetch.enable {handleAuthRequests: True}` on first page creation
  - `Fetch.authRequired` → `Fetch.continueWithAuth {response: ProvideCredentials, ...}`
  - `Fetch.requestPaused` → `Fetch.continueRequest` (**mandatory passthrough** — enabling `handleAuthRequests` pauses every request, not just auth challenges; this is the subtle part your reference `Proxy_Via_Browser` had right)
  - Stale `requestId` errors swallowed in both handlers (CDP fires after nav abandons the request)
- **`Action__Runner` / `Sequence__Runner`** apply `ignore_https_errors` on `new_context()` and invoke the binder on first page creation, keyed by `session_id` so per-session proxy isolation is real.
- **`Playwright__Service.setup()`** threads `Browser__Launcher` + `Proxy__Auth__Binder` into both runners.

## Good-failure / bad-failure notes for your debrief

### Good failures surfaced during Dev

- **Osbot `Safe_Str__Username` stripped hyphens.** Found during the L2 unit test for the binder — assertion `str(auth.username) == 'qa-user'` failed because the schema sanitised it to `qa_user`. Fix: domain-specific `Safe_Str__Basic_Auth__*`. Lesson: when a schema change touches a field that already has an osbot default primitive, verify the char class fits the real-world values before trusting the default. Candidate for `library/guides/` once we have two examples.
- **TestClient thread affinity.** Playwright sync is thread-bound; FastAPI TestClient spins a new thread per request. Multi-request flows (`/session/create` → `/browser/navigate` → `/session/close`) blew up with *"cannot switch to a different thread (which happens to have exited)"*. Dodged by using single-request `POST /sequence/execute` for L3/L4. Full multi-request HTTP coverage needs a real uvicorn subprocess — punted to a follow-up slice.

### Bad failures (still live)

- **No CI integration test for authenticated proxies.** Repeats the gap your debrief already flagged. My L-tier tests are local-only. Until this is in CI, Bug #1 can regress silently during any launcher refactor.
- **No documented caller guidance** on when `ignore_https_errors` is required. Clients using a TLS-intercepting proxy in the wild will hit `NET::ERR_CERT_AUTHORITY_INVALID` with no hint of the fix.
