### Type_Safe Quick Rules

1. **All classes** extend `Type_Safe`
2. **All attributes** have type annotations
3. **Zero raw primitives** — no `str`, `int`, `float`, `list`, `dict`
4. **No Literals** — use `Enum__*` classes for all fixed-value sets
5. **Schemas are pure data** — no methods
6. **Collection subclasses are pure type definitions** — no methods
7. **Use `@type_safe`** on methods that validate parameters
8. **Return `None`** for not found — no exceptions for missing resources
9. **No `str()` casts** — Type_Safe handles conversions
10. **No `Optional[T]`** — returning `None` is always allowed
11. **Simplified defaults** — auto-conversion handles it
12. **No docstrings** — inline comments aligned at end of lines
13. **`═══` 80-char headers** on every file and section

## FastAPI Routes

| Document | Path | When to read |
|---|---|---|
| **Routes development guide** | `guidance/fast_api_routes.md` | Before writing any route class |
| **Service client architecture** | `guidance/unified_service_client.md` | Before writing client package |

### Routes Quick Rules

1. **Extend `Fast_API__Routes`** — never use `@app.get` decorators
2. **`tag` attribute** → URL prefix
3. **Method name** → path (single `_` → `-`, double `__` → path segment)
4. **`setup_routes()`** registers all endpoints
5. **NEVER use Pydantic** — Type_Safe schemas only
6. **`ROUTES_PATHS__*` constants** document all paths
7. **Routes have no logic** — pure delegation to service classes

## Testing Quick Rules

1. **`with SomeClass() as _:`** — context manager pattern
2. **`.obj() == __(...)`** — comprehensive single-assertion comparisons
3. **`__SKIP__`** — for auto-generated fields
4. **`setUpClass`** — shared test objects
5. **No mocks, no patches** — real implementations, in-memory backends
6. **Test file mirrors source** — `test_Routes__Health.py` for `Routes__Health.py`
