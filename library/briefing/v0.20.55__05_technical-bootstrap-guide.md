# Technical Bootstrap Guide

**Version:** v0.20.55

Step-by-step instructions for setting up the SG Playwright service repo.

**Golden rule: Docker image before service code.**

---

## Phase 0: Repo Skeleton + Docker Image

### 0.1 Create Repo Structure

```
SGraph-AI__Service__Playwright/
├── .claude/
│   ├── CLAUDE.md                         # Adapt from claude-md-templates/
│   └── explorer/
│       └── CLAUDE.md
├── .github/
│   └── workflows/
│       ├── ci-pipeline.yml               # Base (workflow_call)
│       ├── ci-pipeline__dev.yml          # Dev trigger
│       ├── ci-pipeline__main.yml         # Main trigger
│       └── ci-pipeline__prod.yml         # Prod (manual)
├── sgraph_ai_service_playwright/
│   ├── __init__.py
│   ├── version                           # v0.1.0
│   ├── schemas/                          # From schema-catalogue-v2.md
│   ├── fast_api/                         # Routes + lambda_handler.py
│   ├── service/                          # 12 service classes
│   ├── dispatcher/                       # Step schema registries
│   ├── client/                           # Stateless client + registration
│   ├── docker/                           # Docker infrastructure classes
│   ├── consts/                           # env_vars.py, version.py
│   └── docker/images/sgraph_ai_service_playwright/      # Dockerfile + handler + requirements
├── tests/
│   ├── unit/
│   ├── integration/
│   ├── docker/
│   └── deploy/
├── team/explorer/                        # 6 roles
├── pyproject.toml
└── README.md
```

### 0.2 Create Dockerfile First

Copy from `dev-specs/ci-pipeline.md` Section 6. Verify it builds:

```bash
cd sgraph_ai_service_playwright/docker/images/sg_playwright
docker build -t sgraph_ai_service_playwright:latest .
docker run --rm -p 8888:8000 sgraph_ai_service_playwright:latest
# Should respond on http://localhost:8888/health/status
```

### 0.3 Create Docker Infrastructure Classes

Implement `Docker__SGraph-AI__Service__Playwright__Base` + 4 subclasses per `dev-specs/ci-pipeline.md` Section 3.

### 0.4 Create CI Pipeline

Copy workflows from `dev-specs/ci-pipeline.md` Section 2. Push to dev — Job 1 (unit tests) should pass.

### 0.5 Verify Local Container

```bash
python -m pytest tests/docker/test_Build__Docker__SGraph-AI__Service__Playwright.py -v -s
python -m pytest tests/docker/test_Local__Docker__SGraph-AI__Service__Playwright.py -v -s
```

**Phase 0 is complete when:** Docker image builds, local container starts, `/health/status` returns 200, CI Job 1 passes.

## Phase 1: Implement Schemas + Routes + Service Classes

### 1.1 Reading Order

1. `dev-specs/schema-catalogue-v2.md` — implement all schemas exactly as specified
2. `dev-specs/routes-catalogue-v2.md` — implement all routes and service classes
3. `guidance/type_safe.md` — if any doubt about Type_Safe patterns

### 1.2 Implementation Order

1. Custom Safe_* primitives (`schemas/primitives/`)
2. Enums (`schemas/enums/`)
3. Core schemas (service info, browser config, viewport, vault refs)
4. Step schemas (16 action types)
5. Step result schemas (base + 3 per-action)
6. Request/response schemas (session, sequence, action)
7. Collection subclasses
8. Step schema registries (STEP_SCHEMAS + STEP_RESULT_SCHEMAS)
9. Service classes (start with Capability__Detector, then Session__Manager, etc.)
10. Route classes (start with Routes__Health, then Session, Browser, Sequence)
11. lambda_handler.py + Fast_API__Playwright__Service

### 1.3 Verify

```bash
python -m pytest tests/unit/ -v --timeout=30
```
