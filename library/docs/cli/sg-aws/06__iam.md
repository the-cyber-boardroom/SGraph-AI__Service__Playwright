# 06 — `sg aws iam`

IAM role lifecycle (list, show, create, delete, audit) plus policy attach / detach / list.

**Mutation gate:** `SG_AWS__IAM__ALLOW_MUTATIONS=1` required for `role create/delete`, `policy attach/detach`.

Subcommand groups:

| Group | What it does |
|-------|--------------|
| `role` | Role lifecycle + audit |
| `policy` | Managed-policy attachment |

---

## `role list`

List IAM roles, optionally filtered by name prefix.

```bash
sg aws iam role list
sg aws iam role list --prefix sg-
sg aws iam role list --prefix sg- --json
```

**Flags:** `--prefix/-p P`, `--json`.

---

## `role show NAME`

Role details, trust policy, inline policies, attached managed policies, last-used.

```bash
sg aws iam role show sg-lambda-executor
sg aws iam role show sg-lambda-executor --json
```

---

## `role create NAME`  ⚠ mutates

Create an IAM role with a trust policy for a named AWS service.

```bash
SG_AWS__IAM__ALLOW_MUTATIONS=1 \
  sg aws iam role create sg-lambda-executor --trust-service lambda
```

**Flags:**
- `--trust-service/-t {lambda|ec2|ecs-tasks|api-gateway}` (default `lambda`)
- `--description/-d TEXT`
- `--json`

The trust policy is generated by `IAM__Trust_Policy__Builder` — one of four canonical shapes. Custom trust policies are not supported via the CLI today (use the AWS console).

---

## `role delete NAME`  ⚠ mutates

Delete a role. **All attached policies are detached first** — the command does the cleanup; you do not have to manually `policy detach` everything before deleting.

```bash
SG_AWS__IAM__ALLOW_MUTATIONS=1 sg aws iam role delete sg-old-role --yes
```

**Flags:** `--yes/-y`, `--json`.

---

## `role check NAME`

Audit a role for over-permissive grants. **Exit codes:**

| Exit | Meaning |
|-----:|---------|
| 0 | Clean |
| 1 | WARN — something to look at |
| 2 | CRITICAL — fix before next deploy |

```bash
sg aws iam role check sg-lambda-executor
sg aws iam role check sg-lambda-executor --json
```

Useful in CI to fail builds that attach `*:*` or other dangerous policies:

```bash
for role in sg-lambda-executor sg-elastic-fetcher sg-cf-invalidator; do
  sg aws iam role check "$role" || exit 1
done
```

---

## `policy attach ROLE`  ⚠ mutates

Attach an AWS-managed or customer-managed policy to a role.

```bash
SG_AWS__IAM__ALLOW_MUTATIONS=1 sg aws iam policy attach sg-lambda-executor \
  --arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole
```

**Flags:** `--arn ARN` (required).

---

## `policy detach ROLE`  ⚠ mutates

Detach a policy.

```bash
SG_AWS__IAM__ALLOW_MUTATIONS=1 sg aws iam policy detach sg-lambda-executor \
  --arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole
```

**Flags:** `--arn ARN` (required).

---

## `policy list ROLE`

List both inline and managed policies attached to a role.

```bash
sg aws iam policy list sg-lambda-executor
sg aws iam policy list sg-lambda-executor --json
```

---

## A typical "stand up a Lambda role" sequence

```bash
# 1. create the role
SG_AWS__IAM__ALLOW_MUTATIONS=1 \
  sg aws iam role create sg-lambda-executor --trust-service lambda \
    --description "Execution role for sg-compute Lambdas"

# 2. attach the minimum permissions
SG_AWS__IAM__ALLOW_MUTATIONS=1 sg aws iam policy attach sg-lambda-executor \
  --arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

# 3. verify
sg aws iam role show   sg-lambda-executor
sg aws iam policy list sg-lambda-executor
sg aws iam role check  sg-lambda-executor   # exit 0 = clean
```
