## Section 9 — Cross-Schema Validation Rules

Rules that span multiple schemas and cannot be enforced at primitive level. Validated by the service logic **per-action** before dispatch.

| Rule | Checked when | Error code |
|---|---|---|
| If `sink=VAULT`, `sink_vault_ref` must be non-None | Per-action artefact capture | `sink_missing_vault_ref` |
| If `sink=S3`, `sink_s3_bucket` must be non-None | Per-action artefact capture | `sink_missing_s3_bucket` |
| If `sink=LOCAL_FILE` and deployment is Lambda | Per-action artefact capture | `sink_incompatible_with_deployment` |
| If `sink=LOCAL_FILE` and deployment is Claude Web | Per-action artefact capture | `sink_incompatible_with_deployment` |
| If `lifetime_hint=PERSISTENT_SINGLE` and deployment is Lambda | Session create | `persistent_not_supported` |
| If `lifetime_hint=PERSISTENT_DISTRIBUTED` | Session create | `distributed_not_supported` |
| If `provider=CDP_CONNECT`, `cdp_endpoint_url` must be non-None | Session create | `cdp_missing_endpoint` |
| If `total_timeout_ms > capabilities.max_session_lifetime_ms` | Sequence execute | `timeout_exceeds_deployment_limit` |
| If `step.action=EVALUATE` and `step.expression` fails allowlist | Per-action dispatch | `evaluate_expression_not_allowed` |
| If `browser_config.proxy.server` starts with `https://` and `ignore_https_errors=False` | Session create | `proxy_tls_intercept_warning` (warning, not reject) |
| Duplicate step IDs within a sequence | Sequence execute | `duplicate_step_ids` |
| `video_start` without a matching `video_stop` | Sequence execute completion | `video_not_stopped` (warning, not reject) |

All rejections return `Schema__Error__Response` with the `error_code` from above, `capabilities` surfaced when relevant, and `trace_id` echoed from request.

## Section 12 — Summary Checklist

When implementing from this catalogue:

- [ ] Every identifier inherits from `Safe_Id`, `Safe_Str`, or a domain-specific Safe primitive — no raw `str`
- [ ] Every enum is `(str, Enum)` with a `__str__` method returning `self.value`
- [ ] **No `Literal` types anywhere** — all fixed-value sets are `Enum__*` classes
- [ ] Every schema inherits from `Type_Safe`, contains only annotations, no methods
- [ ] Every collection subclass inherits from `Type_Safe__Dict`/`List`/`Set`, contains only class-level type vars
- [ ] Every file has the `# ═══` 80-char header
- [ ] Every attribute annotation aligns `:` with neighbours
- [ ] Every default value is immutable (or uses auto-init empty defaults)
- [ ] Every cross-schema reference uses actual import — no `'Schema__Other'` string refs
- [ ] Every env var read uses `get_env(ENV_VAR__NAME)` with module-level constants
- [ ] Every step action has entries in both `STEP_SCHEMAS` and `STEP_RESULT_SCHEMAS`
- [ ] Every response schema is producible at every deployment target (per capabilities check)
- [ ] `launch_args` default behaviour: caller-supplied list replaces defaults entirely; service logs the override
- [ ] Capture config validated per-action (not at session create)
- [ ] Duplicate step IDs within a sequence are rejected (422)
- [ ] `Schema__Vault_Ref.vault_key` accepts both `"drum-hunt-6610"` and `"j4pyy0lhny8jx7osqn4lclhq:mzrp0li8"` formats
