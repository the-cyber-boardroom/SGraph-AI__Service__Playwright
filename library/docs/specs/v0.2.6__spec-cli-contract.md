# v0.2.6 — Spec CLI Contract

**Status:** IN EFFECT from v0.2.6.
**Authority:** This document wins over any per-spec CLI implementation that deviates from it.
**Builder:** `sg_compute/cli/base/Spec__CLI__Builder.py` — generates all 8 standard verbs.

---

## §1. Canonical command set — every spec MUST expose these

| Verb | Signature | Behaviour |
|------|-----------|-----------|
| `list` | `--region` | List all stacks for the spec in region. |
| `info` | `[name?]`, `--region` | Show one stack. `name` resolved by §3 rules. |
| `create` | `[name?]`, `--region`, `--instance-type`, `--max-hours=1`, `--ami`, `--caller-ip`, `--wait` | Provision. `name` auto-generated when omitted. `--wait` polls health until ready. **`--max-hours=1` for every spec — D2.** |
| `delete` | `[name?]`, `--region`, `--yes`, `-y` | Terminate. Confirms unless `--yes`. **D3 — confirms even after auto-pick.** |
| `wait` | `[name?]`, `--region`, `--timeout=600`, `--poll=10` | Block until healthy. Same probe as `health --timeout 600`. |
| `health` | `[name?]`, `--region`, `--timeout=0` | Instant probe by default; `--timeout > 0` makes it a `wait`. **D7 — same code path.** |
| `connect` | `[name?]`, `--region` | `os.execvp('aws', ['ssm', 'start-session', ...])`. |
| `exec` | `[name?]`, `<command>`, `--region`, `--timeout=60`, `--cwd=''` | Run a shell command on the host via SSM SendCommand. **D1 — SSM only.** Returns stdout/stderr/exit_code. |

All defaults live in `sg_compute/cli/base/Spec__CLI__Defaults.py`:
- `DEFAULT_REGION = 'eu-west-2'`
- `DEFAULT_MAX_HOURS = 1` (D2)
- `DEFAULT_TIMEOUT_SEC = 600`
- `DEFAULT_POLL_SEC = 10`
- `DEFAULT_EXEC_TIMEOUT = 60`

## §2. Optional verbs

Each spec **may** add extra verbs (e.g. `ollama models`, `firefox set-interceptor`).
Add them with `app.command()` on the Builder's returned `typer.Typer`.

---

## §3. `name` resolution rule

When a verb takes optional `name` and none is supplied:

1. List all stacks in region.
2. **0 stacks** → print `"No {spec} stacks in {region}. Run: sg-compute spec {spec_id} create"` to stderr in yellow; exit 1.
3. **1 stack** → print `"One stack found — using {name}"` to stdout in dim; use that name.
4. **>1 stacks** → number them, prompt "Pick a stack number"; exit 1 on invalid input.

Implementation: `Spec__CLI__Resolver` — `sg_compute/cli/base/Spec__CLI__Resolver.py`.

---

## §4. Error handling

Every command is wrapped by `@spec_cli_errors` from `Spec__CLI__Errors`. It:

- Catches `KeyboardInterrupt` and re-raises (no traceback).
- Catches `typer.Exit` and re-raises.
- Catches AWS-credential exceptions (`'credential' in str(e).lower()` OR type name ends with `'NoCredentialsError'`) → prints to stderr; exit 1.
- Catches everything else → red `✗ {ExceptionType}: {msg}`; with `--debug` prints the full traceback; exit 2.

Root callback accepts `--debug`/`-D` and enables full tracebacks.

## §5. Service contract — what every `{Pascal}__Service` must offer

The minimum surface the builder calls. Inherit defaults from `Spec__Service__Base` — most specs only need to override the four spec-specific bits.

| Method | Default impl in `Spec__Service__Base`? | Spec must override? |
|--------|----------------------------------------|---------------------|
| `setup()` | no | yes |
| `create_stack(request)` | no | yes |
| `list_stacks(region)` | no | yes |
| `get_stack_info(region, name)` | no | yes |
| `delete_stack(region, name)` | no | yes |
| `health(region, name, timeout_sec=0, poll_sec=10) -> Schema__CLI__Health__Probe` | yes — HTTP probe against `health_path`/`health_port` from `cli_spec()` | only when probe needs auth/path tricks |
| `exec(region, name, command, timeout_sec=60, cwd='') -> Schema__CLI__Exec__Result` | yes — SSM SendCommand (D1 SSM-only) | rarely |
| `connect_target(region, name) -> str` | yes — pulls `instance_id` from `get_stack_info` | rarely |
| `cli_spec() -> Schema__Spec__CLI__Spec` | no | yes — declares `health_path`, `health_port`, `spec_id`, etc. |

## §7. Per-spec configuration consumed by the builder

```python
Schema__Spec__CLI__Spec(
    spec_id               = 'docker',
    display_name          = 'Docker',
    default_instance_type = 't3.medium',     # D4 — per-spec, no global default
    create_request_cls    = Schema__Docker__Create__Request,
    service_factory       = lambda: Docker__Service().setup(),
    health_path           = '/',
    health_port           = 80,
    extra_create_field_setters = None,       # (request, **extra_opts) -> None
)
```

`extra_create_options` passed to `Spec__CLI__Builder.__init__`: `List[Tuple[name, type, default, help]]`.

## Decisions captured here

| # | Decision |
|---|---------|
| D1 | `exec` uses SSM SendCommand only (no sidecar dual-path). |
| D2 | `max_hours=1` for every spec. Override via `--max-hours`. |
| D3 | `delete` always confirms, even after auto-pick. |
| D4 | `--instance-type` default is per-spec, not global. |
| D7 | `wait` and `health --timeout N` use the same code path. |
