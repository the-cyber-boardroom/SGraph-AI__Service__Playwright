### 1.2 Responsibility Split

| Component | Responsibility | Contains logic? |
|---|---|---|
| `Routes__*` | HTTP surface: parse requests, call service, format responses | No — delegation only |
| `Playwright__Service` | Thin orchestrator: wire sub-services, delegate | Minimal — glue only |
| `Request__Validator` | All cross-schema validation (session create, step, sink compat, duplicate IDs) | Yes |
| `Session__Manager` | Session lifecycle: create, lookup, expire, close | Yes |
| `Browser__Launcher` | Chromium process: launch, connect, configure, stop | Yes |
| `Sequence__Runner` | Sequence execution loop: parse steps, run in order, aggregate results | Yes |
| `Action__Runner` | Single-action execution: parse one step, validate, execute, return | Yes |
| `Sequence__Dispatcher` | Step parsing: dict → typed Schema__Step__* via STEP_SCHEMAS registry | Yes |
| `Step__Executor` | Per-action Playwright calls: navigate, click, fill, screenshot, etc. | Yes — **only class touching page.\*** |
| `Capability__Detector` | Detect deployment target, build `Schema__Service__Capabilities` | Yes |
| `Artefact__Writer` | Write artefacts to sinks (vault/inline/s3/local) | Yes |
| `Credentials__Loader` | Read cookies / storage state from vault, apply to browser context | Yes |
| `JS__Expression__Allowlist` | Validate `evaluate` step expressions against trusted patterns | Yes |
