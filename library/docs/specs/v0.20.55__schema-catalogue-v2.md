## Section 1 — Design Principles

### 1.1 Runs Everywhere

The service must execute identically across five deployment surfaces:

| Surface | Constraints |
|---|---|
| Developer laptop | Full control; can run headful; persistent disk; any duration |
| CI pipeline (GitHub Actions) | Headless; ephemeral; short duration; network-allowlisted |
| Claude Web / Claude Code sessions | Headless; ephemeral; `/tmp` only; TLS-intercepting proxy on egress |
| Container / K8s / Fargate | Headless; long-lived; persistent disk via volume mounts |
| AWS Lambda (container) | Headless; 15-min max per invocation; `/tmp` only; 5 GB memory |

**Schema implication:** the caller declares *intent* (e.g. "I want a persistent session that lives across requests") and the service validates against *capability* on the current deployment target. A persistent-session request against Lambda gets rejected with a helpful error suggesting Fargate. The schemas never assume any single environment.

### 1.2 Sink-Flexible Artefacts

Every artefact (screenshot, video, HAR, trace) must be routable to one of four sinks:

- **vault** — committed to an SG/Send vault (the 16 April arch brief's primary path)
- **inline** — base64-encoded in the HTTP response (small artefacts only)
- **s3** — written directly to an S3 key (large artefacts, no vault roundtrip)
- **local_file** — written to a local path (laptop / container dev only; rejected on Lambda)

Each capture flag has an independent sink choice. Rationale: a laptop dev wants inline screenshots for debugging + no vault; a production QA run wants vault-only; a Fargate content pipeline wants S3 video + vault metadata.

### 1.3 Client Package Model

The HTTP/API surface documented here is the **transport layer**, not the primary caller interface. A separate client package (`sg-playwright-client`) will provide domain-specific methods that wrap these endpoints. Callers of the client package do not need to know about HTTP request details, JSON schemas, or API versioning. The client package resolves version compatibility using the `/info` endpoint's version fields (`service_version`, `playwright_version`, etc.) before dispatching requests.

This means:
- **Wire-format backwards compatibility** is managed by adding new fields with defaults (never removing fields)
- **Breaking changes** are handled by the client package detecting the service version and adapting
- **Step action vocabulary** can grow without breaking old clients — unknown actions are rejected by the dispatcher, not by the schema

### 1.4 Schema Boundaries

- Schemas are **pure data containers** — no methods, no logic. Logic lives in `Playwright__Service`, `Sequence__Dispatcher`, `Session__Manager`, `Artefact__Writer__*` helper classes (routes doc).
- Collection subclasses are **pure type definitions** — no methods.
- All identifiers inherit from the appropriate `Safe_*` primitive — no raw `str`.
- All durations are `Safe_UInt__Milliseconds` — no ambiguity about units.
- All sizes are `Safe_UInt__FileSize` — bytes, with `.to_kb()` / `.to_mb()` helpers built in.
- **No `Literal` types.** All fixed-value sets use proper `Enum__*` classes for runtime validation, serialisation, and IDE discoverability.

### 1.5 Forward References

Type_Safe forward references must point to the same class. Cross-schema references resolve through actual class imports — no `'Schema__Other'` string references.

**`launch_args` behaviour:**

The service maintains a default set of Chromium launch arguments:

```
--no-sandbox
--disable-gpu
--disable-dev-shm-usage
--single-process
--use-mock-keychain
```

If the caller provides `launch_args` in the request, the caller's list **replaces the defaults entirely**. The service logs a diagnostic message noting the override. Callers who know what they are doing get full control; callers who omit `launch_args` get safe defaults for every deployment target.

### 8.3 Step Parsing Flow

```python
# ═══════════════════════════════════════════════════════════════════════════════
# Playwright Service — Step Parsing (reference implementation for routes doc)
# ═══════════════════════════════════════════════════════════════════════════════

from fastapi import HTTPException


def parse_step(step_dict: dict                                      ,               # Parse one step from the wire format
               step_index: int                                                      # Position in sequence for default Step_Id
          ) -> Schema__Step__Base:
    action_str = step_dict.get('action')
    if action_str is None:
        raise HTTPException(422, f"Step at index {step_index} missing required field: action")

    try:
        action = Enum__Step__Action(action_str)                                     # Validates against enum; raises on unknown
    except ValueError:
        raise HTTPException(422, f"Step at index {step_index} has unknown action: {action_str}")

    schema_class = STEP_SCHEMAS.get(action)
    if schema_class is None:
        raise HTTPException(500, f"No schema registered for action: {action_str}")

    step = schema_class.from_json(step_dict)                                        # Type_Safe parses + validates all fields

    if not step.id:                                                                 # Default Step_Id to index if caller omitted
        step.id = Step_Id(str(step_index))

    return step


def result_schema_for(action: Enum__Step__Action) -> type:                          # Get result schema class for an action
    return STEP_RESULT_SCHEMAS.get(action) or Schema__Step__Result__Base
```

## Section 9 — Cross-Schema Validation Rules

Rules that span multiple schemas and cannot be enforced at primitive level. Validated by the service logic **per-action** before dispatch.

| Rule | Checked when | Error code |
|---|---|---|
| If `sink=VAULT`, `sink_vault_ref` must be non-None | Per-action artefact capture | `sink_missing_vault_ref` |
| If `sink=S3`, `sink_s3_bucket` must be non-None | Per-action artefact capture | `sink_missing_s3_bucket` |
| If `sink=LOCAL_FILE` and deployment is Lambda | Per-action artefact capture | `sink_incompatible_with_deployment` |
| If `sink=LOCAL_FILE` and deployment is Claude Web | Per-action artefact capture | `sink_incompatible_with_deployment` |
| If `lifetime_hint=PERSISTENT_SINGLE` and deployment is Lambda | Session create | `persistent_not_supported` |
| If `lifetime_hint=PERSISTENT_DISTRIBUTED` | Session create | `distributed_not_supported` |
| If `provider=CDP_CONNECT`, `cdp_endpoint_url` must be non-None | Session create | `cdp_missing_endpoint` |
| If `total_timeout_ms > capabilities.max_session_lifetime_ms` | Sequence execute | `timeout_exceeds_deployment_limit` |
| If `step.action=EVALUATE` and `step.expression` fails allowlist | Per-action dispatch | `evaluate_expression_not_allowed` |
| If `browser_config.proxy.server` starts with `https://` and `ignore_https_errors=False` | Session create | `proxy_tls_intercept_warning` (warning, not reject) |
| Duplicate step IDs within a sequence | Sequence execute | `duplicate_step_ids` |
| `video_start` without a matching `video_stop` | Sequence execute completion | `video_not_stopped` (warning, not reject) |

All rejections return `Schema__Error__Response` with the `error_code` from above, `capabilities` surfaced when relevant, and `trace_id` echoed from request.

## Section 12 — Summary Checklist

When implementing from this catalogue:

- [ ] Every identifier inherits from `Safe_Id`, `Safe_Str`, or a domain-specific Safe primitive — no raw `str`
- [ ] Every enum is `(str, Enum)` with a `__str__` method returning `self.value`
- [ ] **No `Literal` types anywhere** — all fixed-value sets are `Enum__*` classes
- [ ] Every schema inherits from `Type_Safe`, contains only annotations, no methods
- [ ] Every collection subclass inherits from `Type_Safe__Dict`/`List`/`Set`, contains only class-level type vars
- [ ] Every file has the `# ═══` 80-char header
- [ ] Every attribute annotation aligns `:` with neighbours
- [ ] Every default value is immutable (or uses auto-init empty defaults)
- [ ] Every cross-schema reference uses actual import — no `'Schema__Other'` string refs
- [ ] Every env var read uses `get_env(ENV_VAR__NAME)` with module-level constants
- [ ] Every step action has entries in both `STEP_SCHEMAS` and `STEP_RESULT_SCHEMAS`
- [ ] Every response schema is producible at every deployment target (per capabilities check)
- [ ] `launch_args` default behaviour: caller-supplied list replaces defaults entirely; service logs the override
- [ ] Capture config validated per-action (not at session create)
- [ ] Duplicate step IDs within a sequence are rejected (422)
- [ ] `Schema__Vault_Ref.vault_key` accepts both `"drum-hunt-6610"` and `"j4pyy0lhny8jx7osqn4lclhq:mzrp0li8"` formats
