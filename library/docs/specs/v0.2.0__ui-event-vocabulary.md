# v0.2.0 — UI Event Vocabulary

Single source of truth for every `CustomEvent` dispatched on `document` by the SG/Compute admin dashboard.

All events use `{ bubbles: true, composed: true }` so they cross Shadow DOM boundaries.

Status values: **ACTIVE** · **DEPRECATED** (removal target: v0.3.0) · **RESERVED** (defined, not yet wired)

## Deprecation schedule

All events marked DEPRECATED are targeted for removal at v0.3.0. Any external listener must migrate before that release.

The canonical hyphen-form aliases (`sp-cli:stack-selected`, `sp-cli:launch-success`, etc.) were introduced for back-compat with pre-v0.2 tooling. The dot-form (`sp-cli:node.selected`, `sp-cli:launch.success`) is the canonical form.

`sp-cli:plugin:*` → `sp-cli:spec:*` migration is complete in emitters as of FV2.9. Both forms are dispatched for one release window.
