## Section 8 — Request Flow Diagrams

### 8.1 Sequence Execute

```
POST /sequence/execute
       │
       ▼
  Routes__Sequence.execute()
       │
       ▼
  Playwright__Service.sequence_execute()  →  Sequence__Runner.execute()
       │
       ├── Capability__Detector.capabilities()
       ├── Request__Validator.validate_sequence_timeout()
       │
       ├── [if no session_id] → Playwright__Service.session_create()
       │       ├── Request__Validator.validate_session_create()
       │       ├── Browser__Launcher.launch()
       │       ├── Session__Manager.create()
       │       └── Credentials__Loader.apply()
       │
       ├── Sequence__Dispatcher.parse_steps()
       │       └── for each: STEP_SCHEMAS[action].from_json()
       │
       ├── Request__Validator.validate_step_ids_unique()
       │
       ├── for each parsed step:
       │       ├── Request__Validator.validate_step()
       │       ├── Sequence__Dispatcher.execute_step()
       │       │       ├── Step__Executor.execute_{action}()
       │       │       ├── STEP_RESULT_SCHEMAS[action]
       │       │       └── Artefact__Writer.capture_*()
       │       └── Session__Manager.record_action()
       │
       ├── [if close_session_after] → Playwright__Service.session_close()
       │
       └── Schema__Sequence__Response
```

## Section 10 — Implementation Checklist

- [ ] `Fast_API__Playwright__Service.py` is a **pure class** — no module-level singletons
- [ ] `lambda_handler.py` is the **separate entry point** that instantiates and starts everything
- [ ] `lambda_handler.py` includes `sys.path.append('/opt/python')` for LWA compatibility
- [ ] `lambda_handler.py` includes `if __name__ == '__main__':` block for `CMD exec python3 handler.py`
- [ ] Every route method returns `.json()` on the response Type_Safe schema — **no raw dicts returned anywhere**
- [ ] `session_save_state` returns `Schema__Session__State__Save__Response` (not dict)
- [ ] `Playwright__Service` is a thin orchestrator — **no validation, no execution, no artefact logic**
- [ ] `Request__Validator` contains **all** validation (session create, step, sink compat, duplicate IDs, timeout)
- [ ] `Action__Runner` handles Layer 0 single-action execution
- [ ] `Sequence__Runner` handles Layer 3 sequence execution loop
- [ ] `Credentials__Loader` handles vault read/write for cookies and storage state
- [ ] `Capability__Detector.build_capabilities()` uses **Type_Safe constructors** (no intermediate dicts)
- [ ] Each deployment target has its own `capabilities__{target}()` method
- [ ] `Step__Executor` is the **only class** that calls `page.*` Playwright methods
- [ ] `Artefact__Writer` is the **only class** that writes to sinks
- [ ] Registration helpers follow `register_*__in_memory()` / `register_*__remote()` convention
- [ ] All service classes inherit from `Type_Safe`
- [ ] All service classes use inline comments, not docstrings
- [ ] All service classes use `═══` 80-char section headers
- [ ] Test file structure mirrors source: `test_Request__Validator.py`, `test_Action__Runner.py`, etc.
