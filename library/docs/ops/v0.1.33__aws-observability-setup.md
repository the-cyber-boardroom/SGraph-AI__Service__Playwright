# AWS Observability Stack — Account Setup Runbook

**Version:** v0.1.33  
**Date:** 2026-04-20  
**Applies to:** Any AWS account running the SG Playwright EC2 stack  
**Author:** DevOps / SRE  

---

## Overview

This runbook covers the one-time AWS account setup needed to enable the full observability stack for the SG Playwright + agent_mitmproxy EC2 deployment.

The observability architecture has two layers:

| Layer | Where it runs | Data persists after EC2 termination? |
|-------|--------------|--------------------------------------|
| **Self-hosted** (Prometheus, Grafana, Loki, cAdvisor, Node Exporter, Promtail) | Docker containers on the EC2 instance | No — Docker named volumes are local |
| **AWS Managed** (AMP, AMG, OpenSearch, CloudWatch, X-Ray) | AWS managed services | Yes — independent of the instance |

For a dev/spike stack, self-hosted-only is sufficient. For a persistent environment, wire the AWS managed layer so data survives instance termination.

---

## Prerequisites

- AWS CLI configured with credentials that have admin or PowerUser access
- AWS region decided (`eu-west-2` is the default for this project)
- IAM Identity Center (SSO) org-level feature is available (required for Amazon Managed Grafana)

## Part 1 — Amazon Managed Service for Prometheus (AMP)

Prometheus data is stored in AMP instead of (or in addition to) the local Docker volume. The self-hosted Prometheus container remote-writes to AMP using SigV4 via the EC2 instance profile — no extra credentials needed.

### 1.1 Create the workspace

```bash
aws amp create-workspace \
  --alias sg-playwright-dev \
  --region eu-west-2
```

Save the output values:

```
workspaceId:        ws-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
prometheusEndpoint: https://aps-workspaces.eu-west-2.amazonaws.com/workspaces/ws-xxx.../
```

For prod, repeat with `--alias sg-playwright-prod`.

### 1.2 Add IAM policy to the EC2 instance profile

The IAM role `playwright-ec2` (created automatically by `sg-ec2 create`) needs write access to AMP:

```bash
aws iam attach-role-policy \
  --role-name playwright-ec2 \
  --policy-arn arn:aws:iam::aws:policy/AmazonPrometheusRemoteWriteAccess
```

Also add read access so AMG can query the workspace on behalf of the role:

```bash
aws iam attach-role-policy \
  --role-name playwright-ec2 \
  --policy-arn arn:aws:iam::aws:policy/AmazonPrometheusQueryAccess
```

### 1.3 Configure remote_write in prometheus.yml

Add the following `remote_write` block to the `prometheus.yml` embedded in the EC2 user-data (in `render_user_data()` in `scripts/provision_ec2.py`):

```yaml
remote_write:
  - url: https://aps-workspaces.{region}.amazonaws.com/workspaces/{workspaceId}/api/v1/remote_write
    queue_config:
      max_samples_per_send: 1000
      max_shards: 200
      capacity: 2500
    sigv4:
      region: eu-west-2                     # must match the workspace region
```

The Prometheus container uses the EC2 instance profile for SigV4 signing automatically — no `aws_access_key_id` or `aws_secret_access_key` needed in the config.

### 1.4 Verify

After the next `sg-ec2 create`, forward Prometheus and check remote-write metrics:

```bash
sg-ec2 forward prometheus        # opens localhost:9090 tunnel
# in browser: localhost:9090/metrics — look for prometheus_remote_storage_*
```

In AWS Console → AMP workspace → Metrics tab — you should see `sg_playwright_*` metrics appearing within 30 seconds.

## Part 2 — Amazon Managed Grafana (AMG)

AMG connects to AMP, CloudWatch, and OpenSearch as built-in data sources. Access is via HTTPS with IAM Identity Center SSO — no port-forwarding needed.

### 2.1 Enable IAM Identity Center

IAM Identity Center must be enabled at the **AWS Organization level** (or standalone account):

1. AWS Console → IAM Identity Center → **Enable**
2. Create at least one user (your own email)
3. Note the Identity Center instance ARN — needed for AMG workspace creation

If SSO is already enabled for the account, skip to 2.2.

### 2.2 Create the AMG workspace

**Via Console** (easier for first setup):

1. AWS Console → Amazon Managed Grafana → **Create workspace**
2. Workspace name: `sg-playwright`
3. Authentication: **AWS IAM Identity Center**
4. Permission type: **Service managed**
5. Data sources to enable: ✓ Amazon Managed Service for Prometheus, ✓ Amazon CloudWatch, ✓ Amazon OpenSearch Service
6. Notification channels (optional): ✓ Amazon SNS

**Via CLI:**

```bash
aws grafana create-workspace \
  --account-access-type CURRENT_ACCOUNT \
  --authentication-providers AWS_SSO \
  --permission-type SERVICE_MANAGED \
  --workspace-name sg-playwright \
  --workspace-data-sources PROMETHEUS CLOUDWATCH AMAZON_OPENSEARCH_SERVICE \
  --region eu-west-2
```

Save the `workspaceId` from the output.

### 2.3 Add users

```bash
# Get the SSO user ID first
aws identitystore list-users \
  --identity-store-id d-xxxxxxxxxx \
  --filters "AttributePath=UserName,AttributeValue=your@email.com"

# Grant admin access to the workspace
aws grafana associate-license \
  --workspace-id g-xxxxxxxxxx \
  --license-type ENTERPRISE_FREE_TRIAL

aws grafana update-permissions \
  --workspace-id g-xxxxxxxxxx \
  --update-instruction-batch '[{"action":"ADD","role":"ADMIN","users":[{"id":"xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx","type":"SSO_USER"}]}]'
```

Or simply use the Console → Workspace → Users → Assign user → set role to Admin.

### 2.4 Add AMP as a data source

1. Open the Grafana workspace URL (shown in Console — format: `https://g-xxx.grafana-workspace.eu-west-2.amazonaws.com`)
2. Log in via SSO
3. Configuration → Data sources → Add → **Amazon Managed Service for Prometheus**
4. Select the workspace created in Part 1
5. Default region: `eu-west-2`
6. Save & Test — should show "Data source connected and labels found"

Repeat for **CloudWatch** (already available — just add region `eu-west-2`) and **Amazon OpenSearch** (needs the domain endpoint from Part 3).

### 2.5 Import dashboards

Pre-built dashboards for cAdvisor and Node Exporter are available on grafana.com:

| Dashboard | Grafana ID | Purpose |
|-----------|-----------|---------|
| Node Exporter Full | `1860` | Host CPU/mem/disk/net |
| cAdvisor | `14282` | Container-level metrics |
| Loki Logs | `13639` | Log browser panel |

Dashboards → Import → Enter ID → Load → Select AMP data source → Import.

Custom dashboards for `sg_playwright_*` and `sg_mitmproxy_*` metrics will be added in Part 2 of the observability implementation (see `library/docs/ops/` for the dashboard JSON files once authored).

## Part 3 — Amazon OpenSearch Service (Log Aggregation)

OpenSearch replaces self-hosted Loki for persistent log storage. Logs survive EC2 termination and are queryable via OpenSearch Dashboards or Grafana.

### 3.1 Create the domain

**Minimum viable dev domain** (adjust for prod):

```bash
aws opensearch create-domain \
  --domain-name sg-playwright \
  --engine-version OpenSearch_2.11 \
  --cluster-config '{"InstanceType":"t3.small.search","InstanceCount":1}' \
  --ebs-options '{"EBSEnabled":true,"VolumeType":"gp3","VolumeSize":30}' \
  --access-policies '{
    "Version":"2012-10-17",
    "Statement":[{
      "Effect":"Allow",
      "Principal":{"AWS":"arn:aws:iam::{ACCOUNT_ID}:role/playwright-ec2"},
      "Action":"es:*",
      "Resource":"arn:aws:es:{REGION}:{ACCOUNT_ID}:domain/sg-playwright/*"
    }]
  }' \
  --advanced-security-options '{"Enabled":true,"InternalUserDatabaseEnabled":true,"MasterUserOptions":{"MasterUserName":"admin","MasterUserPassword":"CHANGE_ME_STRONG_PASSWORD"}}' \
  --node-to-node-encryption-options '{"Enabled":true}' \
  --encryption-at-rest-options '{"Enabled":true}' \
  --domain-endpoint-options '{"EnforceHTTPS":true}' \
  --region eu-west-2
```

Wait for status `Active` (5–10 minutes):

```bash
aws opensearch describe-domain \
  --domain-name sg-playwright \
  --region eu-west-2 \
  --query 'DomainStatus.Processing'
```

Save the domain endpoint:

```bash
aws opensearch describe-domain \
  --domain-name sg-playwright \
  --region eu-west-2 \
  --query 'DomainStatus.Endpoints'
```

Format: `https://search-sg-playwright-xxx.eu-west-2.es.amazonaws.com`

### 3.2 Add IAM policy to the EC2 instance profile

```bash
aws iam put-role-policy \
  --role-name playwright-ec2 \
  --policy-name OpenSearchIngestPolicy \
  --policy-document '{
    "Version":"2012-10-17",
    "Statement":[{
      "Effect":"Allow",
      "Action":["es:ESHttpPost","es:ESHttpPut"],
      "Resource":"arn:aws:es:{REGION}:{ACCOUNT_ID}:domain/sg-playwright/*"
    }]
  }'
```

Replace `{REGION}` and `{ACCOUNT_ID}` with actual values.

### 3.3 Configure Fluent-bit (replaces Promtail for OpenSearch output)

Add a `fluent-bit` container to `COMPOSE_YAML_TEMPLATE` (Part 2 of the observability plan) and embed `fluent-bit.conf` in user-data:

```ini
[SERVICE]
    Flush         5
    Log_Level     info

[INPUT]
    Name              tail
    Path              /var/lib/docker/containers/*/*.log
    Parser            docker
    Tag               docker.*
    Refresh_Interval  5

[FILTER]
    Name  record_modifier
    Match *
    Record hostname ${HOSTNAME}

[OUTPUT]
    Name            opensearch
    Match           *
    Host            search-sg-playwright-xxx.eu-west-2.es.amazonaws.com
    Port            443
    TLS             On
    AWS_Auth        On
    AWS_Region      eu-west-2
    Index           sg-playwright-logs
    Suppress_Type_Name On
```

`AWS_Auth On` uses the EC2 instance profile credentials for SigV4 signing.

### 3.4 Add OpenSearch as a Grafana data source

1. In AMG: Configuration → Data sources → Add → **Amazon OpenSearch Service**
2. URL: `https://search-sg-playwright-xxx.eu-west-2.es.amazonaws.com`
3. Auth: **AWS SigV4** → Region `eu-west-2`
4. Index name: `sg-playwright-logs`
5. Time field: `@timestamp`
6. Save & Test

## Part 4 — CloudWatch Agent (host + container metrics)

The `CloudWatchAgentServerPolicy` is already in `IAM__POLICY_ARNS` in `provision_ec2.py` (added as part of the observability work). No additional IAM setup needed beyond deploying the container.

### 4.1 Add policy to the EC2 role

```bash
aws iam attach-role-policy \
  --role-name playwright-ec2 \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchAgentServerPolicy
```

### 4.2 What you get for free

Once the `amazon/cloudwatch-agent` container is running on the instance (Part 2 of the observability plan), the following appear in AWS Console → CloudWatch → Metrics → CWAgent namespace with zero extra config:

- EC2 instance memory used %
- EC2 disk used %
- EC2 CPU user/system/idle %
- Per-container CPU and memory (via Docker stats)
- `/var/log/sg-playwright-setup.log` shipped to CloudWatch Logs

**CloudWatch Alarms** can alert on these metrics via SNS without Grafana. Useful as a backup alerting channel.

## Troubleshooting

### AMP remote_write failing

```bash
sg-ec2 exec "docker compose -f /opt/sg-playwright/docker-compose.yml logs prometheus | grep -i remote"
```

Common causes:
- `AmazonPrometheusRemoteWriteAccess` not attached to the role
- Wrong `workspaceId` in `prometheus.yml`
- Prometheus container doesn't have network access to the AMP endpoint (check outbound SG rules)

### AMG can't connect to AMP

- In AMG data source config, ensure the workspace ARN is correct
- AMG workspace must be in the same region as the AMP workspace
- AMG service role needs `AmazonPrometheusQueryAccess` (assigned automatically when using service-managed permissions)

### OpenSearch ingestion failing

```bash
sg-ec2 exec "docker compose -f /opt/sg-playwright/docker-compose.yml logs fluent-bit | tail -50"
```

Common causes:
- Domain endpoint URL has a typo
- `es:ESHttpPost` IAM policy not attached, or wrong resource ARN
- OpenSearch domain still provisioning (check `Processing: false` in describe-domain output)

### SSM port forwarding not working

Ensure the AWS CLI has the `session-manager-plugin` installed locally:

```bash
# macOS
brew install --cask session-manager-plugin

# Linux
curl "https://s3.amazonaws.com/session-manager-downloads/plugin/latest/ubuntu_64bit/session-manager-plugin.deb" -o /tmp/smp.deb
sudo dpkg -i /tmp/smp.deb
```
