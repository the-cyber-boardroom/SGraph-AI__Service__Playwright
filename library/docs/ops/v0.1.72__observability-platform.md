## Overview

The SG Playwright EC2 stack ships a fully-integrated observability platform that collects metrics and logs from every container and forwards them to AWS managed services. Data persists across instance restarts and terminations — a new instance immediately continues populating the same time-series and log stores.

```
┌─────────────────────────────────────────────────────────────┐
│  EC2 Instance (sg-playwright stack)                         │
│                                                             │
│  ┌──────────┐  ┌────────────┐  ┌──────────┐  ┌─────────┐  │

│  └──────────┘  └────────────┘  └──────────┘  └─────────┘  │
│        │              │              │                      │
```

## Environment Variables

Set before running `sp create`. The provisioner warns at preflight if either is missing and falls back to local-only mode.

| Variable | Format | Effect when set |
|----------|--------|-----------------|
| `AMP_REMOTE_WRITE_URL` | Full URL including path (`/api/v1/remote_write`) | Prometheus remote-writes metrics to AMP |
| `OPENSEARCH_ENDPOINT` | Bare hostname — **no `https://` prefix** | Fluent Bit ships logs to OpenSearch |

```bash
# Example (replace workspace ID with yours)
export AMP_REMOTE_WRITE_URL="https://aps-workspaces.eu-west-2.amazonaws.com/workspaces/ws-<id>/api/v1/remote_write"
export OPENSEARCH_ENDPOINT="search-sg-playwright-logs-pffstr2ko4b7k7kuaxlhmykk7m.eu-west-2.es.amazonaws.com"
```

## On-Instance Containers

All containers run in the `sg-net` bridge network. Config files written to `/opt/sg-playwright/config/` by `user-data` at boot.

| Container | Image | Role |
|-----------|-------|------|
| `prometheus` | `prom/prometheus:v2.51.0` | Scrapes cadvisor + node-exporter, remote-writes to AMP |
| `cadvisor` | `gcr.io/cadvisor/cadvisor:v0.49.1` | Container CPU/mem/net metrics |
| `node-exporter` | `prom/node-exporter:v1.7.0` | Host CPU/mem/disk/net metrics |
| `fluent-bit` | `amazon/aws-for-fluent-bit:stable` | Tails container logs, ships to OpenSearch |

Config files:

| File | Written by | Mounted into |
|------|-----------|--------------|
| `/opt/sg-playwright/config/prometheus.yml` | `user-data` | `prometheus` |
| `/opt/sg-playwright/config/fluent-bit.conf` | `user-data` | `fluent-bit` |

## Querying — Quick Reference

### OpenSearch Dashboards (logs)

URL: `https://search-sg-playwright-logs-pffstr2ko4b7k7kuaxlhmykk7m.aos.eu-west-2.on.aws/_dashboards`

Useful DQL queries:

```
# All logs from a specific container
log: playwright

# Errors and warnings only
log: ERROR OR log: WARN OR log: error OR log: warning

# mitmproxy proxy traffic
log: "HTTP/" AND NOT log: "Lowercased"

# Startup logs (instance boot)
log: "SG Playwright"

# Filter by environment
environment: dev
```

### Prometheus / AMP (metrics)

Local access via SSM tunnel:
```bash
sp forward-prometheus <deploy-name>
# → http://localhost:9090
```

Useful PromQL:

```promql
# All instances that are up
up{stack="sg-playwright"}

# Container CPU usage rate (5m avg)
rate(container_cpu_usage_seconds_total{stack="sg-playwright"}[5m])

# Container memory usage
container_memory_usage_bytes{stack="sg-playwright"}

# Host free memory
node_memory_MemAvailable_bytes{stack="sg-playwright"}

# Disk usage %
1 - node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"}
```

## Observability Data Flow — Sequence

```
[EC2 boot]
    │
    ├─ user-data writes prometheus.yml + fluent-bit.conf to /opt/sg-playwright/config/
    ├─ user-data generates self-signed cert for browser-proxy nginx
    ├─ docker compose up -d (all 8 containers)
    │
    ├─ cadvisor starts → exposes :8080/metrics
    ├─ node-exporter starts → exposes :9100/metrics
    │
    ├─ prometheus starts → scrapes cadvisor:8080 + node-exporter:9100 every 15s
    │       └─ remote_write → AMP (SigV4, no credentials embedded)
    │
    └─ fluent-bit starts → tails /var/lib/docker/containers/**/*.log
            └─ adds stack=sg-playwright, environment=dev to every record
            └─ POST /_bulk → OpenSearch (AWS_Auth SigV4, no credentials embedded)

[OpenSearch Dashboards]
    └─ Discover → sg-playwright-logs index pattern → live log exploration

[Grafana AMG]
    └─ grafana-amazonprometheus-datasource → AMP → metrics dashboards
```

## Onboarding a New AWS Account

To enable the full observability stack in a fresh account:

1. **Create AMP workspace** — note the remote write URL
2. **Create OpenSearch domain** — `sg-playwright-logs`, OpenSearch 3.5 (use `sp ob create`)
3. **Set resource policy** on OpenSearch to `"AWS": "*"` (safe with FGAC enabled)
4. **Open OpenSearch Dashboards** (`/_dashboards`) with master user, map backend roles:
   - `arn:aws:iam::<account>:role/playwright-ec2`
   - `arn:aws:iam::<account>:role/service-role/AmazonGrafanaServiceRole-<id>`
5. **Attach inline policy** to `playwright-ec2` role for `es:ESHttp*` on the domain ARN
6. **Create AMG workspace** — add AMP and OpenSearch as data sources
7. **Set GH Actions secrets**: `AMP_REMOTE_WRITE_URL`, `OPENSEARCH_ENDPOINT`
8. **Export locally** and `sp create`
