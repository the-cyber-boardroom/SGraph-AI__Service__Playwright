## Lambda Deploy Pattern

**Playwright service (`scripts/provision_lambdas.py`):**

- `--mode=full` — full image pull + upsert (used after ECR push)
- `--mode=code-only` — skips image pull wait (~30–60 s saving)
- Controlled by CI job `provision-lambdas` which picks mode from `build-and-push-image` result.

**SP CLI Lambda (`deploy/provision.py`):**
```bash
python -m sgraph_ai_service_playwright__cli.deploy.provision --stage dev
```
Idempotent — role → image → Lambda. Prints Function URL on success.

## Security Group Naming Rules

**NEVER** start a `GroupName` with `sg-` — AWS reserves `sg-*` for security group IDs and rejects `CreateSecurityGroup` with `InvalidParameterValue`.

- Use suffix: `{stack}-sg` (e.g. `playwright-ec2` not `sg-playwright-ec2`)
- Use `Stack__Naming.sg_name_for_stack()` to generate compliant names
- Also: `GroupDescription` must be ASCII-only — AWS rejects multi-byte characters (e.g. em-dash)

Precedent: `scripts/provision_ec2.py:83` (`SG__NAME = 'playwright-ec2'`) and `sg_name_for_stack` helper in `Elastic__AWS__Client.py`.

---

## AWS Name Tag — No Double-Prefix

When the logical name already carries the section prefix (e.g. `elastic-quiet-fermi`), do NOT wrap it again into `elastic-elastic-quiet-fermi`. Use `Stack__Naming.aws_name_for_stack()` which prefixes only when the prefix is missing.

## Stack Naming Convention

All stack names follow `{section}-{adjective}-{scientist}` pattern (e.g. `elastic-quiet-fermi`).
Random names generated by `Random__Stack__Name__Generator` (matching pools across sections by design).

## Secrets — Where They Live

- **AWS credentials:** GH Actions repository secrets (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_DEFAULT_REGION`)
- **API keys:** GH Actions secrets (`FAST_API__AUTH__API_KEY__VALUE`)
- **Vault keys:** shared out-of-band — NEVER committed to git
- **`.env` files:** template only in `.env.example` — never commit actual keys
