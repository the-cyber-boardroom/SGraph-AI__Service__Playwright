# Decisions Log — Playwright Service

**Session:** Librarian, 16 April 2026
**Participants:** Dinis Cruz (project lead), Claude (Librarian role)

All decisions made during the specification session that produced this dev pack.

---

| # | Decision | Rationale | Alternatives considered |
|---|---|---|---|
| 1 | **Vault key format** supports both `"drum-hunt-6610"` (friendly) and `"j4pyy0lhny8jx7osqn4lclhq:mzrp0li8"` (opaque) | Both formats are in active use | Single format only |
| 2 | **No Literals anywhere** — all fixed-value sets use Enum__* classes | Enums provide runtime validation, IDE discoverability, and consistent serialisation | Literal for small inline sets |
| 3 | **`action` field on Schema__Step__Base** as the shared discriminator | Avoids repeating the discriminator pattern in every subclass | action only on subclasses |
| 4 | **Per-action result subclasses** (Schema__Step__Result__Get_Content, etc.) | Type safety for return values; `Any` only survives on evaluate's return_value | Single result schema with `result_value: Any` |
| 5 | **Capture config validation per-action**, not at session create | Late-binding allows sink changes mid-sequence; more flexibility | Validate once at session create |
| 6 | **Duplicate step IDs rejected** (422 with `duplicate_step_ids`) | Step IDs used as keys in artefact maps; duplicates cause data loss | Allow duplicates (last-write-wins) |
| 7 | **launch_args: caller replaces defaults entirely** | Callers who know what they're doing get full control; service logs the override | Merge with defaults; reject custom args |
| 8 | **Separate lambda_handler.py** from service class | Service class importable without side effects; critical for test injection via `register_*__in_memory()` | Module-level singletons in service class file |
| 9 | **No raw dicts returned from any route or service method** | Every return is `.json()` on a Type_Safe schema; added Schema__Session__State__Save__Response | Raw dicts for simple confirmations |
| 10 | **Playwright__Service refactored into 5 dedicated classes** | Single responsibility; orchestrator is thin glue only. Request__Validator, Action__Runner, Sequence__Runner, Credentials__Loader extracted | Monolithic orchestrator |
| 11 | **Capability__Detector uses Type_Safe constructors** — one method per deployment target | No intermediate dicts; each `capabilities__{target}()` returns Schema__Service__Capabilities directly | Dict-based config lookup |
| 12 | **Schema versions not in schemas** — `/info` endpoint exposes versions; client package handles compatibility | Simpler schemas; version skew managed at client level | Version field on every schema |
| 13 | **List[dict] for steps on the wire** | Type_Safe auto-casting would lose data with heterogeneous step types; dispatcher parses per-step via STEP_SCHEMAS registry | Union type; List[Schema__Step__Base] |
| 14 | **Safe_Str__Trace_Id** (hex + hyphens, 32-char max) replaces generic Trace_Id | Safe_Str's default regex is aggressive; trace IDs need lowercase hex | Reuse Safe_Str directly |
| 15 | **100% clean state per call** — every `/quick/*` and `/sequence/execute` request starts a fresh `sync_playwright` Node subprocess AND a fresh Chromium, torn down via try/finally at end of request. No singleton runtime, no pooling. Launcher registry asserted empty after 20 sequential calls. *(v0.1.13, 2026-04-17 — commits `8922484`, `cdba60c`, `22dd98a`, `6869632`, `906ba05`)* | Eliminates cross-request state bleed entirely — proxies, cookies, `extra_http_headers`, stuck handles, thread affinity cannot leak into the next caller. The ~550 ms fixed overhead per call buys a security/correctness invariant that would otherwise cost a debugging session months later when state from one request contaminated a sensitive scrape. Well inside the 30 s API Gateway ceiling. | Singleton `sync_playwright` with per-session Browser instances (classic pooling) — would share Node subprocess state across requests and defeat the isolation guarantee. Deferred optimisations (Lambda provisioned concurrency, browser pre-warm, BROWSERLESS / CDP_CONNECT) only to be introduced when a concrete workload demands them. |
| 16 | **Per-phase timings on every sequence response** — `Schema__Sequence__Timings` (`playwright_start_ms`, `browser_launch_ms`, `steps_ms`, `browser_close_ms`, `total_ms`). JSON field on `/sequence/execute` + `/quick/html`; HTTP `X-*-Ms` headers on `/quick/screenshot` since its body is raw PNG. *(v0.1.13, 2026-04-17)* | Lets callers see at a glance where latency went — Chromium boot? page load? teardown? — without instrumenting their own stopwatch. Makes cold-start budgeting data-driven instead of a guess. | Surface nothing and force callers to time the round trip themselves. Rejected — the information costs the service nothing to emit. |
| 17 | **Payload-size primitives lifted to 10 MB / 20 MB** — `Safe_Str__Page__Content` (HTML, 10 MB) and `Safe_Str__Artefact__Inline` (base64, 20 MB) replace the `Safe_Str__Text__Dangerous` 64 KB default on those two fields. *(v0.1.13, 2026-04-17)* | Real page HTML runs 100 KB – 5 MB; a full-page PNG routinely pushes past 64 KB. The default cap was silently truncating real data. Two narrow primitives lift the cap without weakening the dangerous-content regex elsewhere. | Lift `Safe_Str__Text__Dangerous` globally — rejected; would weaken the guard on free-text fields where 64 KB is correct. |
| 18 | **Swagger-default `timeout_ms=0` treated as user input** — accept the zero, skip setting a per-action timeout rather than zero-timing every step. *(v0.1.13 hotfix, 2026-04-17 — commit `6869632`)* | Swagger UI auto-fills numeric defaults as `0` when the caller doesn't override them. Treating that as "user said 0" and applying it to `page.set_default_timeout` made every action time out instantly. Fix: `if timeout_ms is not None and int(timeout_ms) > 0`. | Raise 422 on `timeout_ms=0` — rejected, breaks Swagger's "Try it out" flow. |
