# Local Proxy-Auth Validation — Laptop Setup

Run the authenticated-proxy L1–L4 test pyramid on your laptop with Docker. This is the same setup used to validate Bug #1 (CDP Fetch wiring) and the one we need to support as a first-class developer workflow going forward.

**Goal:** prove end-to-end that credentials travel correctly from `Schema__Proxy__Auth__Basic` → `Browser__Launcher` → `Proxy__Auth__Binder` → Chromium → upstream proxy, without depending on a deployed Lambda.

---

## What you're standing up

```
┌──────────────────────────────┐        ┌───────────────────┐       ┌──────────────┐
│ pytest (L1/L2/L3/L4)         │ ──►    │ mitmproxy :8888   │ ──►   │  ipify.org   │
│  or the service in Docker    │        │  proxyauth on     │       │  (probe)     │
└──────────────────────────────┘        └───────────────────┘       └──────────────┘
        │
        └── real Chromium (Playwright)
```

- **mitmproxy** runs in Docker on `:8888`, required basic-auth `qa-user:qa-pass-x7`.
- **Chromium** runs on the host (launched by Playwright sync API).
- Optionally, the **service itself** runs in Docker on `:8000` and drives the same proxy when you hit it over HTTP.

---

## Prereqs

| Tool                                        | Version         | Notes |
|---------------------------------------------|-----------------|-------|
| Docker (rootless or daemon)                 | 20+             | Only needed for mitmproxy and the service image |
| Python                                      | 3.12            | Keep a dedicated venv — don't reuse your system Python |
| Playwright browsers                         | Chromium 1208   | Installed into the venv; absolute path is exported as an env var |
| `mitmdump` (optional, if you prefer host)   | any recent      | Docker path below is preferred so you don't pollute the host |

No AWS creds needed. No vault access needed. Network egress to ipify.org is enough.

## 1. Create a clean Python 3.12 venv

The repo pins 3.12 and assumes `osbot-utils` + `osbot-fast-api-serverless` + `playwright` are on the path. A shared venv collects stale dependencies fast, so make a fresh one per slice:

```bash
python3.12 -m venv /tmp/py312-sgp
/tmp/py312-sgp/bin/pip install -U pip
/tmp/py312-sgp/bin/pip install -r requirements.txt
/tmp/py312-sgp/bin/pip install -r requirements-test.txt
```

Then install the Playwright browsers **into the same venv**:

```bash
/tmp/py312-sgp/bin/playwright install chromium
```

The install lands under `/opt/pw-browsers/chromium-*` (Linux) or `~/Library/Caches/ms-playwright/chromium-*` (macOS). Record the absolute path to the `chrome` binary — you'll export it in step 4.

> **Gotcha — `osbot_docker` isn't in `requirements.txt`.** The docker-tier tests (`tests/docker/*`) lazy-import it inside `Create_Image_ECR.__init__`. You can ignore this for L1–L4; if you need the docker tier later, `/tmp/py312-sgp/bin/pip install osbot-docker` clears it.

---

## 2. Start mitmproxy in Docker

One-liner:

```bash
docker run --rm -it --name sgp-mitmproxy \
  -p 8888:8080 \
  mitmproxy/mitmproxy:latest \
  mitmdump --listen-port 8080 --proxyauth qa-user:qa-pass-x7
```

- Container port `8080` (mitmproxy's default), host port `8888` (what the tests expect).
- `--proxyauth` enforces basic-auth — this is what makes the tests *meaningful*; without it, wrong-creds wouldn't fail differently from right-creds.
- `--rm -it` keeps it foreground so Ctrl-C tears it down cleanly. For long-lived use add `-d` and drop `-it`.

Keep this running in one terminal. In another terminal, sanity-check:

```bash
curl -x http://qa-user:qa-pass-x7@127.0.0.1:8888 -k https://api.ipify.org/?format=json
# → {"ip":"<your egress IP>"}
```

If this fails: no test will pass. Fix the proxy first.

---

## 3. Install the mitmproxy CA (only if you want HTTPS probes without `ignore_https_errors`)

mitmproxy is a TLS interceptor and ships its own CA. The test suite uses `ignore_https_errors: true`, so you don't need to trust the CA for CI-style test runs. Install it only if you want to browse through mitmproxy manually:

```bash
# On first mitmdump run the CA is written to ~/.mitmproxy/mitmproxy-ca-cert.pem (on the host if you bind-mount the dir, or in the container otherwise).
```

**Skip this step for L1–L4.** `ignore_https_errors` is baked into every test's proxy config.

---

## 4. Export env vars

```bash
export SG_PLAYWRIGHT__CHROMIUM_EXECUTABLE=/opt/pw-browsers/chromium-1208/chrome-linux64/chrome   # ← your actual path
# Adjust the version (1208) to whatever `playwright install chromium` produced.
```

`Browser__Launcher` honours this and skips its normal discovery, which is exactly what you want on a laptop without the Playwright image's baked-in browser layout.

No API key is needed for L1 or L2 (they don't hit the HTTP layer). L3 and L4 set `FAST_API__AUTH__API_KEY__NAME` + `FAST_API__AUTH__API_KEY__VALUE` themselves inside the test via `_EnvScrub`, so you don't need to export those either.

## 5. Run the tier you want

From the repo root:

```bash
# L1 — raw Playwright + CDP Fetch (no service code)
/tmp/py312-sgp/bin/pytest tests/local/test_L1__pure_playwright_cdp.py -v

# L2 — service-in-process against the same proxy (real Chromium, no HTTP)
/tmp/py312-sgp/bin/pytest tests/local/test_L2__service_in_process_proxy.py -v

# L3 — FastAPI TestClient end-to-end over POST /sequence/execute
/tmp/py312-sgp/bin/pytest tests/local/test_L3__local_http_proxy.py -v

# L4 — credential variants (THE regression probe for Bug #1)
/tmp/py312-sgp/bin/pytest tests/local/test_L4__credential_variants.py -v

# All four in one shot
/tmp/py312-sgp/bin/pytest tests/local/ -v
```

Each test file gates on mitmproxy reachability + `SG_PLAYWRIGHT__CHROMIUM_EXECUTABLE`, so if either is missing you get a clean `SKIP`, not a confusing failure.

### What "green" looks like

- **L1** — 3/3 PASSED, each under ~4 s.
- **L2** — 2/2 PASSED. Correct creds navigate; wrong creds fail in <10 s.
- **L3** — 1/1 PASSED. `/sequence/execute` completes with `steps_passed == 2` and ipify JSON in the response body.
- **L4** — 1/1 PASSED. `correct_result['status'] == 'completed'`; `wrong_user`, `wrong_pass`, `empty` all `!= 'completed'` AND each took `< 10 s`.

The wall-clock differentiation is the critical assertion — the Lambda regression signature was **every** variant hitting a uniform ~5.5 s timeout, proving creds never reached Chromium. If your laptop run reproduces that, the CDP wiring is broken again.

## 6. (Optional) Run the service image in Docker, driving the same proxy

For a true "how would Lambda behave" rehearsal, run the real container and hit it over HTTP. This is the closest thing we have to L5 without AWS.

```bash
# Build (from the repo root)
/tmp/py312-sgp/bin/python - <<'PY'
from sgraph_ai_service_playwright.docker.Build__Docker__SGraph_AI__Service__Playwright import Build__Docker__SGraph_AI__Service__Playwright
Build__Docker__SGraph_AI__Service__Playwright().setup().build()
PY

# Run (note the port shuffle — mitmproxy already owns 8888)
docker run --rm -d --name sgp-service \
  --add-host=host.docker.internal:host-gateway \
  -p 8000:8000 \
  -e FAST_API__AUTH__API_KEY__NAME=X-API-Key \
  -e FAST_API__AUTH__API_KEY__VALUE=local-validation-key \
  -e SG_PLAYWRIGHT__DEPLOYMENT_TARGET=laptop \
  sgraph_ai_service_playwright:latest
```

> **Why `host.docker.internal`?** The proxy `server` field inside the request body needs to be an address the *container* can reach. `127.0.0.1` inside the container means "the container itself", not your laptop. Use `http://host.docker.internal:8888` on macOS/Windows; Linux needs the explicit `--add-host=host.docker.internal:host-gateway`.

Smoke it:

```bash
curl -X POST http://127.0.0.1:8000/sequence/execute \
  -H 'X-API-Key: local-validation-key' \
  -H 'Content-Type: application/json' \
  -d '{
    "browser_config": {
      "proxy": {
        "server"             : "http://host.docker.internal:8888",
        "ignore_https_errors": true,
        "bypass"             : [],
        "auth"               : {"username": "qa-user", "password": "qa-pass-x7"}
      }
    },
    "capture_config" : {},
    "sequence_config": {"halt_on_error": true},
    "close_session_after": true,
    "steps": [
      {"action": "navigate"   , "url": "https://api.ipify.org/?format=json", "timeout_ms": 15000},
      {"action": "get_content", "inline_in_response": true}
    ]
  }'
```

A successful response has `"status":"completed"`, `"steps_passed":2`, and the ipify JSON inside the `get_content` step's `content` field.

Tear down:

```bash
docker rm -f sgp-service sgp-mitmproxy
```

## Troubleshooting cheat-sheet

| Symptom | Likely cause | Fix |
|--------|--------------|-----|
| Test SKIPs with *"mitmdump not running"* | Container didn't start, or host port ≠ 8888 | Re-check `docker ps`; `-p 8888:8080` mapping |
| Test SKIPs with *"SG_PLAYWRIGHT__CHROMIUM_EXECUTABLE not set"* | Env var missing in the pytest process | Export it in the **same shell** you run pytest from |
| L3 / L4 fail with *"cannot switch to a different thread (which happens to have exited)"* | You added a multi-request flow in the test | Use single-request `/sequence/execute` — Playwright sync is thread-bound; TestClient changes threads per request |
| Wrong creds succeed (or produce identical timing to right creds) | CDP wiring is broken — creds aren't reaching Chromium | This is **exactly** the Bug #1 regression. Check `Proxy__Auth__Binder.bind()` is called at page creation and that `Playwright__Service.setup()` wires it into both runners |
| Credentials with hyphens/dots get corrupted (e.g. `qa-user` → `qa_user`) | Schema field is typed as osbot `Safe_Str__Username` | Must be `Safe_Str__Basic_Auth__Username` / `__Password` (they preserve hyphens, dots, punctuation) |
| Container navigate hangs to timeout against `127.0.0.1:8888` | Container can't reach the host | Use `host.docker.internal` + `--add-host=host.docker.internal:host-gateway` on Linux |
| `NET::ERR_CERT_AUTHORITY_INVALID` in page errors | Proxy is a TLS interceptor but `ignore_https_errors` is false | Set `ignore_https_errors: true` in the proxy config (required for mitmproxy) |
| `osbot_docker` ModuleNotFoundError on docker-tier tests | Lazy import inside `Create_Image_ECR.__init__` — not in `requirements.txt` | `pip install osbot-docker` into the venv (only needed if you run `tests/docker/*`) |

## When to re-run this

- Before any PR that touches `Browser__Launcher`, `Proxy__Auth__Binder`, `Action__Runner`, `Sequence__Runner`, or any schema under `schemas/browser/` or `schemas/primitives/auth/`.
- Before bumping the Playwright base image version.
- Before changing how `Playwright__Service.setup()` wires dependencies.
- Whenever a QA round against dev Lambda reports a proxy-related symptom — confirm on laptop first, in this harness, before assuming it's environment-specific.
