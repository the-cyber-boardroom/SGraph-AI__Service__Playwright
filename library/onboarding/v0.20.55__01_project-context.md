**Version:** v0.20.55

## What is SG/Send?

SG/Send is a zero-knowledge encrypted file sharing platform at [send.sgraph.ai](https://send.sgraph.ai). Files are encrypted in the browser using AES-256-GCM before upload. The decryption key never leaves the sender's device. The server only stores encrypted ciphertext.

## What is the Playwright Service?

The Playwright service is a **browser automation API** for the SG/Send ecosystem. It provides:

1. A **declarative step language** — callers send JSON describing browser actions (navigate, click, fill, screenshot, etc.), the service executes them
2. **Vault integration** — load cookies/credentials from encrypted vaults, write artefacts (screenshots, videos) back to vaults
3. **Runs-everywhere deployment** — the identical Docker image runs on 5 deployment targets with capabilities auto-detected per environment

## The 5 Deployment Targets

| Target | Constraints | Use case |
|---|---|---|
| **Developer laptop** | Full control; headful possible; persistent disk; any duration | Interactive development, debugging |
| **CI pipeline** (GitHub Actions) | Headless; ephemeral; 10-min limit; network-allowlisted | Automated testing, QA runs |
| **Claude Web / Claude Code** | Headless; ephemeral; `/tmp` only; TLS-intercepting proxy | AI-driven browser automation |
| **Container / K8s / Fargate** | Headless; long-lived; persistent disk via volumes | Production content pipelines |
| **AWS Lambda** (container) | Headless; 15-min max; `/tmp` only; 5 GB memory | Serverless QA, on-demand automation |

The service exposes a `GET /health/capabilities` endpoint that returns what the current deployment can do. Callers check capabilities before sending requests — the service rejects incompatible requests with actionable error messages.

## How It Fits in the SG/Send Ecosystem

```
SGraph-AI__App__Send              SG_Send__QA                 SG Playwright Service (NEW)
(main repo)                       (QA test suite)             (this project)
│                                 │                           │
├── User Lambda (transfers)       ├── Fast_API__QA__Server    ├── Fast_API__Playwright__Service
├── Admin Lambda (management)     ├── QA_API__Runner          ├── Playwright__Service
├── UI packages                   ├── Routes__Browser         ├── Routes__Browser (Layer 0)
└── Vault routes                  └── Routes__Workflow        ├── Routes__Sequence (Layer 3)
                                                              └── Routes__Session
```

The Playwright service **replaces and generalises** the browser automation capabilities currently split between SG_Send__QA (QA-specific) and OSBot-Playwright (general but unmaintained). It becomes the single browser automation API for all callers — QA tests, content pipelines, Claude sessions, and production workflows.

## Key Technical Decisions

- **Microsoft Playwright base image** — includes Chromium + all system deps (~2 GB)
- **AWS Lambda Web Adapter** (not Mangum) — HTTP translation at the container level, same image works everywhere
- **Declarative step language** (not arbitrary Python exec) — 16 action types, JSON on the wire, typed schemas on both sides
- **Evaluate action is allowlist-gated** — the 04/09 architect security constraint
- **Per-action capture validation** — each step that triggers artefact capture validates the sink against deployment capabilities
- **Stateless client package** — follows the v0.34.0 unified service client architecture
